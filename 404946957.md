## 생성 기능

- form 다루는 방법

- Create 추가 절차

  1.  create link 생성(\<a href=”/create”…)

  2.  setMode('CREATE') 정의

  3.  App() 함수에 else if Mode='Create' 정의

  4.  Create 함수 정의 (function Create(){) =\> form 정의

  5.  form의 onSubmit 이벤트 로직 정의 (event.preventDefault() 선언 → submit 시 페이지 리로드 않하도록 처리)

  6.  form 의 onSubmit 에서 title과 body 값을 props의 onCreate() 에 전달

  7.  Create에서 **추가한 항목들이 페이지의 목록**에 나오도록 하려면 **APP 함수를 리로드**해야 함. 따라서 **topics를 state로 업그레이드** 필요 =\> const \[topics, setTopics\] = useState(\[…\]);

  8.  App 함수의 else if(mode ==='CREATE') 에 **Create 태그에 onCreate()** 정의

  import {useState} from 'react';  

  function Article(props) {
    return <article>
    <h2>{props.title}</h2>
    {props.body}
    </article>
  }

  function Header(props) {
    return <header>
    <h1><a href="/" onClick={(event)=>{
      event.preventDefault(); 
      props.onChangeMode();  
    }}>{props.title}</a></h1>
    </header>
  }

  function Nav(props) {
    
    const list = []
    for(let i = 0; i < props.topics.length; i++) {
      let t = props.topics[i];
      list.push(<li key={t.id}><a id={t.id} href={'/read/'+ t.id} onClick={event=>{
        event.preventDefault();
        props.onChangeMode(Number(event.target.id);  
      }}>{t.title}</a></li>)
    }
    return <nav>
    <ol>
      {list}
    </ol>
    </nav>
  }

  function Create(props) {
    return <article>
    <h2>Create</h2>
    <form onSubmit={event=>{
      event.preventDefault();
      const title = event.target.title.value;
      const body = event.target.body.value;
      props.onCreate(title, body);
    }}>
    <p><input type="text" name="title" placeholder="title"/></p>
    <p><textarea name="body" placeholder="body"></textarea></p>
    <p><input type="submit" value="Create"></input></p>
    </form>
    </article>
  }

  function App() {
    const [mode, setMode] = useState('WELCOME');   
    const [id, setId] = useState(null);  
    const [nextId, setNextId] = useState(4);
    
    const [topics, setTopics] = useState([
      {id:1, title:'html', body: 'html is ...'},
      {id:2, title:'css', body: 'cssis ...'},
      {id:3, title:'javascript', body: 'javascript is ...'}
    ]);
    let content = null; 
    if(mode === 'WELCOME') {
      content = <Article title="Welcome" body="Hello, WEB"></Article>
    } esle if(mode === 'READ') {
      let title, body = null;
      for(let i = 0; i< topicslength; i++){
        console.log(topics[i].id, id);
        if(topics[i].id === id) {
          title = topics[i].title;
          body = topics[i].body;
        }
      }
      content = <Article title={title} body={body}></Article>
    } else if(mode === 'CREATE') {
      content = <Create onCreate={(_title, _body) =>{
        const newTopic = {id:nextId, title:_title, body:_body}
        const newTopics = [...topics]  // 목록을 복제해서 변경 후 처리 
        newTopics.push(newTopic);
        setTopics(newTopics);
        setMode('READ');
        setId(nextId);
        setNextId(nextId+1);
      }}></Create>
    }
    return (
      <div>
        <Header title="WEB" onChangeMode={() =>{
          setMode('WELCOME');
        }}></Header>
        <Nav topics={topics} onChangeMode={(_id)=>{
          setMode('READ'); 
          setId(_id);
        }}></Nav>
        {content}
        <a href="/create" onClick={event=>{
          event.preventDefault();
          setMode('CREATE');
        }}>Create</a>
      </div>
    );
  }

  export default App;

- const\[value, setValue\] = useState(PRIMITIVE) : string, number, bigint, boolean, undefined, symbol, null

- const\[value, setValue\] = useState(Object) : object, array

  - 다음 절차대로 처리(original data는 변경하지 않도록 처리)

    1.  newValue = {…value} : 복사(… 이 복제 명령어). 배열이면 \[…value\] 로 복제

    2.  newValue 변경 : 복사한 객체를 변경

    3.  setValue(newValue) : 변경된 값을 설정

  - 이렇게 하는 이유는 react는 같은 데이터면 굳이 rendering 안함

- 테스트 예제

  - 배열 테스트

    - const\[value, setValue\] = useState(\[1\]); 배열 정의

    - value.push(2); // 오리지널 데이터(배열)를 변경

    - setValue(value); // 오리지널 데이터(배열)를 입력. setValue를 호출할 시 오리지널 데이터와 새로 들어온 데이터가 같으면 렌더링 안함 =\> 배열이나 object는 value 를 비교하는 것이 아닌 참조값을 비교해서 같은 데이터라 인식

  - primitive 테스트

    - const\[value, setValue\] = useState(1); // primitive 데이터 정의

    - setValue(2); // 새로운 데이터는 2

## 수정 기능

- update 기능 추가 절차

  1.  상세보기에서 update 할 수 있도록 “READ” 모드에서만 나오도록 \<update\> 링크 추가

  2.  setMode('UPDATE') 추가

  3.  ‘UPDATE’ 일 때 로직 추가 Update function 추가 및 if 로직 추가

  4.  이전 값을 가져와서 처리할 때 props로 처리하면 수정 화면에서 값이 입력 안 됨 =\> state로 정의하고 onChange 로 처리해야 함 (function Update 정의) → **개념 중요**

## 삭제 기능
