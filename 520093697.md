# Tomcat Accesslog field

- 기본형 : ‘%{yyyy-MM-dd HH:mm:ss}t %s %r %{User-Agent}i %{Refer}i %a %b %D’

  - 예: \[2023-07-04 17:12:50\] 200 GET /test/healthcheck HTTP/1.1 PostmanRuntime/7.32.3 {Referer}i 0:0:0:0:0:0:0:1 42 1058

- 필드 정의

  - %a: Remote Ip address

  - %A: Loca IP address

  - %b: Bytes sent, excluding HTTP headers, or '-' if zero

  - %B: Bytes sent, excluding HTTP headers

  - %h: Remote host name (or IP address if enableLookups for the connector is false)

  - %H: Request protocol

  - %l: Remote logical username from identd (alway returns '-')

  - %m: Request method (GET, POST, etc)

  - %p: Local port on which this request was received. See also %{xxx}p below

  - %q: Query string (prepended with a '?' if it exists)

  - %r: First line of the request (method and request URI)

  - %s: HTTP status code of the response

  - %S: User session ID

  - %t: Date and time, in common Log Format

  - %u: Remote user that was authenticated (if any), els '-'

  - %U: Requested URL apth

  - %v: Local server name

  - %D: Time taken to process the request, in millis

  - %T: Time taken to process the request, in seconds

  - %F: Time taken to commit the respone, in millis

  - %I: Current request thread name (can compare later with stacktraces)

- request, response head의 parameter를 추출

  - %{xxx}i : write value of incoming header with name xxx (reqeust header)

  - %{xxx}o : write value of outgoing header with name xxx (response header)

  - %{xxx}c : write value of cookie with name xxx

  - %{xxx}r : write value of ServletRequst attribute with name xxx

  - %{xxx}s : write value of HttpSession attribute with name xxx

  - %{xxx}p : write local (server) post (xxx=local) or remote (client) port (xxx=remote)

  - %{xxx}t: write timestamp at the end of request formatted using the enchned SimpleDateFormat pattern xxx

## Spring boot 설정

- server.tomcat.access.pattern=%{X Forwarded For}i %h %l %u %t “%r” %s %b %v “%{Referer}i” “%{User-Agent}i”

  - 필드 정의에 ““ 로 감싼 이유는 regex를 사용하여 추출할 때 용이하기 위해서 사용(일반적으로 각 필드간 공백으로 split 할 수 있는데 User-agent 같은 경우는 내용 내에 공백이 많이 사용되어 user-agent만 추출하기 힘들기 때문에 ““ 으로 감싸서 regex를 적용하면 수월함.

- common : %{X-Forwarded-For}i %l %u %t “%r” %s %b

- combined : %{X-Forwarded-For}i %l %u %t “%r” %s %b %{User-Agent}i %{Referer}i

- response header에 파라미터를 태울 경우 key를 다음과 같이 정의

  - %{tenantId}o

- 실제 사용 사례

  - as-is : ‘%t %h%r %s %p %l %u %{X-Forwarded-For}i %{Referer}i “%{User-Agent}i” %b %D %{typeId}o %{typeNm}o’

  - to-be: ‘%t %h%A %r %q %s %p %l %u %S %{X-Forwarded-For}i %{Referer}i “%{User-Agent}i” %b %D %F %{typeId}o %{typeNm}o’

    - %A : local ip → server ip

    - %q: query String → request URL

    - \$S: User session id

    - %F: Time taken to commit response
