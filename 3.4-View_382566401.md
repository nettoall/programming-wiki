# View

page 424

- MVC아키텍처에서 DispatcherServlet에서 View정보 전달하는 주요 2가지 방법

  - View타입 Object를 ModelAndView타입에 담아 전달

  - View이름 전달 → View renderer가 view이름으로 처리

- Spring View interface

<div class="code panel pdl" style="border-width: 1px;">

<div class="codeContent panelContent pdl">

``` syntaxhighlighter-pre
org.springframework.web.servlet;
public interface View {

  String getContentType();
  void render(Map<String, ?> model, HttpServletRequest request, HttpServletResponse response) throws Exception;
}
```

</div>

</div>

- View사용

  - API 등으로 View로직 작성

  - JSP나 FreeMarker같은 템플릿 파일을 사용 하거나 Model을 자동으로 View로 전환하는 로직 사용

## InternalResourceView 와 JstlView

- RequestDispatcher의 forward()나 include() 이용하는 view

- forward() : 다른 servlet을 실행해서 그 결과를 현재 서블릿의 결과로 사용하거나 추가

- Servlet을 forward()용도로 사용하는 경우, JSP서블릿으로 JSP뷰 적용할 때 사용

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  View view = new InternalResourceView("/WEB-INF/view/hello.jsp"); //컨트롤러가 사용할 view정보를 생성
  return new ModelAndView(view, model);
  ```

  </div>

  </div>

- InternalResourceView가 include()로 사용하려면 alwaysInclude=true로 설정 하거나 view를 호출하기 전에 HttpServletResponse에 결과가 쓰여진 경우에는 include() 사용

- JstlView는 InternalResourceView의 서브 클래스

  - JSP를 뷰 템플릿으로 사용시 JstlView로 여러가지 추가 기능 적용 가능

    - 예) 지역화 fmt태그 \<fmt:message)

- ViewResolver

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  return new ModelAndView("/WEB-INF/view/hello.jsp", model);
  ```

  </div>

  </div>

  - view resolver 사용시 뷰이름 정의 (“/WEB-INF/view/hello.jsp”)

  - Controller에서 view object생성하여 사용하는 것보다 view resolver사용이 편함 (InternalResourceView, JstlView등)

## RedirectView

- HttpServletResponse의 sendRedirect()호출해 주는 기능

- 실제 View 생성이 아닌 redirect만 처리

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  return new ModelAndView(new RedirectView("/main"));
  return new ModelAndView("redirect:/main");
  ```

  </div>

  </div>

- context relative 는 true 로 상대 경로로 작성 가능

- redirect주소는 appliication path부터 서술

- contetxtRelative=true, false로 상대경로, 절대경로 선택

## VelocityView, FreeMarkerView

- 자바 템플릿 엔진 View

  - 직접 view 오브젝트 생성보다 View resolver사용이 편리

  - 테스트시에도 일반 JSP사용시 ModelAndView의 정확한 결과를 알기 어려우나 템플릿 엔진 View 는 정확한 결과를 알 수 있음

## MarshallingView

page 430

- OXM(Object-XML Mapping) : 추상화 기능을 활용하여 application/xml 타입의 XML content 작성하는 view

- marshaller Bean, 모델에서 변환에 사용한 오브젝트 지정 등 OXM마샬러 부터 변환에 관련된 정보 설정 필요

- OXM 을 자동처리해 주는 castor라이브러리 등 사용

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  <bean id="marshaller" class="org...CastorMarshaller"/>
  <bean id="helloMarshallingView" class="org...MarshallingView">
    <property name="marshaller" ref="marshaller"/> // model marshaller 정의
    <property name="modelKey" value="info"/>
  </bean>
  ```

  </div>

  </div>

<div class="code panel pdl" style="border-width: 1px;">

<div class="codeContent panelContent pdl">

``` syntaxhighlighter-pre
public class HelloController implements Controller {
  @Resource MarshallingView helloMarshallingView;
  public ModelAndView handleRequest(...) {
    ...
    return new ModelAndView(helloMarshallingView, Model);
  }
}  
```

</div>

</div>

- MarshallingView Object 를 Controller 에서 생성해서 사용하는 방법보다는 View Resolver를 사용하는 것이 나음

## AbstractExcelView, AbstractJExcelView, AbstractPdfView

- Excel, Pdf 생성 view

  - AbstractExcelView : Apache Poi 사용

  - AbstractJExcelView : Jexcel API사용

  - AbstractPdfView : iText프레임워크 API로 PDF생성

- Abstract view상속받아 view클래스를 구현하고 Bean등록하여 사용

- Controller에 DI하거나 View Resolver를 통해 매핑

- page 433 PDF view만드는 예제

## AbstractAtomFeedView, AbstractRssFeedView

- application/atom+xml, application/rss+xml 타입 feed 문서 생성 view

- hook메소드 구현하여 생성

## XsltView, TitlesView, AbstractJasperReportsView

- XSLT 변환을 이용해 view 생성 → javax.xml.transform.TransFormerFactory타입의 factory class와 적용할 Model Object 지정

- TilesView : Tiles 1, 2를 이용해 View생성

- report 작성용 프레임워크 Jasper Report로 csv, html, pdf, excel report생성

## MappingJacksonJsonView

- JSON 타입의 content작성해 주는 View

# View Resolver

- DispatcherServlet default view resolver : InternalResourceViewResolver

- 하나 이상의 view resolver등록 가능

- order로 우선순위 설정 가능

## InternalResourceViewResolver

- JSP를 view로 사용

- default상태인 경우, view의 full path(“/WEB-INF/view/…)로 정의해야 함

- prefix, suffix property설정을 하려면 Bean등록해야 함.

- DispatcherServlet전략 중 RequestToViewNameTranslator → 뷰 이름을 controller가 넘겨주지 않으면 URL을 이용해 자동으로 뷰 이름 생성

- JSTL라이브러리가 클래스 패스에 존재하면 JSTL의 부가 기능을 지원하는 jstlview사용

- 존재하지 않으면 inteernalResourceView사용

## VelocityViewResolver, FreeMakerViewResolver

- 템플릿 엔진 기반 view

- 템플릿 경로 만들 때 사용할 root path를 미리 velocityConfigurer나 FreeMarkerconfigurer로 지정 → prefix는 잘 사용하지 않음

## ResourceBundleViewResolver, XmlViewResolver, BeanNameViewResolver

- Controller 마다 view의 종류가 달라질 수 있을 때 외부 리소스 파일에 각 뷰 이름에 해당하는 View class와 설정을 담아두고 이를 참조하는 ResourceBundleViewResolver와 XmlViewResolver사용

- properties나 xml에 정의된 class와 view url 매핑 참고하는 view resolver

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  # properties
  hello.(class)=org.springframework.web.servlet.view.JstlView
  hello.url=/WEB-INF/view/hello.jsp
  ```

  </div>

  </div>

- 모든 view를 properties에 관리하는 건 현실적으로 비효율적임 → 특별한 케이스에 대해서만 properties로 처리하고 나머지는 InternalResourceViewResolver가 처리하도록 같이 사용하는게 나음

### XMLViewResolver

- /WEB-INF/views.xml 참고

### BeanNameViewResolver

- view이름과 동일한 bean이름을 가진 bean을 찾아 view 사용

## ContentNegotiatingViewResolver

- Mediatype정보 활용하여 다른 view resolver에게 view를 찾도록 위임

- view resolver를 결정해주는 resolver

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  if("xml".equals(req.getParameter("type"))) {
    return new ModelAndView(helloMarshallingView, model);
  } else {
    return new ModelAndView("/WEB-INF/view/hello.jsp", model);
  }
  ```

  </div>

  </div>

- 후보 view 설정(mediatype위임)

  - 미디어 타입으로 각각의 view resolver로 분기 위임시킴

  - view 후보에 사용할 view resolver는 ViewResolver Property로 지정가능함
