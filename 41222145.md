# Encoding

## 정의

- 데이터의 형태나 형식을 변환하는 처리나 처리 방식

- 동일한 알고리즘을 사용하여 되돌릴 수 있음

- 키가 사용되지 않음

- ex) ASCII, UNICODE, URL Encoding, Base64

## Base64

- 8비트 이진 데이터를 공통 ASCII영역의 문자들로만 이루어진 일련의 문자열로 바꾸는 인코딩 방식

- 인코딩된 문자열은 알파벳 대소문자와 숫자 그리고 “+”, “/” 기호 64개로 이루어지며 “=”는 끝을 알리는 코드로 쓰임

## sample

<div class="code panel pdl" style="border-width: 1px;">

<div class="codeContent panelContent pdl">

``` syntaxhighlighter-pre
void Base64Sample(String text) throws UnsupportedEncodingException{
  Encoder encoder = Base64.getEncoder();
  String encodedString = encoder.encodeToString(text.getBytes("UTF-8"));
  
  Decoder decoder = Base64.getDecoder();
  byte[] decodedBytes = decoder.decode(encodedString);
  String decodedString = new String(decodedBytes,"UTF-8");
}
```

# Encryption

## 정의

- 데이터의 기밀성을 유지하기 위해 사용

  - 원래의 값으로 되돌리기 위해 비밀유지가 필요한 키 사용

  - ex) RSA, AES, Blowfish

## AES(Advanced Encryption Standard)

- 미국 정부 표준으로 지정된 블록 암호 형식

- 128비트의 블록 크기를 가지고 있으며 128,160,192,224,256 비트 등 128비트 이상의 모든 32의 배수 비트 길이의 키를 사용할 수 있으며 미국 표준으로 인정받은 것은 128비트임

## DES(Data Encryption Standard)

- 블록 암호 일종

- 미국 국가표준(AES 이전) 대칭키 암호

- 56비트 키 사용

# Hashing

## 정의

- 임의의 길이의 데이터를 고정된 길이의 데이터로 매핑

  - 암호화 헤시 함수는 헤시값으로부터 원래의 입력값과의 관계를 찾기 어려움

  - ex) SHA-3, MD5

## Cryptographic Hash Function

- 암호학적 헤시 함수

- 임의의 길이를 갖는 메시지 입력 =\> 고정된 길이의 헤시값 출력

- 일방향 함수 포함.

- 통신의 암호화 보조수단, 사용자 인증, 디지털 서명에 응용

## Hash함수 종류

- MD(Message Digest)

  - 공개키 기반 구조

  - RSA와 함께 개발

  - MD2, MD4, MD5

- SHA\*Secure Hash Algorithm)

  - MD4가 발전한 형태

  - MD5보다 조금 느리지만 안전

  - SHA-1, SHA-2, SHA-256
