## 개요

- IPv4는 32bit로 구성

- 각 8bit.8bit.8bit.8bit

- 2의 8승 : 0 ~ 255

- IPv6는 128 bit 루성

- ip 주소는 네트워크 주소와 호스트 주소로 구성

  - 네트워크 주소: 호스트들을 모은 네트워크 지칭

  - 호스트 주소: 하나의 네트워크 내에 존재하는 호스트(단말)을 구분하기 위한 주소

- 네트워크 주소와 호스트 주소 간에 명확한 구분점은 없음 → 필요한 호스트 ip 개수에 따라 네트워크의 크기를 다르게 할당

## 클래스

- 클래스 구분 : A, B, C, D, E

  - D 클래스 : 멀티 캐스트를 위해 사용

  - E 클래스: 예약된 클래스

  - 일반적으로 A, B, C 클래스 사용

- Ip 주소의 클래스

  - 각 클래스에 할당되는 IP 주소의 개수

    - A 클래스: 약 16,000,000

    - B 클래스: 약 65,000

    - C 클래스 : 약 250

- 클래스 구조(네트워크, 호스트 구분)

  - A 클래스 : 네트워크.호스트.호스트.호스트

  - B 클래스: 네트워크.네트워크.호스트.호스트

  - C 클래스: 네트워크.네트워크.네트워크.호스트

- 클래스 구분

  - IP 주소로 어떤 클래스 속하는지 구분

  - A 클래스 : 1 ~ 127.0.0.0 (127로 시작하면 로컬 호스트)

  - B 클래스: 128 ~ 191.0.0.0

  - C 클래스: 192 ~ 223.0.0.0

  - D 클래스: 224 ~ 239.0.0.0

- 현재는 클래스 기반으로 네트워크 분할하는 기법 사용하지 않음.

- 네트워크 주소를 세밀하게 분할하고 할당하기 위해서 네트워크 크기에 맞쳐 1비트 단위로 네트워크를 상세히 분할( How?)

- 용어 정의 클래스 기반의 IP 주소 체계를 Classful, 현재의 구조는 클래스 개념 자체를 버린다고 해서 Classless 기반의 주소 체계라 함.

- Classless와 subnet mask

  - classful 기반의 IP 주소 체계에서 맨 앞 숫자를 보고 구분(192.0.0.0)

  - classless 에서는 구분할 수 있는 구분자 필요(subnet mask)

  - Subnet mask는 호스트 주소와 네트워크 주소를 구분할 때 사용

  - 2진수 숫자 1은 네트워크 주소, 0은 호스트 주소로 표시

  - 예) 클래스 표현

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    A 클래스    : 네트워크.호스트.호스트.호스트
    서브넷마스크 :   255  .  0  . 0   . 0
                11111111.00000000.00000000.00000000
    B 클래스    : 네트워크.네트워크.호스트.호스트
                   255  .    255.    0.    0
                11111111.11111111.00000000.00000000
    ```

    </div>

    </div>

- 예) IP 주소 102.44.23.111 subnet mask가 255.255.255.0 이면 네트워크 주소는 102.44.23.0 , 호스트 주소는 0.0.0.111

- 서브넷 마스크 : classless 기반 IP 주소에서 네트워크 주소와 호스트 주소를 구분하기 위한 구분자

- subnet mask는 1이 연속으로 나와야 함(규칙)

  - 00011111(올바른 형식), 00101111( 틀린 형식)

- subnet mask 표현

  - 192.168.0.1/24 : c 클래스, 서브넷 마스크: 255.255.255.0 (1이 25개 있다는 표현)

  - 기본 마스크란 의미 : C 클래스 기반 이기 때문에 기본적으로 서브넷 마스크가 255.255.255.0이 적용되어 있다는 말

## 논이 AND 연산

- IP 주소에 subnet mask 로 논리 AND 연산을 하여 네트워크 IP 주소 구함

- 예

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  1101 0010. 0110 0100. 0110 0100. 0000 0001  : IP 주소 210.100.100.1
  1111 1111. 1111 1111. 1111 1111. 0000 0000  : 서브넷 마스크 255.255.255.0

  1101 0010. 0110 0100. 0110 0100. 0000 0000  : 논리AND 결과 210.100.100.0 네트워크주소
  ```

  </div>

  </div>

## 네트워크 계산(호스트 개수)

- 201.222.5.0 IP 에 255.255.255.248 subnet mask 적용

- 예

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  11001001 11011110 00000101 00000000  : 201.222.5.0
  11111111 11111111 11111111 11111000  : 255.255.255.248 (subnet mask)

  000 이 호스트 영역. 총 8개의 호스트 
  1개의 네트워크 주소, 1개는 브로드 캐스느 주소 할당으로 총 6개의 호스트 주소 가짐
  ```

  </div>

  </div>

- 예(10.5.101.0/22) 의 의미는

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  00000110 00000110 00111000 00000000
  11111111 11111111 11111100 00000000

  호스트 개수 00 00000000 2의 10승
  ```
