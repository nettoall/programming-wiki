# XSS(Cross Site Scripting)

## XSS 공격 유형

- Reflected XSS: URL을 통해 Script 실행

- Stored XSS: Script를 서버에 저장하여 요청/응답 과정을 통해 실행

- DOM Based XSS: 피해자의 브라우저가 DOM 생성할 때 악성 Script 실행

## 위험성

- 쿠키 및 세션 정보 탈취

- 악성 프로그램 다운 유도

- 의도하지 않은 페이지 노출

## Spring Boot 에서 XSS 보안 취약 처리

- lucy-xss-filter: request 에서 HTML 태그를 치환 (네이버 제작)

  - https://github/com/naver/lucy-xss-filter

  - XSS 필터링이 필요한 곳에 개발자가 필터를 삽입하여 방어

  - 단점: 필요한 곳에 XSS 방어 코드 누락, 불필요한 곳에 XSS 방어 코드 적용, XSS 방어 코드 분산되어 유지보수 증가

- luxy-xss-servlet-filter

  - 어플리케이션으로 들어오는 모든 요청에 대해 XSS 방어 수행

  - https://github.com/naver/luxy-xss-servlet-filter

  - url, 파라미터 명으로 filtering 제외 설정 가능

  - 필터링 해야 하는 파라미터, URL에 대한 정밀한 설정이 필요

  - form-data, request param에 유효, RestAPI의 request body에는 적용 안 됨

- ObjectMapper customize

  - Json Body를 DTO로 변환할 때 필터링 처리(request)

  - DTO를 JSON 데이터로 변환할 때 필터링 처리(response)

### ObjectMapper 필터링 처리

#### HandlerInterceptor의 postHandle()

- Response에 공통으로 처리하도록 Interceptor의 postHandle() 구현

  - HttpServletResponse 에서 response by를 꺼내어 수정한 후 다시 삽입하는 과정이 어려움

#### MessageConver Customize

- MessageConver를 커스터 마이징한 후, WebMvcConfigurerAdapter에 conver 적용

- WebMvcConfigurerAdapter가 deprecated 됨

#### WebMvcConfigurer를 상속받는 config 구현

- configurerMEssageConverters() 메소드 override 하여 XSS 처리하는 converter 등록하여 처리

#### Filter 구현

- config 으로도 해결 안 될 경우 차선책

- request에서 HTML 태그를 수정하여 반여

### Filter 제작 시 유의해야할 점

- @EnableWebMvc를 사용하여 기존 Converter 설정이 모두 override 됨

- HttpMessageConverter는 bean 으로 등록시 자동으로 MessageConverter 목록에 추가됨

- 기존에 등록한 ObjectMapper bean이 있다면 JSON 요청/응답 에서 사용하기 위해 의존성 주입을 받아 사용

- 샘플

  @Slf4j
  @RequiredArgsConstructor
  @Configuration
  public class WebMvcConfig {
    private final ObjectMapper objectMapper;

    @Bean
    public MappingJackson2HttpMessageConverter jsonEscapeConverter() {
      ObjectMapper copy = ObjectMapper.copy();
      copy.getFactory().setCharacterEscapes(new HtmlCharacterEscape());
      return new MappingJackson2HttpMessageConverter(copy);
    }
  }
