- 클라이언트의 요청이 없어도 서버에서 데이터를 전달해야 하는 경우 (예: 뉴스피드, 댓글 알림 등)

- 동일한 화면을 보고 있는 클라이언트들에 다른 클라이언트가 요청한 변경 사항 적용

  - ㅇdeploy 목록에서 특정 항목에 대해 deploy시 그 변셩 사항을 통지하는 등

- 일반적으로 http에서는 client의 요청이 없으면 서바가 나서서 갱신하기 어려움

  - 해결방법

    - Short Polling : 클라이언트가 주기적으로 서버에 요청을 보내는 방법 : 구현 단순, 비실시간일 때 적합

    - Long Pooling : 클라이언트가 요청을 보내고 서버에서 변경이 얼아날 때까지 대기 : 서버의 상태가 빈번하게 변하지 않는 경우 적합, 서버 상태가 빈번하게 변경시 연결 요청이 늘어남

    - Server-Sent-Event : 서버와 한번 연결을 맺고나면 일정 시간동안 서버에서 변경이 발생할 때마다 데이터 전송받는 방법

      - 서버에서 클라이언트로 Text message를 보내는 브라우저 기반 웹 애플리케이션 기술, http persistent connections 을 기반으로 하는 HTML5 표준 기술

    - Web Socker을 이용한 양방향 통신도 고려

- 구현 프로세스

  1.  client가 서버의 이벤트를 구독하기 위한 요청

      - GET 방식, 미디어 타입 text/event-stream, no-cache

  2.  서버에서 응답

      - transfer-encoding: chunked

  3.  서버에서 이벤트 전달

- 코드 분석(최초 등록) : 이미 구현된 소스 분석

  - react로 구현 app.js 클라이언트에서 /api/emitter 를 호출 : 처음 접속할 때 subscribe함

  - CLIENTS(CopyOnWriteArraySet) 에 setEmitter 객체로 생성하여 add 함

  - 30X60X1000L 타임아웃 설정. 타임아웃 시간이 지나거나 완료했을 때 갱신을 하거나 CLIENTS에서 제거

  - 최초 접속시 Response Body에 공백값 넣어서 리턴 → 503 방지하기 위해 리턴(Emitter 생성 후 만료 시간까지 어떤 데이터도 보내지 않으면 재연결 요청시 503 Service Unavailable 에러 발생 → 처음 SSE 연결 후 더미 데이터 전달하도록 구현 필요)
