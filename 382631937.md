# FlashMap Manager

page 455

## FlashMap

- Flash attribute 가 저장되는 Map : 동일한 url을 refresh할 때 동일한 값이 계속 입력되는 것을 방지하기 위하여 attribute를 휘발성으로 한번만 사용하도록 함

- spring에서 파라미터를 간편하게 전달하기 위한 자료 구조

- flash attribute : <u>하나의 요청에서 생성되어 다음 요청으로 전달되는 정보, 다음 요청에서 한번 사용되고 바로 제거되는 특징이 있음</u>

- post/redirect/get 적용시 post단계의 작업결과 메시지를 redirect된 page로 전달할 때 주로 사용 : POST/REDIRECT/GET패턴

- 주로 같은 도메인에서 controller간 웹페이지에서 발생하는 Redirect시 데이터 처리를 간단하게 하기 위한 목적으로 사용됨

- url에 데이터를 노출시키지 않으면서 session 데이터에 넣기에는 휘발성(?)인 데이터

# WebApplicationInitializer를 이용한 ApplilcationContext등록

- 프레임워크에서 직접 서블릿 컨텍스트를 초기화하는 모듈 지원(servlet 3.0부터) 이전에는 web.xml 만 가능

- ServletContainerInitializer같은 API제공

- Spring WebApplicationInitializer를 구현한 클래스로 컨텍스트 초기화

## Root Web Context 등록

- Listener 생명주기는 RootContext와 같기 때문에 ContextLoaderListener로 RootContext를 초기화해줌

- WebApplicationInitializer는 같지 않기에 (시작은 ServletContext가 초기화될때 자동실행, 종료시점은 destroy같은 메소드가 없기 때문에 알지 못함) 종료 시점에 따라 메모리 누수 우려가 있음 =\> Listener 로 RootContext 관리하도록 권장(listener 에 init, destroy 메소드가 정의되어 있는 EventListener)

  1.  AnnotationConfigWebapplicationContext 사용시

      - ContextLoaderListener 는 기본적으로 /WEB-INF/applicationContext.xml 을 설정파일로 사용하는 XmlWebApplicationContext타입 사용함)

  2.  applicationContext.xml이외의 파일도 추가시

      - ContextLoaderListener가 처리하는 파일(applicationContext.xml)과 이름과 타입이 틀리므로 별도 정의 필요 : \<context-param\> 으로 처리

        <div class="code panel pdl" style="border-width: 1px;">

        <div class="codeContent panelContent pdl">

        ``` syntaxhighlighter-pre
        # web.xml
        <context-param>
          <param-name>contextClass</param-name>
          <param-value>org.springframework...AnnotationConfigWebApplicationContext</param-value>
        </context-param>
        <context-param>
          <param-name>contextConfigLocation</param-name>
          <param-value>myproject.config</param-value>
        </context-param>
        ```

        </div>

        </div>

      - Java Code로 listener등록 예제(page 460)

        <div class="code panel pdl" style="border-width: 1px;">

        <div class="codeContent panelContent pdl">

        ``` syntaxhighlighter-pre
        ServletContextListener listener = new ContextLoaderListener();
        servletContext.addListener(listener);
        // context param 설정
        servletContext.setInitParameter("contextClass","org.springframework...AnnotationConfigWebApplicationContext");
        servletContext.setInitParameter("contextConfigLocation","myproject.config");

        //Root Context 등록 예제
        AnnotationConfigWebApplicationContext ac = new AnnotationConfigWebApplicationContext();
        servletContext.scan("com.mycompany.myproject.config"); // @Configuraton 클래스 등록
        servletContextListener listener = new ContextLoaderListener(ac);
        servletContext.addListener(listener);
        ```

        </div>

        </div>

## ServletContext 등록

1.  Listener안에서 root Context초기화

    - servletcontext 는 servlelt안에서 초기화 (root context안에 서블릿 개수만큼 servlet context가 존재함)

- spring에선 DispatcherServlet사용(web.xml 에 서블릿 초기화 관련 설정 정의함)

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  <servlet>
  ...
  </servlet>
  <servlet-mapping>
  ...
  </servlet-mapping>
  ```

  </div>

  </div>

- 자바 코드로 서블릿 초기화 코드 정의시

  <div class="code panel pdl" style="border-width: 1px;">

- Default설정과 다른 설정을 추가 및 변경시 \<init-param\>사용하여 설정

- RootContext등록 예제에서 사용한 scan 대신 config를 등록하여 사용 가능

  ``` syntaxhighlighter-pre
  dispatcher.setLoadOnStartup(1);
  dispatcher.addMapping("/app");
  ```
