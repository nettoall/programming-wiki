데이터베이스 연동에 사용되는 기술

- JDBC

- Spring DAO

- Mybatis : SQL을 개발자가 직접 XML 파일에 등록

- Hibernate : 프레임워크에서 SQL 생성

- JPA(Java Persistence API)

## JPA 개념

- ORM (Object Relational Mapping)

- Hibernate: 기존의 EJB 기술인 엔티티 빈이 가지는 여러 문제들을 대체하기 위한 오픈 소스 프레임워크로 등장

- Hibernate 가장 많은 개발자가 사용하는 ORM, 하이버네이트 개발자들이 중심이 되어 만든 ORM 표준 JPA

- JDBC API와 유사하게 JPA가 제공하는 인터페이스를 이용하여 데이터 베이스 처리

- JPA를 구현한 구현체는 Hibernate, EclipseLink, DataNucleus 등

- Spring Boot 에서는 Hibernate사용

### JPA 동작 원리

- 자바 객체를 Collection에 저장하고 관리하는 것과 비슷한 개념

- Collection에 저장된 객체를 테이블의 row와 매핑하기 위해 JDBC API를 이용해 연동 작업 진행 : JPA가 처리

- 데이터 베이스 연동 코드 및 SQL 제공

## 실습

### DB 구성

- h2 db

  - jdbc 지원

  - in-memory, server 모드 모두 지원

  - 브라우저 기반의 관리 콘솔 제공

- h2 db 설치

  1.  <a href="https://www.h2database.com" class="external-link" rel="nofollow">https://www.h2database.com</a> 에서 All Platforms 다운로드

  2.  zip 파일 압축 해제 후 bin/h2w.bat 실행 (DB 구동)

  3.  driver class : org.h2.Driver, jdbc url : jdbc:h2:tcp://localhost/~/test, 사용자명 : sa, 비밀번호 : 없음

### 프로젝트 생성

- JPA 프로젝트 생성

- maven 생성시 : org.apache.maven.archetypes:maven-archetype-quickstart 로 생성

- Elicpse 인 경우 Project Facets 에서 JPA 선택 (Intellij 는?)

- META-INF 폴더에 JPA 환경설정 파일 persistence.xml 생성

- dependency 설정

  - org.hibernate:hibernate-entitymanager:5.1.0.Final

  - com.h2database:h2:1.4.197

  - junit:junit:3.8.1

  - 또는 boot-starter-data-jpa 추가하면 자동으로 hibernate-core 가져옴

### 엔티티 생성

- JPA는 테이블이 없으면 자바 클래스를 기준으로 매핑할 테이블을 자동 생성

- Value Object 클래스를 테이블과 동일한 클래스 이름 사용, 컬럼과 매핑될 맴버 변수 선언

- persistence.xml에 엔티티 등록(idea/modules/META-INF/persistence.xml)

- persistence.xml : JPA의 메인 환경설정 파일

  - Persistence Unit 설정 : JPA에 연동할 데이터베이스에 대한 정보 포함

  - 샘플

    <persistence version="2.1" xmlns="http://xmlns.jcp.org/xml/ns/persistence"
