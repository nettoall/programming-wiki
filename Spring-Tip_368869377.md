# Spring을 web.xml 없이 실행하기

- servlet 3.0

1.  web.xml에서 spring contextLoaderListener실행해서 spring시작

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    <context-param>
      <param-name>contextConfigLocation</param-name>
      <param-value>classpath*:spring/context-*.xml</param-value>
    </context-param>
    <listener>
      <listener-class>org.springframework.web.context.ContextLoaderListener</listener-class>
    </listener>
    ```

    </div>

    </div>

2.  서블릿 3.0에서부터 web.xml없이 서블릿 context초기화 가능

    - javax.servlet.ServletContainerInitializer 인터페이스를 구현한 클래스 작성

    - /META-INF/services/javax.servlet.ServletContainerInitializer 텍스트 파일에 해당 인터페이스를 구현한 클래스 정보를 적음

    - 구현된 클래스는 다음과 같이 @HandlesType() annotation 으로 클래스를 지정하면 WAS기동시 onStartup() 인자로 지정된 클래스와 서블릿 컨텍스트 객체를 전달

      <div class="code panel pdl" style="border-width: 1px;">

      <div class="codeContent panelContent pdl">

      ``` syntaxhighlighter-pre
      @HandlesType(WebApplicationInitializer.class)
      public SpringServletContainerInitializer implements ServletContainerInitializer {
        /**
         * WebApplicationInitializer 인터페이스를 구현한 클래스들의 onStartup()메소드 호출
         **/
        @Override
        public void onStartup(Set<class<?>> webAppInitializerClasses, ServletContext servletContext) throws ServletException {
          ...
        }
      }
      ```

      </div>

      </div>

    - Spring-web-4.3.24.RELEASE.jar 내 META-INF/services/javax.servlet.ServletContainerInitializer 에 다음과 같이 정의됨

      - org.springframework.web.SpringServletContainerInitializer

      - Spring의 webApplicationInitializer interface를 구현하면 web.xml 없이 spring web 실행 가능함

      - 예제

        <div class="code panel pdl" style="border-width: 1px;">

        <div class="codeContent panelContent pdl">

        ``` syntaxhighlighter-pre
        public class SpringStart implements WebApplicationInitializer {
          public void onStartUp(ServletContext servletContext) throws ServletException {
            AnnotationConfigWebApplicationContext rootContext = new AnnotationConfigWebApplicationContext();
            rootContext.register(com.test.init.RootContextConfiguration.class);
            servletContext.addListener(new ContextLoaderListener(rootContext));
          }
        }


        // 설정에 관련된 정보
        @Configuration
        public class RootContextConfiguration {

        }
        ```

        </div>

        </div>

3.  ContextLoaderListener와 설정 클래스를 이용한 시동

    - web.xml 의 contextConfigLocation 과 contextClass 사용

      <div class="code panel pdl" style="border-width: 1px;">

      <div class="codeContent panelContent pdl">

      ``` syntaxhighlighter-pre
      <context-param>
        <param-name>contextConfigLocation</param-name>
        <param-value>com.text.init.RootContextConfiguration</param-value>
      </context-param>
      <context-param>
        <param-name>contextClass</param-name>
        <param-value>org.springframework.web.context.support.AnnotationConfigWebApplicationContext</param-value>
      </context-param>

      <listener>
        <listener-class>org.springframework.web.context.ContextLoaderListener</listener-class>
      </listener>
      ```

      </div>

      </div>

4.  WebApplicationInitializer를 사용하여 시작하고 설정은 xml 사용

    - 2번 예제에서 RootContextConfiguraton 에 annotation으로 xml 설정 파일 지정

      <div class="code panel pdl" style="border-width: 1px;">

      <div class="codeContent panelContent pdl">

      ``` syntaxhighlighter-pre
      @Configuration
      @Import({DatabaseConfig.class, ComponentConfig.class})
      @ImportResource("classpath*:egovframework/spring/com/context-*.xml")
      public class RootContextConfiguration {}
      ```

      </div>

      </div>

- @ImportResource 를 사용하여 xml 설정 파일 지정

  - @Import 설정 클래스 지정

# Spring Batch 사용 정리

## Batch

- FlatFile : FlatFileItemReader는 일반적으로 고정 위치에 정의된 데이터 필드나 특수 문자에 의해 구별된 데이터의 행을 읽는다.

- XML : XMLItemReader 는 파싱,매핑, 유효성 검증을 XML에서 독립적으로 작업할 수 있도록 처리

  - 입력 데이터는 XSD 스키마에 대해 XML 파일의 유효성 검증이 가능

- DB

### FlatFileItemReader

#### 속성

- Resource

- LineMapper : 데이터에서 읽은 1라인을 Object로 변환하는 총 과정

  - FieldSetMapper : FieldSet 형태의 데이터를 원하는 Object로 변환

  - LineTokenizer : 1라인을 구분자 방식 또는 고정 길이 바익으로 Tokenizer한 후 FieldSet방식으로 변환

    - DelemitedLineTokenizer, FixedLengthTokenizer, EgovEscapableDelimitedLineTokenizer(Escape문자를 사용하여 문자열에 추가하기 할 수 있는 방식)

### eGovFlatFileItemReader

- Spring Batch의 대용량 데이터 처리 시간을 줄이기 위해 개발

- 성능 저하 요인인 LineMapper를 개선하여 제공

- 전자정부에서는 FieldSet을 사용하지 않음

- 제한사항 : EgovDefaultLineMapper 사용시 propeprty로 사용하는 class는 모두 전자정부에서 제공해야 하는 클래스여야 함(EgovFixedLengthTokenizer, EgovByteLengthTokenizer, EgovDelimitedTokenizer)와 EgovObjectMapper사용

- EgovObjectMapper 사용시 VO의 필드는 기본 타입과 BigDecimal로 제한됨(클래스 등 사용 안 됨)

- DefaultLineMapper에서는 Tokenizer에서 ‘names’ property를 사용하지만 EgovDefaultLineMapper사용시 EgovObjectMapper에서는 names 속성 사용

#### Spring DefaultLineMapper

- xml설정

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  <bean id="itemReader" class="org.springframework.batch.item.file.FlatFileItemReader" scope="step">
    <property name="resource" value="inputfile"/>
    <property name="lineMapper">
      <bean class="org.springframework.batch.item.file.mapping.DefaultLineMapper">
        <property name="lineTokenizer">
          <bean class="org.springframework.batch.item.file.transform.DelimitedLineTokenizer">
            <property name="delimiter" value=","/>
            <property name="names" value="name,credit"/>
          </bean>
        </property>
        <property name="fieldSetMapper">
          <bean class="org.springframework.batch.item.file.mapping.BeanWrapperFieldSetMapper">
            <property name="targetType" value="egovframework.brte.sample.domain.trade.CutomerCredit"/>
          </bean>
        </property>
      </bean>
    </property>
  </bean>
  ```
