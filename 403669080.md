- @RestController Controller method가 리턴한 데이터 자체를 클라이언트에 전달

  - 클라이언트에 전달되는 데이터는 대부분 문자열이거나 VO나 Collection 형태의 자바 객체

  - VO객체를 JSON 데이터로 변환하여 응답 바디에 출력

  - 자바 객체가 전달되는 경우에는 자동으로 JSON 으로 변환하여 처리

- lombok 제공 annotation

  - @Getter

  - @Setter

  - @RequiredArgsConstructor

  - @ToString

  - @EqualsAndHashCode : equals(), hashCode() 메소드 생성

  - @Data : @Getter, @Setter, @RequiredArgsConstructor, @ToString, @EqualsAndHashCode 모두 포함하여 생성

2.  1에서 구현한 starter를 dependency 로 추가하는 프로젝트 구현

    - JDBC 등록하는 connectionManager의 설정값 변경 필요 - 자동 설정하는 DB 가 아닌 다른 DB 사용시

      1.  변경된 설정값을 적용하는 @Configuration 구현

      2.  application.properties에 항목 추가 - <u>starter에 있는 JDBCConnectonManager와 충돌남</u>

          - **spring.main.allow-bean-definition-overriding=true**

          - 새로운 bean이 기존의 bean을 덮어쓰도록 함

      3.  starter의 @Configuration 구현 클래스에 @ConditionalOnMissingBean 추가 : 중복 Bean 등록 안 되게 처리

          - **@SpringBootApplication 실행 순서** : @ComponentScan 으로 패키지내 Bean 생성, 등록, @EnableAutoConfiguration 자동 설정에 의한 Bean 등록 =\> H2 DB가 아닌 Oracle DB가 등록됨

          - **@ConditionalOnMissingBean** 은 <u>등록하려는 Bean이 메모리에 없는 경우에만 Bean 등록</u> =\> @ComponentScan 으로 H2 Db Connection Manager를 등록한 후에 @ConditinoalOnMissingBean annotation에 따라 @EnableAutoConfiguration 이 설정된 Bean Scan 안 함

# 독립 실행 가능 JAR

- Spring Boot는 독립적으로 실행 가능한 어플리케이션을 빠르게 개발하는 것을 목적

- WAR 대신 JAR 를 선호

- 다른 jar와 비교시 Spring Boot는 BOOT-INF 디렉토리를 가짐

- BOOT-INF 는 실제 application class와 library(톰캣 라이브러리도 포함)로 구성

- META-INF에 실행될 어플리케이션 정보 저장됨

  - Main-class, start-class, Spring-boot-class, spring-boot-lib 등

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    Main-Class: org.springframework.boot.loader.JarLauncher
    Start-Class: com.nettoall.ChapterApplication
    Spring-Boot-Classes: BOOT-INF/classes/
    Spring-Boot-Lib: BOOT-INF/lib/
    Created-By: Apache Maven 3.5.3
    Build-Jdk: 1.8.0)291-1-ojdkbuild
    Implementation-URL: https://projects.spring.io/Spring-boot/$/spring-boot-starter-parent/Chapter
    ```

    </div>

- 패키징된 jar 파일 실행 : 일반 jar는 불가능함

  - 일반적으로 jar 내에 여러 jar 포함시 허용 안 되며, 원칙적으로는 여러 jar 안에 클래스를 풀어서 하나의 jar로 만들어야 함

  - 스프링 부트는 패키징된 JAR 파일 안에 있는 또 다른 JAR 파일을 읽어서 클래스들을 로딩하는 유틸리티 클래스 제공 (org.springframework.boot/loader/jar)

  - JarFile 로더 클래스가 BOOT-INF/lib 폴더에 있는 수 많은 JAR 파일을 사용할 수 있도록 로딩
