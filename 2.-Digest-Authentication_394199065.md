# Digest Authentication

- `다이제스트 인증은 사용자 이름, 암호, 서버에서 제공하는 nonce 값, HTTP 메서드 및 요청된 URI에 해시 함수를 적용하여 암호화된 형식으로 자격 증명을 전달합니다.`

- `다이제스트 액세스 인증은 웹 서버가 사용자의 웹 브라우저와 자격 증명(예: 사용자 이름 및 암호)을 협상하기 위해 허용되는 기술 중 하나입니다. `

- `이것은 온라인 뱅킹 거래 내역과 같은 민감한 데이터를 제공하기 전에 사용자의 신원을 확인하는 데 사용할 수 있습니다. 네트워크를 통해 로그인 및 비밀번호를 전송하기 전에 해시 기능을 사용합니다. `

- `반면 기본 인증은 해싱 대신 쉽게 되돌릴 수 있는 Base64 인코딩을 사용하므로 TLS와 결합하지 않는 한 안전하지 않습니다.`

## 구현

1.  `DigestAuthenticationFilter는 HTTP 요청의 Digest Authorization 헤더를 처리하여 결과를 SecurityContextHolder에 배치합니다.`

2.  `인증에 성공하면 결과 인증 개체가 SecurityContextHolder에 저장됩니다.`

3.  `인증에 실패하면 AuthenticationEntryPoint 구현이 호출됩니다. 이것은 항상 DigestAuthenticationEntryPoint이며 사용자에게 Digest 인증으로 다시 인증하도록 요청합니다.`

4.  `DigestAuthenticationFilter는 진입점을 포함하고 UserDetailsService는 사용자 이름, 비밀번호, 활성화 정보 등과 같은 사용자 정보로 구성됩니다.`

5.  `다이제스트 인증의 경우 DigestAuthenticationEntryPoint에 인증 진입점이 필요합니다. 진입점에는 키와 영역이 필요합니다.`

## 예제

- gradle 은 basic authentication 설정과 동일

- application.yml 사용자 정보 설정은 없음

- AdminSecurityConfiguration 설정 구현 추가

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @Configuration
  @Order(1)
  public class AdminSecurityConfiguration extends WebSecurityConfigurerAdapter {

      // Configure DigestAuthenticationEntryPoint, DigestAuthenticationEntryPoint의 의미?
      private DigestAuthenticationEntryPoint getDigestEntryPoint() {
          DigestAuthenticationEntryPoint digestEntryPoint = new DigestAuthenticationEntryPoint();
          digestEntryPoint.setRealmName("admin-digest-realm");
          digestEntryPoint.setKey("somedigestkey");
          return digestEntryPoint;
      }

      // PasswordEncoder Bean
      @Bean
      public PasswordEncoder passwordEncoder() {
          // NoOpPasswordEncoder deprecated
          return NoOpPasswordEncoder.getInstance();
      }

  //    Configure authenrication manager
      protected void configure(AuthenticationManagerBuilder auth) throws Exception {
          auth.inMemoryAuthentication()
                  .withUser("testUser")
                  .password(passwordEncoder().encode("digestsecret"))
                  .roles("USER")
                  .and()
                  .withUser("TestAdmin")
                  .password(passwordEncoder().encode("adminsecret"))
                  .roles("ADMIN");
      }

      // default UserDetailsService configured by UserDetailsServiceAutoConfiguration

      @Bean
      public UserDetailsService userDetailsServiceBean() throws Exception {
          return super.userDetailsServiceBean();
      }

  //    DigestAuthenticationFilter 설정
  //    Notice the we are injecting userDetailsService and DigestAuthennticationEntryPoint
      private DigestAuthenticationFilter getDigestAuthFilter() throws Exception {
          DigestAuthenticationFilter digestFilter = new DigestAuthenticationFilter();
          // 사용자 정보 가져오는 서비스 설정
          digestFilter.setUserDetailsService(userDetailsServiceBean());

          digestFilter.setAuthenticationEntryPoint(getDigestEntryPoint());
          return digestFilter;
      }

  //    Notice that we are configuring /admin/** for the SecurityFilterChain
  //    added DigestAuthenticationFilter, also configured the EntryPoint authentication
      @Override
      protected void configure(HttpSecurity http) throws Exception {
          http.antMatcher("/admin/**")
                  .addFilter(getDigestAuthFilter()).exceptionHandling()
                  .authenticationEntryPoint(getDigestEntryPoint())
                  .and()
                  .authorizeRequests().antMatchers("/admin/**").hasRole("ADMIN");
      }
  }
  ```
