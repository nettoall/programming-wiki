# 동기화(Synchronization)

- 다중 CPU, 다중 Process, 다중 Thread 환경에서 공통된 자원에 접근할 때 발생하는 순서와 동시성을 배제하기 위한 목적으로 설계

- Process가 수행 불가능한 교착 상태 유발

## 동기화 용어

- 경쟁조건(Race Condition)

  - 2개 이상의 Process(Thread)들이 공유 자원을 읽고 쓰고 그 결과가 실행 순서에 따라 달라질 수 있는 현상

- 임계영역(Critical Section)

  - 여러 Process(Thread)가 동시에 접근하는 공유 자원(**단 하나의 스레드만 실행할 수 있는 코드 영역**)

- 상호 배제(Mutual Exclusion)

  - 경쟁조건을 피하기 위해 한 Process(Thread)가 공유 자원을 사용하고 있을 때 다른 Process(Thread)가 사용하지 못하게 배제

- Mutax

  - **임계 영역**을 가진 Thread 들의 실행 시간이 겹치지 않게 각각 단독으로 실행되게 하는 기술

- 상호배제 구간

  - Thread가 공유 자원 사용하는 경우, 다른 Thread가 이를 사용하는 것을 막는 구간

  - 구간이 정의되지 않으면 높은 순위의 Thread가 참조, 잘못된 결과 생성

- 행동

  - 상호배제구간(임계 영역)에 들어가기 전에 mutex 변수 lock, 구간이 끝나면 unlock

## 동기화 방법

- Mutex

  - Mutex 공용 변수를 사용, lock에 소유자를 따져야 하는 방식

- Semaphore

  - mutex와 비슷한 “binary semaphore방식”과 “수를 게수하는 counting semaphore방식”이 있다.

  - 둘 다 value 공용변수 사용, value에 +나 - 방식으로 count하여 자원 접근 여부 결정

  - Mutex처럼 lock 소유자 개념이 없고 단순한 연산(+,-)로 자원 사용 여부 판단

## Thread와 공유객체

- 하나의 Thread가 아닌 multi Thread 사용시 하나의 객체를 공유해야 함

- 예로 MusicBox 라는 공유 객체를 multi thread가 사용한다면 각각 값이 변경되어 예상치 않은 결과가 발생

- 공유 객체를 각 Thread가 동시에 사용하지 않도록 구현 필요

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public class Musicbox{
    public void playMusicA(){
    
    }
    
    public void playMusicB(){
    
    }
    
    public void playMusicC(){
    
    }
  }

  public class MusicPlayer extends Thread{
    int type;
    MusicBox musicBox;
    
    public MusicPlayer(int type, MusicBox musicBox){
      this.type = type;
      this.musicBox = musicBox;
    }
    
    @Override
    public void run(){
      switch(type){
        case 1: musicBox.playMusicA(); break;
        case 2: musicBox.playMusicA(); break;
        case 3: musicBox.playMusicA(); break;
      }
    }
    
    public static void main(String[] args){
      MusicBox box = new MusicBox();
      MusicPlayer kim = new MusicPlayer(1,box);
      MusicPlayer lee = new MusicPlayer(2,box)
    }
  }
  ```

## Java에서의 동기화 처리

- 동기화 메소드 및 동기화 블록 사용

- 스레드 작업이 끝날 때까지 객체에 잠금을 걸어서 다른 스레드가 사용할 수 없도록 함

- 임계 영역을 지정하기 위해 동기화 메소드와 동기화 블록 제공

### 동기화 메소드

- 메소드에 synchronized prefix 붙임

- 먼저 호출한 메소드가 객체의 사용권(Monitoring lock)을 얻는다.

### 동기화 블록

- 메소드에 synchronized를 붙여서 사용시 마지막 thread 의 대기 시간이 길어질 경우, 메소드가 아닌 동기화해야 할 부분만 동기화 적용

- 일부 내용만 임계 영역으로 만들시 사용

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  methodA(){
    synchnized(this){
      System.out.println("aaa");
    }
  }

  methodB(){
    synchronized(this){
      System.out.println("bbb");
    }
  }
  ```

### 예제 코드

- 시작점과 끝점을 수동으로 명시적으로 동기화

- import java.util.concurrent.locks.ReentrantLock

- private final 로 선언

- 동기화를 시킬 지점의 앞에 lock 메소드 사용

- 동기화의 내용을 try~catch 문을 사용하여 작성하고 finally에 unlock 메소드 사용

- wait(), notify(), notifyAll() 메소드 사용 못함. await(), signal(), signalAll() 사용

- ReentrantLock이 세부적으로 동기화 가능
