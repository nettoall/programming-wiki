## 패키지

### 정의

파이썬 모듈을 계층적으로 관리할 수 있게 해줌

- 각 디렉토리별 ‘**\_\_init\_\_.py’** 파일 필수(해당 디렉토리가 패키지의 일부라는 정의)

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  game/
      __init__.py
      sound/
           __init__.py
           echo.py
  ```

  </div>

  </div>

### 패키지 만들기

- PYTHONPATH에 game 상의 디렉토리까지 설정함 (예 : C:\doit 설정)

- PYTHONPATH에 구현한 python 실행 파일 참조하도록 설정

  1.  모듈 import

      <div class="code panel pdl" style="border-width: 1px;">

      <div class="codeContent panelContent pdl">

      ``` syntaxhighlighter-pre
      import game.sound.echo
      game.sound.echo.test() // echo test() 함수 호출
      ```

      </div>

      </div>

  2.  모듈 디렉토리까지 import

      <div class="code panel pdl" style="border-width: 1px;">

      <div class="codeContent panelContent pdl">

      ``` syntaxhighlighter-pre
      from game.sound import echo
      echo.test()
      ```

      </div>

      </div>

  3.  test()함수 import

      <div class="code panel pdl" style="border-width: 1px;">

      <div class="codeContent panelContent pdl">

      ``` syntaxhighlighter-pre
      from game.sound.echo import test
      test()
      ```

      </div>

      </div>

- import한 디렉토리에 있는 모듈만 호출 가능

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  import game
  game.sound.echo.test() # 오류 발생 -> game 디렉토리에 있는 모듈만 사용 가능함

  또는
  import game.sound.echo.test # 오류발생 -> import 할 때 마지막 항목은 무조건 모듈 및 패키지여야함
  ```

  </div>

  </div>

### **‘\_\_init\_\_.py’의 용도**

- 해당 디렉토리가 패키지의 일부임을 알려주는 역할

- python 3.3버전부터는 ‘\_\_init\_\_.py’ 파일이 없어도 패키지로 인식(하위 버전 호환을 위해 정의 권장)

- python에서 import \* 의 의미

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  >>> from game.sound import *
  >>> echo.echo_test() # 오류 발생
  ```

  </div>

  </div>

  - 하위에서 echo 모듈을 못 찾음 python에서 \* 은 모든 것을 의미하지 않은가 봄

  - 해결방안 : ‘\_\_init\_\_.py’ 파일안에 ‘\_\_all\_\_’ 변수 설정하고 import할 수 있는 모듈 정의

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    __all__ = ['echo']
    ```

    </div>

    </div>

  - import \* 에서 \*은 모듈은 해당 안 됨.

### relative 패키지

- 예제 패키지 경로

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  sound/
        __init__.py
        echo.py
  graphic/
         __init__.py
         render.py
  ```

  </div>

  </div>

- render.py 에서 echo.py를 사용하는 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  # render.py
  from game.sound.echo import echo_test
  def render_test():
    print("render")
    echo_test()
  ```

  </div>

  </div>

- 상대 경로로 import 하는 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  # render.py
  from ..sound.echo import echo_test  # 상대 경로로 import
  def render_test()
    print("render")
    echo_test()
  ```

  </div>

  </div>

- 상대 경로 접근법은 py 파일 안에서만 사용 가능. 인터프리터에서 사용시 오류 발생
