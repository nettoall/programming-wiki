# <style>[data-colorid=pelyvcvxrq]{color:#ff5630} html[data-color-mode=dark] [data-colorid=pelyvcvxrq]{color:#cf2600}</style>DAO 패턴

- 데이터 엑세스 계층을 DAO 패턴으로 분리

## 주요 요점

- 데이터 엑세스 기술을 외부에 노추랗지 않도록 DAO 구현기술 외부 공개 안 함

- DAO를 이용하는 서비스 계층의 코드를 순수한 POJO로 개발 가능

### DAO 인터페이스와 DI

- DAO는 interface를 이용해 접근하고 DI되도록 구현

### 예외처리

- 데이터 엑세스 중에 발생하는 예외는 runtime Exception 으로 던짐, 즉 runtime exception으로 전환하여 처리

- 의미 있는 예외(중복 키, locking 등)을 처리하기 위해 데이터 엑세스 기술에 의존하지 않는 일관된 의미를 갖는 데이터 예외 추상화 제공

- 각 기술과 DB에서 발생하는 예외를 Spring의 데이터 예외로 변환해 주는 변환 서비스 제공

- 데이터 엑세스 기술의 API를 직접 사용할 때는 Spring에 내장된 AOP를 이용해 예외로 전환해 주는 기능 사용

# 템플릿과 API

- DI의 응용 패턴인 템플릿/콜백 패턴을 이용해 try/catch/finally와 리소스를 적절히 반환해 주는 코드를 정의하지 않고 필요한 바뀌는 내용만을 담을 수 있도록 데이터 엑세스 기술을 위함 템플릿 제공

- 예외 변환과 트랜잭션 동기화 기능도 제공

- 단점 : 데이터 엑세스 기술의 API보다 템플릿이 제공하는 API 사용. 일부는 데이터 엑세스 API를 사용하도록 Spring 에서 지원

# DataSource

- Pooling 기법 사용

- Spring에서는 DataSource를 독립된 Bean 으로 등록하도록 권장

- 공유가능한 DataSource 로 등록

  - 데이터 엑세스 기술이 DataSource를 사용

- <span colorid="pelyvcvxrq">학습 테스트, 통합 테스트를 위한 DataSource( 별 의미 없음. 사용도 안 함)</span>

  - SimpleDriverDataSource : 매번 Connection 생성하고 pool 관리 안 함

  - SingleConnectionDataSource : 하나의 DB Connection 만들고 이를 계속 사용, 두 개 이상의 스레드 동작시 문제됨

## 오픈 소스, 상용 DB Connection Pool

- Apache Commons DBCP

- C3pq JDBC/DataSource Resource Pool

  - JDBC 3.0 스펙 준수
