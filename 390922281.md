## 설치방법

### NVM(Node Version Manger) 로 여러 버전의 nodejs 설치 및 사용

- nodejs 를 설치하거나 버전을 관리해 주는 프로그램

- Windows : <a href="https://github.com/coreybutler/nvm-windows/releases" class="external-link" data-card-appearance="inline" rel="nofollow">https://github.com/coreybutler/nvm-windows/releases</a> 에서 nvm-setup.zip 다운로드후, 설치

### WSL에 Node.js 설치

- sudo apt-get install curl (curl 명령이 없을 경우)

- nvm 설치 : `curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash`

- nvm 설치 확인 : command -v nvm =\> nvm 리턴

- wsl 창을 닫고 새로 연다

- nvm ls : 현재 설치된 node 버전 나열

  - 나열되는 node 버전이 없음

- Node.js 의 현재 버전와 안정적인 LTS 버전 모두 설치

  - 안정적인 LTS 릴리스 설치 : nvm install --lts

  - 현재 릴리스 설치 : nvm install node

- nvm ls 로 체크

- node.js 버전 체크 : node --version

  - which node : node 경로 확인

  - which npm : npm 경로 확인

- node.js 버전 변경 : nvm use \<버전번호\>

- 특정 버전 설치 : nvm ls 결과로 나온 정보를 가지고 버전 선택

  - ex) lts 16 버전 설치 : nvm install lts/gallium

#### NVM 명령어

- nvm 버전 확인

  - nvm -v

- node.js 설치

  - nvm install 10.10.0 (mvn install 버전)

- 설치되어 있는 node.js 목록 조회

  - nvm list

- 사용할 node.js 버전 지정

  - nvm use 10.10.0

- 설치할 특정 버전을 찾지 못할 때에는 해당 버전의 마지막 버전을 설치

  - nvm install 18

- node.js 버전 조회

  - node -v

### NPM 사용법

#### NPM에 남아 있는 캐시 삭제(최신 버전 업데이트시 오류 발생 방지를 위해)

npm cache clean --force

npm cache verify

#### NPM 최신 버전 설치

npm install -g n

- n 이라는 패키지를 통해 node 버전이 업그레이드 됨

- n 옵션은 윈도우에서 지원 안 되기 때문에 nvm를 설치하는 경우가 많음
