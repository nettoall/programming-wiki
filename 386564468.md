# Spring Boot 3.0 하위 프로젝트를 업그레이드 하는 방법

## 단계1. JDK17 설치

- JDK17보다 하위 버전의 java를 사용하고 있으면 Spring Boot 3.0에서 지원되지 않음

- 최소 JDK 버전이 17이기 때문에 JDK 17 설치

## 단계2. 프로젝트를 최근 버전 Spring Boot 2.7.x 버전으로 업그레이드

- Spring Boot 3.0 으로 전환하기 전에 최근 버전인 Spring Boot 2.7.x 버전으로 단계별 업그레이드 진행

- 바로 업그레이드 하게 되면 새로운 기능과 많은 구성을 변경해야 해서 마이그레이션이 너무 복잡해질 수 있음

## 단계3. 라이브러리 종속성 검사

- Spring Boot 공식 마이그레이션 지침에 제시된 2.7.x 및 3.0.x 버전 모두에 대한 종속성 목록 제공

- 종속성 목록에 기반해 라이브러리 체크 및 변경

### Spring Security 종속성 검사

- Spring Boot 3.0은 Spring Security 6.0을 사용

- Spring Security 업그레이드를 간소화 하기 위해 Spring Security에서 5.8 버전 제공

- Spring 2.7.x 버전에서 Spring Security 5.8 로 업그레이드

- Spring Security 에서 제공하는 마이그레이션 가이드를 기반으로 종속성 체크 및 변경

- Spring Boot 3.0 을 사용한 Spring Security UserDetailsService 예제도 있음

## 단계4. Spring Boot 2.x 의 Deprecations 검사

- Spring Boot 2.x에서 더 이상 사용되지 않는 일부 클래스, 메서드, annotations, 속성 제거됨

- 실제 변경은 단계 6에서 진행하고 deprecation만 검사함.

## 단계5. Spring Boot 3으로 업그레이드

- maven의 pom.xml 이나 gradle 의 설정 파일의 Spring Boot 버전을 3.0.0 으로 업그레이드 진행

## 단계6. 컴파일 오류 및 deprecated된 요소 수정

- 컴파일 오류과 deprecated된 요소 수정

## 단계7. 구성 속성 전환

- application.properties 나 application.yml 업데이트

- Spring Boot는 마이그레이션을 돕기 위해 ‘spring-boot-properties-migrator' 모듈 제공

- maven pom 정의

  ``` syntaxhighlighter-pre
    <scope>runtime</scope>
  </dependency>
  ```

- gradle 정의

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre

  ```

## 단계8. ‘javax’로 시작되는 패키지 ‘jakarta’ 로 업그레이드

- Java EE가 Jakarta EE로 변경됨에 따라 Spring Boot 3.0도 모든 종속성에 따라 업그레이드 됨

- 기존 Java EE에 속한 ‘javax’ 패키지 등을 ‘jakarta’ 로 변경

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  javax.persistence.*     -> jakarta.persistence.*
  javax.validation.*      -> jakarta.validation.*
  javax.servlet.*         -> jakarta.servlet.*
  javax.annotation.*      -> jakarta.annotation.*
  javax.transaction.*     -> jakarta.transaction.*
  ```

  </div>

  </div>

## 단계9. 핵심 변경사항이 있는 경우 업그레이드 확인

### 이미지 배너 지원이 제거됨

- 이미지 기반 애플리케이션 배너 지원이 제거됨

- banner.gif, banner.jpg, banner.png 파일은 이제 무시되며 텍스트 기반 banner.txt로 대체됨

### 로깅 날짜 형식이 변경됨

- Logback 및 Log4j2에 대한 로그 메시지의 날짜 및 시간 에 대한 형식이 변경됨

- 새로운 형식은 ISO-8601 표준과 일치함('yyyy-MM-dd’T’HH:mm:ss.SSSXXX'

- 공백문자가 아닌 날짜와 시간 사이의 구분 기호로 'T'문자 사용

- 끝에 시간대 오프셋 추가

- 이전 기본값인 ‘yyyy-MM-DD HH:mm:ss.SSS'를 복원하려면 LOG_DATEFORAT_PATTERN 환경변수 또는 ‘logging.pattern.dateformat’ 속성 사용

### @ConstructingBinding 이 더 이상 필요하지 않음

- @ConstrucingBinding 제거

- class 또는 record가 여러 생성자가 있는 경우, 속성 바인딩을 위해서 생성자 중 하나에 계속 사용할 수 있음

### YamlJsonParser가 제거됨

- SnakeYAML의 JSON 구문분석이 다른 구문분석기 구현과 일치하지 않기 대문에 YamlJsonParser가 제거됨

### 자동 구성 파일

- Spring Boot 2.7은 ‘spring.factories’ 의 등록과 하위 호환성을 유지하면서 자동 구성을 등록하기 위한 새로운 ‘META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports’ 파일을 도입

- Spring Boot 3.0 release 에서는 ‘spring.factories’의 자동 구성 등록 지원이 제거됨

## 단계10. 웹 애플리케이션에서 업그레이드 검토

### Spring MVC와 WebFlux URL 매칭 변경

- Spring MVC 의 Controller 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @Restcontroller
  public class EmployeeController {
    @GetMapping("/emp/register")
    public String showRegisterPage {
      return "register";
    }
  }
  ```

  </div>

  </div>

- Spring Framework 6.0부터는 후행 슬래시 일치 구성 옵션이 더 이상 사용되지 않음

  - 이전 버전에서는 “/emp/register” 및 “/emp/register/” 모두가 일치

  - 더 이상 “/emp/register/” 는 “/emp 와 더 이상 일치 하지 않음

- 더 이상 후행 슬래시를 추가하지 말기

- 애플리케이션이 이 변경 사항에 적응할 때까지 다음과 같은 전역 구성으로 사용

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @Configuration
  public class WebConfiguration implements WebMvcConfigurer {
    @Override
    public void configurePathMatch(PathMatchConfigurer configurer) {
      // 후행 슬래시 허용
      configurer.setUseTrailingSlashMatch(true);
    }
  }
  ```

  </div>

  </div>

### Jetty 사용시(현재도 동일한지 체크 필요)

- Jetty는 아직 Servlet 6.0 지원하지 않음

- Spring Boot 3.0에서 Jetty를 사용하려면 Servlet API 5.0 으로 다운 그레이드 해야 함

- jakarta-servlet.version 속성을 사용하여 지정
