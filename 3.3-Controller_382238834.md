# Controller 종류 및 Handler Adapter

- 종류(4가지)

## Servlet과 SimpleServletHandlerAdapter

- javax.servlet.Servlet클래스를 Controller로 사용(web.xml에 등록하지 않고 bean 등록)

  - Spring 이 아닌 프레임워크나 웹을 Spring기반으로 바꿔 갈 때 유용

  - 점차적으로 DI 구조로 변경해 감

  - 장점 : 서블릿 클래스 코드를 그대로 유지하면서 Spring Bean으로 등록

  - MVC 가 아니기 때문에 HttpServletResponse 에 넣어둔 정보를 확인하여 처리

  - Bean등록시 Servlet의 init,destroy 호출 필요

## HttpRequestHandler, HttpRequestHandlerAdapter

- Servlet과 유사

- HttpReqeustHandler interface를 상속받아 Controller 생성

- Http 프로토콜을 기반으로 한 전용 서비스 구축시

- Model, View 개념이 없는 HTTP기반의 RMI와 같은 서비스 지원

- Default 전략 : 별도로 Bean 등록 등 설정 필요 없음

## Controller, SimpleControllerHandlerAdapter

- Controller(org.springframework.web.servlet.mvc.Controller) interface를 상속받아 구현

- Controller ← AbstractController(필수 기능 추가) ← 실제 구현한 Controller

- AbstractController

  - synchronizeonSession : Http Session 동기화 여부(세션 동시접근 제어)

  - supportedMethods ; Http Method 지정

  - useExpiresHeader, useCacheControlHeader, useCacheControlNoStore, cacheSeconds : Cache 설정(브라우저)을 http 헤더를 사용하여 보내줄 것인지 결정

## AnnotationMethodHandlerAdapter

- 특정 interface를 구현한 Controller 지원 : SimpleControllerHandlerAdapter, HttpRequestHandlerAdapter

  - interface의 handleRequest() 메소드를 구현하여 처리

  - 하나의 Controller가 하나의 URL에만 대응

- AnnotationMethodHandlerAdapter는 annotation 정보와 메소드 이름, 파라미터 리턴 타입에 대한 규칙 등을 종합적으로 분석하여 컨트롤러 선별하고 호출방식 결정

- 컨트롤러 하나가 하나 이상의 URL에 매핑. 메소드단위로 URL 매핑 가능

- DefaultAnnotationHandlerMapping과 같이 사용

- DefaultHandlerAdapter로 별도 설정 필요 없음

 

# HandlerMapping

- HTTP 요청 정보로 Controller를 찾는 전략(기능)

- Spring 제공 HandlerMapping

  - default : 별도 설정없이 spring에서 지원

    - BeanNameUrlHandlerMapping

    - DefaultAnnotationHandlerMapping

- 그외의 HandlerMapping은 Bean으로 등록해야 사용가능

## BeanNameUrlHandlerMapping

- Bean 이름에 있는 URL과 HTTP 요청의 URL이 일치하는 bean 검색

- URL에 ANT 패턴 \*, \*\* 등 사용가능

- 간단한 application에 사용

 

## ControllerBeanNameHandlerMapping

- Bean의 ID나 Bean name을 이용해 mapping

## SimpleUrlHandlerMapping

- URL과 Controller 매핑 정보를 한 곳에 모아 놓을 수 있는 Handler Mapping

- 매핑 정보는 SimpleUrlHandlerMapping Bean의 Property에 설정함

- Default Handler Mapping이 아니므로 사용하려면 Bean 등록해야함

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  <bean class="org.springframework...SimpleUrlHandlerMapping">
    <property name="mappings">
      <props>
        <prop key="/hello">helloController</prop>
        <prop key="/sub/*">myController</prop>
        <prop key="/deep/**/sub">subController</prop>
      </props>
    </property>
  </bean>
  <bean id="helloController" ... />
  <bean id="myController" ... />
  ```

  </div>

  </div>

- property format에 따라 정의

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  <property name="mappings">
    <value>
      /hello=helloController
      /sub/*=myController
    </value>
  </property>
  ```

## 기타 공통 설정정보

#### 주요 property

- order

  - HandlerMapping 우선순위 정의 (두개 이상의 HandlerMapping 정의시)

- defaultHandler

  - url 매핑할 대상을 찾지 못할 떄 자동으로 default handler 선택

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    <bean class="...BeanNameUrlHandlerMapping">
      <property name="defaultHandler" ref="defaultController"/>
    </bean>
    ```

    </div>

    </div>

- alwaysUseFullPath

  - url 매핑은 web applicatoin의 root context path와 servlet path를 제외한 나머지를 매핑

  - context path는 /sub, servlet path는 /app/\* 라고 할 때 url 매핑이 /hello 인 경우, full path는 /sub/app/hello 임

  - 브라우저에 full path로 입력해야 url 매핑이 이루어짐. 일반적으로 url 매핑은 상대 경로로 사용

  - full path로 사용하려면 alwaysUseFullPath를 true로 설정

- detectHandlersInAncestorContexts

  - 일반적 Handler Mapping은 서블릿 Context 안에서만 매핑한 Controller 를 찾음.

  - detectHandlersInAncestorContexts=false 설정되어 있음 true로 변경시 부모 context까지 url 매핑 controller 검색함

# HandlerIntercepter

- DispatcherServlet이 Controller를 호출하기 전과 후에 요청과 응답을 참고하거나 가공할 수 있는 일종의 filter

- HandlerMapping : Url Mapping하여 controller를 찾거나 HandlerEecutionChain전달함

  - HandlerInterceptor는 ServletFilter와 유사

  - request, response, controller bean object, ModelAndView, Exception등을 전달받아 처리 가능

## HandlerInterceptor

- HandlerInterceptor 인터페이스를 상속받아 구현

- 메소드

  - preHandle(request, response, handler) throws Exception

  - postHandle(request, response, handler, ModelAnView) throws Exception

  - afterCompletion(request, response, handler, exception) throws Exception

- 적용방법

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  <bean class="...BeanNameUrlHandlerMapping">
    <property name="interceptors">
      <list>
        <ref bean="simpleInterceptor"/>
        <ref bean="eventInterceptor"/>
      </list>
    </property>
  </bean>
  <bean id="simpleInterceptor" class="..."/>
  <bean id="eventInterceptor" class="..."/>
  ```

  </div>

  </div>

   

- HandlerMapping마다 HandlerIntreceptor를 등록해야 함

  - 동일한 HandlerInterceptor를 HandlerMapping마다 반복 등록(참고 : 위 적용방법 예제)

  - Spring 3.0부터 URL패턴 등록방식으로 HandlerMapping에 일괄적용기능 추가

# Controller 확장 == HandlerAdapter?

page 417

- Controller inteface를 상속받아 사용하는 경우 Controller구현 절차

  - Controller interface를 구현한 공통적인 기본 기능을 추가한 AbstractController 구현

  - 실제 Controller는 AbstractController를 상속받아 구현

- 다양한 Controller 존재 → Adaptor로 적합한 Controller확장 → HandlerAdapter를 구현

  - request, response만 처리하는 Controller

  - Navigation을 처리하는 Controller

  - Session관련 기능 처리하는 Controller 등

- 입맛에 맞는 Controller를 얼마든지 설계하고 적응하기 위해서 HandlerAdapter를 구현

- HandlerAdapter interface를 구현하여 새로운 HandlerAdapter 생성

  - 메소드

    - supports() : 특정 타입 Controller지원 여부

    - handle() : 실제 도출

### SimpleController interface구현 Controller

- SimpleController interface를 구현한 controller예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public class HelloController implements SimpleController {
    @ViewName("/WEB-INF/view/hello.jsp")
    @RequiredParams({"name"})
    public void control(Map<String, String> params, Map<String, Object> model){
      model.put("message", "hello " + param.get("name"));
    }
  }
  ```

  </div>

  </div>

- SimpleController type을 Spring MVC에서 사용하기 위한 HandlerAdapter구현

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public class SimpleHandlerAdapter implements HandlerAdapter {
    public boolean supports(Object handler) {
      return (handler instanceof SimpleController);
    }
    
    public long getLastModified(HttpServletRequest req, HttpServletResponse res, Object handler) {
      return -1;
    }
    
    public ModelAndView handle(...) {
      Method m = ReflectionUtils.findMethod(handler.getclass(), "control", Map.class, Map.class);
      ViewName viewName = AnnotationUtils.getAnnotation(m, ViewName.class);
      RequiredParams requiredParams = AnnotationUtils.getAnnotation(m. RequiredParams.class);
      ((SimpleController)handler).control(params, model);
      return new ModelAndView(viewName.value(), model);
    }
  }
  ```

  </div>
