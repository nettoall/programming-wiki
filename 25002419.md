# Synchronized Collection

- 대부분의 collection은 싱글 Thread 에서만 사용하도록 설계

- Vector, HashTable 은 synchronized method 로 구성되어 multi-thread 환경에서 사용 가능

- ArrayList나 HashMap 등 synchronized metod가 없는 collection 을 multi-thread 환경에서 사용 가능하도록 동기화된 메소드로 wrapping 하는 메소드 제공

<!-- -->

- 메소드

<div class="table-wrap">

|  |  |  |
|----|----|----|
| **리턴 타입** | **메소드** | **설명** |
| List\<T\> | synchronizedList(List\<T\> list) | List를 동기화된 List로 리턴 |
| Map\<K,V\> | synchronizedMap(Map\<K,V\> map) | Map을 동기화된 Map 으로 리턴 |
| Set\<T\> | synchronizedSet(Set\<T\> set) | Set을 동기화된 Set 으로 리턴 |

</div>

- 사용 예

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  List<T> list = Collections.synchronizedList(new ArrayList<T>)();
  Set<E> set = Collections.synchronizedSet(new HashSet<E>));
  Map<K,V> map = Collections.synchronizedMap(new HashMap<K,V>));
  ```

  </div>

  </div>

# 병렬처리 Collection

- 동기화된 collection 은 멀티 스레드 환경에서 안전하나 전체 요소들을 빠르게 처리 못함

- 멀티 스레드가 컬렉션의 요소를 병렬적으로 처리할 수 있는 컬렉션 제공

- ConcurrentHashMap, ConcurrentLinkedQueue

- 부분잠금 기능 제공 : 처리하는 요소만 잠금하고 나머지 부분은 다른 스레드가 사용 가능

- Thread-Safe 한 collection 은 전체 잠금

<!-- -->

- ConcurrentHashMap 기본 생성 예

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  Map<K,V> map = new concurrentHashMap<K,V>();
  ```

  </div>

  </div>

<!-- -->

- ConcurrentLinkedQueue 는 lock-free 알고리즘으로 구현\

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  Queue<E> queue = new ConcurrentLinkedQueue<E>();
  ```

  </div>

  </div>

  - lock-free : 여러 개의 스레드가 동시에 접근시 잠금을 사용하지 않고도 최소한 하나의 스레드가 안전하게 요소 저장 및 얻음
