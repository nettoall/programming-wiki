# Spring Boot 3.0 Release

- Java 17 기준

- EE 9 기준으로 Jakarta EE 10 지원

- 실험적인 Spring Native를 대체하는 GraalVM 으로 native images 생성 지원

- Micrometer 및 Micrometer Tracing으로 관찰 가능성(obsrevability) 향상

- Spring Framework 6.0 기반

## 제약사항

- Java 17 이상을 꼭 사용해야 Spring Boot 3.0 사용 가능

# Micometer와 Micrometer Tracing에 대한 관찰 가능성 향상

- Observability 란 실행중인 시스템 내부 상태를 외부에서 관찰할 수 있는 능력

- 즉, 출력을 검사하여 시스템 내부를 얼마나 잘 이해할 수 있는지에 대한 내용

- 로깅, 메트릭 및 추적의 세가지 기둥으로 구성됨

- 메트릭 및 추적은 Spring Boot 는 Micrometer Observation을 사용

- 고유한 관찰을 생성하려면 ObservationRegistry를 주입할 수 있음

  1.  Spring Boot 3.0은 Micrometer 1.10에 도입된 새로운 관찰 API를 지원. 새로운 ObservationRegistry interface를 사용하여 지표와 추적 모두에 단일 API 제공하여 관찰 생성

      <div class="code panel pdl" style="border-width: 1px;">

      <div class="codeContent panelContent pdl">

      ``` syntaxhighlighter-pre
      @Component
      public class MyCustomObservation {
        // ObservationRegistry 사용 
        private final ObservationRegistry observationRegistry;
        
        public MyCustomObservation(ObservationRegistry observationnRegistry) {
          this.observationRegistry = observationRegistry;
        }
        
        public void doSomething(){
          Observation.createNotStarted("doSomething", this.observationRegistry)
                     .lowCardinalityKeyValue("locale", "en-US")
                     .highCardinalityKeyValue("userId", "42")
                     .observe(() -> {
                            // Execute business logic here
                      });
        }
      }
      ```

      </div>

      </div>
