# 모델 바인딩과 검증

- Controller method에 @ModelAttribute 가 지정된 파라미터를 추가시 다음 3가지 작업이 자동 수행

  - parameter type의 object 생성 (@ModelAttribute User user : user 생성 → User의 default 생성자 정의 필요)

    - @SessionAttributes 에 의해 세션에 저장된 Model object 존재시 세션 객체 가져옴

  - 준비된 Model Object의 Properties에 Web Parameter Binding

    - http 로 전달되는 파라미터를 model object에 바인딩함

  - Model의 값을 검증

    - 필수 값 검증

    - 지정된 값의 범위 검증 등

    - 스프링이 기본저긍로 제공하는 Binding 용 타입 변환 API가 적용

- 첫번째 과정은 앞에서 설명함

<!-- -->

- Spring에서 Binding 이란

  - Object 의 Property에 값을 넣는 것

    - XML 설정 파일을 사용해서 빈을 정의하면 property 태그로 값을 주입

    - Http를 통해 전달되는 request를 Model Object 로 변환

      - HTTP로 전달되는 정보는 기본적으로 text 정보

      - 프로퍼티 타입에 맞게 주어진 값을 적절히 변환 필요

      - Spring 에서 Binding 과정에서 필요한 변환을 위해 두가지 종류의 API 제공

## PropertyEditor

- Java Bean 표준에 정의된 인터페이스

- 유래는 GUI 환경에서 각 프로퍼티에 입력된 값(text)를 각 타입별로 변환하는 것에서 시작

- 이 기능은 Spring에서 default property editor 로 사용함

### Default Property Editor

- PropertyEditor는 XML의 value 뿐만 아니라 @Controller 의 parameter 설정에도 사용

- @Controller 의 메소드에 적용 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @RequestMapping("/hello")
  public void hello(@RequestParam Charset charset, Model model){}
  ```

  </div>

  </div>

  - URL 이 “/hello?charset=UTF-8” 이면 method parameter charset에 charset=UTF-8 값이 바인딩됨

  - PropertyEditor가 적용이 되어 binding됨

- URL “/hello/charset=UTF-8” 서비스 테스트 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @Test
  public void charsetEditor() {
    CharsetEditor charsetEditor = new CharsetEditor();
    charsetEditor.setAsText("UTF-8");
    assertThat(charsetEditor.getValue(), is(instanceOf(Charset.class)));
    assertThat((Charset)charsetEditor.getValue(), is(Charset.forName("UTF-8")));
  }
  ```

  </div>

  </div>

- Binding 과정에서 변환할 parameter 또는 Model Parameter 의 타입에 맞는 PropertyEdittor가 자동으로 선정되어 적용됨

### Custom Property Editor

- Spring 이 default로 등록하는 Property Editor는 20여가지에 불과

- 어플리케이션에서 직접 정의한 타입으로 직접 binding 시 커스텀 프로퍼티 에디터 작성 필요

#### 설명

- User 클래스 내 enum 타입을 가진 level property 를 정의

- spring 은 level 이라는 enum은 어플리케이션에서 직접 정의한 타입으로 paramter에 직접 매핑할 방법 없음

- level enum 정의

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public enum Level {
    GOLD(3, null), SILVER(2, GOLD), BASIC(1, SILVER);
    ...
    public int intValue() { return value;}
    
    public static Level valueOf(int value) {
      switch(value) {
        case 1: return BASIC;
        case 2: return SILVER;
        case 3: return GOLD;
        default: throw new AssertionError("Unknown value: " + value);
      }
    }
  }
  ```

  </div>

  </div>

  - int 값에 대응되는 Level Object를 가져오는 valueOf() , Level Object 에서 값을 읽어 오는 intValue() 제공

- URL “/user/search?level=1 “ 에 대응하는 Controller method 구현

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @RequestMapping("/user/search")
  search(@RequestParam Level level) {...}
  ```

  </div>

  </div>

- Level 타입에 대한 Property Editor 구현

  - PropertyEditor 변환을 위한 메소드 (PropertyEditor interface 메소드)

    - setAsText()

    - getAsText()

    - setValue()

    - getValue()

  - 메소드 구조

    - Object 변환 : setAsText() 에 스트링 타입의 문자열을 넣고, getValue()로 변환된 object를 가져옴

    - String 변환 : setValue()로 Object를 넣고 getAsText() 로 변환된 문자열 가져옴

  - PropertyEditor interface 직접 구현하기보다 **PropertyEditorSupport 클래스를 상속**해서 필요한 메소드만 오버라이드하는 것이 효율적임

  - setAsText() 에서는 String 타입의 파라미터를 Object로 변환

  - getAsText() 에서는 Object 를 String으로 변환

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    public class LevelPropertyEditor extends PropertyEditorSupport {
      public String getAsText() {
        return String.valueOf(((Level)this.getValue()).intValue());
      }
      
      public void setAsText(String text) throws IllegalArgumentException {
        this.setValue(Level.valueOf(Integer.parseInt(text.trim())));
      }
    }
    ```

    </div>

    </div>

  - Level 타입 PropertyEditor 테스트

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    @Test
    public void testLevelPropertyEditor() throws Exception {
      LevelPropertyEditor levelEditor = new LevelPropertyEditor();
      levelEditor.setAsText("3");
      assertThat((Level)levelEditor.getValue(), is(Level.GOLD));
    }

    @Test
    public void testLevelPropertyEditor2() throws Exception {
      LevelPropertyEditor levelEditor = new LevelPropertyEditor();
      levelEditor.setValue(Level.BASIC);
      assertThat(levelEditor.getAsText(), is("1"));
    }
    ```

    </div>

    </div>

### @InitBinder

- AnnotationMethodHandlerAdapter가 @RequestParam, @ModelAttribute, @PathVariable 가 있는 controller에서 request를 바인딩해야 할 때 WebDataBinder 만듬

- **CustomPropertyEditor가 필요할 경우, WebDataBinder에 직접 등록해서 적용 필요**

- **WebDataBinder**는 AnnotationMethodHandlerAdapter가 복잡한 과정을 통해 method parameter와 annotation을 분석하고 binding을 진행하는 과정 내부에서 만들어지므로 **외부로 직접 노출 안 됨** =\> **Spring이 제공하는 WebDataBinder 초기화 메소드 사용** (**@InitBinder**)

- **@InitBinder가 부여된 Controller method 정의**

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @InitBinder
  public void initBinder(WebDataBinder dataBinder) {
    dataBinder.registerCustomEditor(Level.class, new LevelPropertyEditor());
  }
  ```

  </div>

  </div>

  - WebDataBinder Object의 registerCustomEditor() 메소드 호출하여 타입변환이 필요한 클래스와 CustomPropertyEditor 등록

  - @InitBinder가 붙은 메소드는 메소드 파라미터를 바인딩하기 전에 자동으로 호출됨

- WebDataBinder 테스트 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  WebDataBinder dataBinder = new WebDataBinder(null);
  dataBinder.registerCustomEditor(Level.class, new LevelPropertyEditor());
  assertThat(dataBinder.convertIfNecessary("1", Level.class), is(Level.BASIC));
  ```

  </div>

  </div>

- @InitBinder에 의해 등록된 customerEditor는 같은 Controller 메소드에서 HTTP request를 바인딩하는 모든 작업에 적용됨

- Binding 적용 대상 : @RequestParam, @CookieValue, @RequestHeader, @PathVariable, @ModelAttribute

- CustomerPropertyEditor 추가는 Controller code를 구현시, 구현하는 개발자의 역할

- WebDataBinder에 CustomPropertyEditor를 등록하는 방법(또는 이유?)

  1.  특정 타입에 무조건 적용되는 PropertyEditor 등록

  2.  특정 이름의 Property에 적용되는 PropertyEditor 등록

- Default PropertyEditor보다 CustomPropertyEditor가 우선 순위를 가짐

- page 534 ~ 537 CustomerPropertyEditor를 WebDataBinder에 등록해서 사용하는 case에 대한 심도있는 설명

### WebBindingInitializer

- @InitBinder는 메소드가 있는 Controller 클래스 안에서만 동작

- **어플리케이션 전반에 걸쳐 사용하려면 WebBindingInitializer interface 구현하여** 사용

- WebBindingInitializer 구현

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public class MyWebBindingInitializer implements WebBindingInitializer {
    public void initBinder(WebDataBinder binder, WebRequest request) {
      binder.registerCustomEditor(Level.class, new LevelPropertyEditor());
    }
  }
  ```

  </div>

  </div>

- WebBindingInitializer를 구현하여 만든 클래스 MyWebBindingInitializer를 Bean 으로 등록

- AnnotationMethodHandlerAdapter의 webBindingInitializer의 property에 DI

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  <bean class="org...AnnotationMethodHandlerAdapter">
    <property name="webBindingInitializer">
      <bean class="sampel...MyWebBindingInitializer" />
    </property>
  </bean>
  ```

  </div>

  </div>

- MyWebBindinginitializer의 initBinder() 메소드는 모든 Controller 메소드 파라미터 Binding 작업에 일괄 적용됨

### Prototype Bean PropertyEditor

- PropertyEditor는 상태를 가지므로 singleton bean이 될수 없음

- 따라서 다른 Spring Bean를 참조해야 할 경우가 있는 경우에는 프로토타입 빈으로 만들어 져야 함

- page 538 ~ 549 까지 설명됨. 필요시 정독

## Converter와 Formatter

- PropertyEditor는 매번 Binding을 할때마다 새로운 Object를 생성해야 함.

- Spring 3.0 부터는 Converter interface로 PropertyEditor를 대체할 수 있음

- Converter는 Singleton bean 등록 가능하면 멀티스레드 환경에서 안전하게 사용 가능

### Converter

- source type 에서 target type 방향으로 단방향 변환만 지원

- PropertyEditor는 양방향 변환 지원

- Converter interface 정의

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public interface Converter<S, T> {
    T convert(S source);
  }
  ```

  </div>

  </div>

- Level enum 을 변환하는 converter 예제(String → Level)

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public class LevelToStringConverter implements Converter<Level, String> {
    public String convert(Level level) {
      return String.valueOf(Level.IntValue());
    }
  }
  ```

  </div>

  </div>

- converter 예제(Level → String

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public class StringToLevelConverter implements Converter<String, Level> {
    public Level convert(String text) {
      return Level.valueOf(Integer.parseInt(text));
    }
  }
  ```

  </div>

  </div>

### ConversionService

- Controller의 Binding 작업에 Converter 설정시 사용

- Converter를 ConversionService 타입의 Object를 통해 WebDataBinder에 설정해서 사용 가능

- ConversionService : 여러 종류의 converter를 이용해서 하나 이상의 타입 변환 서비스를 제공해주는 Object를 만들 때 사용하는 interface

- ConversionService를 구현한 **GenericConversionService 클래스를 Bean 으로 등록해서 사용**

- Spring 3.0에 추가된 새로운 타입 변환 Object GenericConverter, ConverterFactory : 타입 변환 Object 생성 지원

- GenericConverter : 하나 이상의 source-target 타입 변환을 한번에 처리할 수 있는 converter 생성 지원

- ConverterFactory : Generic를 활용해서 특정 타입에 대한 converter object를 만들어 주는 factory 구현할 때 사용

- GeneriacConversionService는 일반적으로 Bean으로 등록하고 필요한 Controller 에서 DI 받아서 @InitBinder 메소드를 통해 WebDataBinder에 설정하는 방식으로 사용

- @MVC 컨트롤러 메소드 파라미터를 위해 사용하는 WebDataBinder에 GenericConversionService를 설정하는 방법

#### WebDataBinder 에 GenericConversoinService 설정 방법

1.  @InitBinder를 통한 수동 등록

    - 일부 Controller에 적용 하거나 하나 이상의 ConversionService 만들어 두고 Controller에 따라 다른 변환 Service 선택시

      1.  GenericConversionService를 상속받아 새로운 클래스 생성

          - addConverter() 메소드를 이용해 Converter Object 등록

      2.  추가할 converter class를 bean 으로 등록. ConversionServiceFactoryBean을 이용해서 property로 DI받은 converter들로 초기화된 GenericConversionService 가져오기

          <div class="code panel pdl" style="border-width: 1px;">

          <div class="codeContent panelContent pdl">

          ``` syntaxhighlighter-pre
          <bean class="...ConversionServiceFactoryBean">
            <property name="converters">  <!-- GeneraicConversionService에 등록할 컨버터 지정-->
              <set>
                <bean class="...LevelToStringConverter" />
                <bean class="...StringToLevelConverter" />
              </set>
            </property>
          </bean>
          ```

          </div>

          </div>

          - controller에 설정

            <div class="code panel pdl" style="border-width: 1px;">

            <div class="codeContent panelContent pdl">

            ``` syntaxhighlighter-pre
            @Controller
            public static class SearchController {
              @Autowired ConversionService conversionService;
              
              @InitBinder
              public void initBinder(WebDataBinder dataBinder) {
                dataBinder.setConversionService(this.conversionService);
              }
            }
            ```

            </div>

            </div>

2.  ConfigurableWebBindingInitializer를 이용한 일괄 등록

    - 모든 Controller에 Converter 적용

      1.  WebBindingInitializer를 상속받아 구현한 class를 bean에 등록

      2.  ConfigurableWebBindingInitializer를 bean 설정 =\> 간편

          <div class="code panel pdl" style="border-width: 1px;">

          <div class="codeContent panelContent pdl">

          ``` syntaxhighlighter-pre
          <bean class="...AnnotationMethodHandlerAdapter">
            <property name="webBindingInitializer" ref="webBindingInitializer" />
          </bean>

          <bean class="...ConfigurableWebBindingInitializer">
            <property name="conversionService" ref="conversionService" />
          </bean>

          <bean id="conversionService" class="...ConversionServiceFactoryBean">
            <property name="converters">
              ...
            </property>
          </bean>
          ```

          </div>

          </div>
