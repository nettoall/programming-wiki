# Spring Core Basics

## Spring Container 란

- 객체를 관리하는 Spring Framework의 프로그램

- Container는 설정 메타데이터를 읽어 객체를 인스턴스화, 구성, 어셈블 처리

- 설정 메타 데이터를 XML 파일, java Annotation, Java Code 로 표현

- Spring Container는 다음과 같은 책임이 있음

  1.  Spring Beans 을 찾고 스캔

  2.  스캔 후 Bean의 객체를 생성

  3.  HAS-A 와 같은 관계를 가지고 있을 때 객체의 연결 및 삽입

  4.  객체의 파기

### Spring containers 의 유형

- containers 는 그 자체로 프로그램

- Spring Framework는 Spring container로 작동하는 두 개의 Java interface 제공 (container 를 생성하는 interface 종류에 따라 container 유형이 분류됨)

#### Bean Factory (interface)

- XML 설정만 지원

- org.springframework.beans 패키지에 소속

- legacy container 로 간주

- 기본으로 XmlBeanFactory 구현 클래스 사용

#### ApplicationContext

- XML, Java, Annotation 등 모든 형태의 설정 지원

- org.springframework.context 패키지에 소속

- 제공하는 구현 클래스

  1.  ClassPathXmlApplicationContext : 설정 파일이 프로젝트 클래스 패스에 위치함

  2.  FileSystemXmlApplicationContext : 설정 파일이 서버의 파일 시스템에 위치

  3.  AnnotationConfigApplicationContext : annotation 설정 기반

### Spring Container 생성하기

- 프로그래밍방식으로 Spring Container 생성

- BeanFactory보다는 ApplicationContext 로 구현(요즘 추세 : ApplicationContext 가 널리 사용됨)

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  ApplicationContext context = new ClassPathXmlApplicationContext("com/dev/spring/example/config.xml");
  ```

  </div>

  </div>

  - ApplicationContext interface 를 구현한 ClassPathXmlApplicationContext 구현 클래스 사용

  - config.xml 파일을 설정 파일로 사용

- Annotation 설정 Spring Container 코드 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  AnnotationConfigApplicationContext context = new AnnotationConfigApplicationContext(AppConfig.class);
  ```

  </div>

  </div>

  - Container context를 생성하기 위해 AnnotationConfigApplicationContext 클래스 사용

  - 설정 클래스로 AppConfig.class 정의

### Spring Container로부터 Bean 얻기

- Spring Container 는 getBean() 메소드 제공

- getBean() 메소드는 여러 매개변수 조합이 있음. 요구사항에 따라 그 중 하나를 사용

- Bean 가져오기 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  ApplicationContext ctx = new ClassPathXmlApplicationContext("com/dev/spring/example/config.xml");
  Person person = ctx.getBean(Person.class);
  System.out.println(person);
  ```

  </div>

  </div>

### Spring Bean 이란

- Spring Container가 제공하는 규칙을 따르는 Spring class

- Spring bean 을 구현할 때 따라야 할 규칙

  1.  클래스는 public 선언

  2.  변수는 private 로 선언

  3.  메소드는 public 으로 정의

  4.  클래스는 패키지 아래에 정의

  5.  다음 두가지 조합 중 최소 하나 제공

      1.  setter, getter 메소드와 기본 생성자

      2.  매개변수가 있는 생성자

  6.  Spring Framework API의 일부인 클래스 또는 interface만 상속받아 구현한 클래스. Spring Framework API는 org.springframework 하위에 존재

  7.  Object 클래스의 toString(), hashcode(), equals() 재정의 가능

  8.  java.io.Serializable interface 구현 가능

  9.  Core Java Annotation, Spring Framework Annotation, 일부 Integration Annotaion 사용 가능. 핵심 Java annotation(Deprecated, Override, SafeVarargs, SupressWarnings) 는 java.lang 패키지 하위에 있음

### 객체를 생성하기 위해 Spring Container 가 필요한 정보?

- Spring Bean 및 Spring 구성정보(bean 이름, 다른 bean 간의 연결 기타 정보 등 구성 메타 데이터)

- Spring 구성 정보는 다음 세가지 형식으로 제공됨

  - XML 기반의 구성

  - Java 기반의 구성

  - Annotation 기반의 구성

- Annotation 기반의 구성 방법을 가장 많이 사용하고 있으며 Spring boot 에서는 XML 기반의 구성을 사용하지 않음

### 구성 메타데이터를 제공하는 방법

#### XML 기반의 구성

- Bean 이름, dependencies, Spring container 에 필요한 기타 서비스 정의를 XML 형식의 구성 파일에 정의

- \<bean\> 태그로 bean 정의하면 하나 내지 여러 파일에 정의될 수 있음

#### Annotation 기반의 구성

- Spring 2.5부터 Annotation 구성 도입

- Annotation 기반 구성을 사용하기 위해 \<context:annotation-config/\> 태그 지정하여 활성화 필요

- 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  <beans>
    <context:annotation-config/>
    <!-- bean definitions go here -->
  </beans>
  ```

  </div>

  </div>

- Spring Boot 에서는 annotation만 정의하여 사용하면 됨(\<context:annotation-config/\> 정의 필요 없음)

- 클래스 상단에 bean 사용 목적에 따라 @Component, @Controller, @Service, @Repository 등 적용

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @Component
  public class Employee {
    private Long id;
    private String name;
    private String department;
  }
  ```

  </div>

  </div>

- @Component는 기본적으로 사용하는 Annotation 임

- Spring Bean Annotation 추가 정보 <a href="https://javatechonline.com/spring-boot-bean-annotations-with-examples/" class="external-link" data-card-appearance="inline" rel="nofollow">https://javatechonline.com/spring-boot-bean-annotations-with-examples/</a>

#### Java 기반의 구성

- Spring 3.0부터 Pure-Java 설정 Container가 제공됨

- @Configuration annotated 된 클래스와 @Bean 정의된 메소드로 정의

- @Bean annotation은 \<bean/\> 요소와 동일한 역할

- @Configuration 정의된 클래스는 클래스내 다른 @Bean 메소드를 호출하여 bean간 종속성 정의

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @Configuration
  public class EmployeeConfig {
    
    @Bean
    public EmployeeBean permanentEmployee() {
      return new EmployeeBean();
    }
    
    @Bean(name="contractEmployee")
    public EmployeeBean otherEmployee(){
      return new EmployeeBean();
    }
  }
  ```

  </div>

  </div>

- 일반적으로 메소드 이름이 bean id로 사용되며, bean id를 정의하고자 할 때는 ‘name’ 속성을 사용해 지정해 준다.

### 언제 BeanFactory container 와 ApplicationContextContainer 구분하여 생성하는지

- 메모리 제한이 있는 소규모의 경량 Spring 기반 어플리케이션(모바일 앱, 임베디드 시스템 앱, IOT 앱 등) 구현시 사용 : BeanFactory

- 웹 앱, 엔터프라이즈 앱, 분산 앱, 데스크 톱 앱 등 메모리 제한이 없는 어플리케이션 : ApplicationContext 선호

- 일반적으로 ApplicationContextContainer 선호
