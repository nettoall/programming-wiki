# 단위 테스트

- 서버를 구동하지 않고 컨트롤러만 단독 테스트

- 컨트롤러와 연관된 비즈니스 컴포넌트를 실행하지 않고 컴포넌트 독립 테스트

- 단위 테스트 환경 설정

## dependency 추가

- spring-boot-starter-test

## 테스트 케이스 작성

- src/test/java 에 테스트 케이스 구현

- junit4

  - @RunWith(SpringRunner.class) : JUnit 제공 runner 가 아닌 SpringRunner 사용을 위한 설정, @SpringBootTest 를 사용하기 위해 필수 선언

  - @SpringBootTest : test에 필요한 설정, Bean 자동 초기화 (@SpringBootApplication 가 동일한 기능)

- junit5

  - @ExtendWith(SpringExtension.class) : 자동 초기화

### MockMvc

- 테스트 하기 위해 전체 환경을 로딩하기 보다 가짜 객체를 만들어서 테스트에 필요한 기능만 가지도록 모킹

- 서블릿 컨테이너를 모킹하기 위해 @WebMvcTest , @AutoConfigureMockMvc 사용

  - ServletContainer, DispatchServlet 객체 모킹

- @WebMvcTest 설정 예

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  import static org.springframework.test.web.servlet.request.MockMvcRequestBuilders get;
  import static org.springframework.test.web.servlet.result.MockMvcResultHandlers.print;
  import static org.springframework.test.web.servlet.result.MockMvcResultHandlers.content;
  import static org.springframework.test.web.servlet.result.MockMvcResultHandlers.status;
  ...
  @RunWith(SpringRunner.class)
  @WebMvcTest
  public class BoardControllerTest {
      @Autowired
      private MockMvc mockMvc;
      
      @Test
      public void testHello() throws Exception {
          mockMvc.perform(get("/hello").param("name", "둘리"))
               .andExpect(status().isOk())
               .andExpect(content().string("Hello:둘리"))
               .andDo(print());
      }
  }
  ```

  </div>

  </div>

  - @WebMvcTest 는 @Controller , @RestController 를 찾아 bean 생성

  - org.springframework.test 에 MockMvcRequestBuilders 에 get, post 등 함수

  - MockMvcResultHandlers 의 메소드 print(), content(), status()

- @AutoConfigureMockMvc 설정

  - @SpringBootTest 에는 웹 어플리케이션 테스트를 지원하는 webEnvironment 속성 존재

  - 해당 속성 생략시 기본값으로 WebEnvironment.MOCK이 설정됨(정의 안 하면 자동으로 설정됨)

    - webEnvironment에 설정할 수 있는 항목\*\*?\*\*

  - @SpringBootTest(webEnvironment=WebEnvironment.MOCK) 설정된 객체에 의존성 주입받으려면 @AutoConfigureMockMvc 선언

  - 설정 예

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    @RunWith(SpringRunner.class)
    @SpringBootTest(webEnvironment=WebEnvironment.MOCK)
    @AutoConfigureMockMvc
    public class BoardControllerTest {
        @Autowired
        private MockMvc mockMvc;
        
        @Test
        public void testHello() throws Exception {
            mockMvc.perform(get("/hello").param("name", "둘리"))
                 .andExpect(status().isOk())
                 .andExpect(content().string("Hello:둘리"))
                 .andDo(print());
        }
    }
    ```

    </div>

    </div>

    - @AutoConfigureMockMvc 가 @Controller, @RestController 뿐만 아니라 @Service, @Repository 객체 로딩

- 컨트롤러만 테스트시 @WebMvcTest 사용, 웹 환경 테스트시 @AutoConfigureMockMvc 사용

- @SpringBootTest 와 @WebMvcTest 는 같이 사용 못함. 둘 다 MockMock 모킹으로 인해 충돌

#### MockMvc 메소드

- ResultActions perform(RequestBuilder)

  - MockMvcRequestBuilders get(), post(), put(), delete() : return MockHttpServletRequestBuilder

  - request parameter, header, cookie 설정

- ResultActions

  - ResultActions andExpect(ResultMatcher) : ResultMatcher는 MockMvcResultMatchers의 static method 로 생성

- MockMvcResultMatchers : 서버의 응답 결과 검증

  - ContentResultMatchers content() : content 검증

  - CookieResultMatchers cookie() : 쿠키 검증

  - ResultMatcher forwardUrl(String) : forwardurl 검증

  - HandlerResultMatchers handler()

  - HeaderResultMatchers header()

  - JsonPathResultMatchers jsonPath(String, …)

  - StatusResultMatchers status() : 응답 상태 코드 검증. 상세 내용은 StatusResultMatchers api 함수 참조

    - isOk() : 상태 코드 200

    - isNotFound() : 404 Not Found

    - isMethodNotAllowed() : 405

    - isInternalServerError() : 500

    - is(int status) : 어떤 응답 상태 코드 인지 확인

  - 뷰/ 리다이렉트 검증

    - view() : view 검증 (andExpect(view().name(“hello”) : 뷰이름이 “hello” 인지 검증)

    - redirectedUrl() : redirect 검증 ( andExpect(redirectedUrl(“/index”) : ‘index’ 화면으로 리다이렉트 했는지 검증)

  - 모델 정보 검증

    - MockMvcResultMatchers.model() : 모델 정보 검증

    - attributeExists(String name) : name에 해당하는 데이터가 Model에 포함되어 있는지 검증

    - attribute(String name, Object value) : name에 해당하는 데이터가 value 객체인지 검증

- ResultActions andDo(ResultHandler)

  - 요청/응답 전체 메시지 콘솔 출력

  - andDo(print()) : ConsolePrintingResultHander 객체 리턴

  - MockMvcResultHandlers.print() : ResultHandler를 구현한 ConsolePrintingResultHandler 객체 리턴

  - ConsolePrintingResultHandler를 andDo() 메소드 인자로 넘겨주면 콘솔에 출력

#### 내장 톰켓으로 테스트

- 서블릿 컨테이너를 구동해 테스트 결과를 알고 싶은 경우

- MockMvc 객체를 Mock up 하여 테스트 하면 Tomcat 구동 안함

- @SpringBootTest 의 webEnvironment 값 설정

  - MOCK(모킹된 서블릿 컨테이너 제공), RANDOM_PORT, DEFINED_PORT, NONE

  - @SpringBootTest(webEnvironment=WebEnvironment.RANDOM_PORT)

- 내장 톰켓 사용시 MockMvc 사용 불가 - TestRestTemplate 등으로 http call 호출 필요

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @RunWith(SpringRunner.class)
  @SpringBootTest(webEnvironment=WebEnvironment.RANDOM_PORT)
  public class BoardControllerTest {
      @Autowired
      private TestRestTemplate restTemplate;
      
      @Test
      public void testHello() throws Exception {
          String result = restTemplate.getForObject("/hello?name=둘리", String.class);
          assertEquals("Hello : 둘리", result);
      }
  }
  ```

  </div>

  </div>

- WebEnvironment 상수 DEFINED_PORT(application.properties 에 정의된 포트 사용), MOCK, NONE, RANDOM_PORT

- 톰캣으로 구동시 application.properties에 spring.main.web-application-type=servlet 설정 확인. none 일 경우 톰캣 구동 안함

### 서비스 계층과 연동하는 컨트롤러 테스트

- 서비스 Layer와 연동하는 컨트롤러 mock 구현

- 서비스를 interface와 implementation class 로 구분 (spring 일반적인 사용법)

### 비즈니스 컴포넌트 mocking

- 서비스 계층의 환경적인 제약이나 컨트롤러 만을 테스트하고 싶을 때 비즈니스 컴포넌트를 mocking

- 사용하려는 서비스 계층을 @MockBean 으로 선언

- 가상의 mock 서비스 클래스가 호출되었을 때 리턴할 가상의 값을 정의

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @RunWith(SpringRunner.class)
  @SpringBootTest(webEnvironment=WebEnvironment.MOCK)
  @AutoConfigureMockMvc
  public class BoardControllerTest {
    @Autowired
    private MockMvc mockMvc;
    
    // 서비스 계층 mocking
    @MockBean
    private BoardService boardService;
    
    @Test
    public void testHello() throws Exception {
        // 호출될 때 리턴할 가상의 결과값 설정
        when(boardService.hello("둘리")).thenReturn("Hello : 둘리"); 
        
        mockMvc.perform(get("/hello").param("name","둘리"))
          .andExpect(status().isOk())
          .andExpect(content().string("Hello : 둘리"))
          .andDo(print());
    }
  }
  ```
