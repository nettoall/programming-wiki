# NPM

- Node Package Manager

- 자바 스크립트 프로그래밍 언어를 위한 패키지 관리자

## 프로젝트 생성

1.  프로젝트를 진행할 디렉토리 생성

2.  npm init

    - node.js 에서 package name

    - package.json 생성됨 (package.json : 프로젝트 정보와 패키지의 의존성을 일괄 관리)

    - npm init --yes (-y) : default 설정값으로 package.json 생성

      - 항목 : dependencies (패키지의 이름과 버전)

      - devDependencies (개발 시에만 사용하는 개발용 의존 패키지 명시)

3.  npm install

    - 지역 설치

      - 옵션 별도로 지정하지 않을 시

      - 프로젝트 루트 디렉토리에 node_module 자동 생성 및 패키지 설치

      - 설치돈 패키지는 해당 프로젝트 내에서만 사용됨 → 지역 설치의 의미

    - 전역 설치

      - -g 옵션 사용

      - 모든 프로젝트가 공통 사용하는 패키지 전역으로 설치

      - 설치 경로 : C:\Users\\UserName%\AppData\Roaming\npm\node_modules

      - mac : /usr/local/lib/node_modules

      - 전역으로 모듈을 설치하게 되면 설치한 모듈명을 명령어로 사용 가능

        - npm -g install nodemon

        - nodemon \<패키지명\>

      - 전역으로 설치시 package.json에 기록이 되지 않음

      - -D 옵션으로 devDependencies에 설치하고 npx 명령으로 global하게 실행하는 방식을 자주 사용

        - npm install -D nodemon

        - npx nodeman \<패키지명\>

      - --save-dev (-D) 패키지 설치와 함께 package.json의 devDependencies에 설치된 패키지와 버전 기록됨

    - npm install 명령어 패키지명 @버전 : 패키지 버전 지정 가능

    - 버전 정보 앞에 기호 부여

      - \>Version: 명시된 Version보다 높은 버전

      - \>=Version : 명시된 version과 같거나 높은 버전

## NPM 명령어

### 패키지 제거

- 패키지 제거

  - npm uninstall \<package-name\>

- 전역 패키지 제거(-g)

  - npm uninstall -g \<package-name\>

### 패키지 업데이트

- 패키지 업데이트

  - npm update \<package-name\>

- 전역 패키지 업데이트(-g)

  - npm update -g \<package-name\>

### 패키지 목록 조회

- 사용법

  - npm list

  - npm ls

  - npm ls \<package-name\> : 특정 패키지 조회

- 전역 패키지 조회

  - npm list -g

  - npm ls -g

  - npm ls -g --depth=0

### npm 실행(package.json 실행)

- 명령어

  - npm start

### 패키지 버전 변화 확인

- 명령어

  - npm -v

  - npm outdated

### 패키지 버전 올림

- 명령어

  - npm version patch

  - npm version nimor

  - npm version major

### 패키지 배포

- 명령어

  - npm deprecated \[패키지명\]\[버전\]\[메시지\]

  - npm publish

### 패키지 배포 중단

- 명령어

  - npm unpublish --force

### 패키지 검색

- 명령어

  - npm search \<검색어\>

### 사용자 확인

- 명령어

  - npm login

  - npm whoami

- 로그아웃

  - npm logout

# PM2

- 프로세스 매니저

- 서비스를 제공하는 도중 중지될 시 모니터링하다고 재시작 시켜주거나

- Node.js 가 싱글 스레드 기반인데 멀티 코어 혹은 하이퍼 스레딩 사용 지원

- npm 설치 명령 : npm install pm2 -g (g global 로 설치)

## 명령어

- pm2 -version : 버전 확인

- pm2 start example.js : example.js 를 데몬으로 시작

  - 옵션

    - --watch : 실행 프로세스를 감시하다가 이상시 재시작

    - -i max( 코어개수) : 코어 개수로 클러스터링

- pm2 kill : 모든 프로세스 종료

- pm2 log : log 확인

- pm2 monit : pm2로 실행한 프로세스 상황 확인

- pm2 status

- pm2 delete \[프로세스ID\]

- pm2 restart \[프로젝트 실행 파일\]

- pm2 reload \[프로젝트 실행 파일\] : 프로세스 죽이지 않고 바로 적용

- pm2 unstartup

## Tip

- exam01 예제에서 pm2 로 main.js 를 실행시켰을 경우, 부가적으로 data 디렉토리의 파일이 추가나 수정 될 때마다 앱을 리로드하게 됨

- data 디렉토리의 파일의 수정에 영향을 받지 않도록 하려면 다음과 같이 반영
