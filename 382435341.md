# HandlerExceptionResolver

page 446

- Controller작업중 발생한 예외 처리

- 일반적으로 controller나 그 이후의 Layer에서 발생한 exception은 dispatcherservlet이 처리( 500 Internal Server Error로 처리)

- web.xml에 \<error-page\>로 설정하여 jsp로 error출력을 하도록 할 수 있음

- ExceptionResolver를 등록하여 Servlet이 예외발생시 우선적으로 HandlerExceptionResolver 에서 처리하도록 설정 가능

- HandlerExceptionResolver interface 상속받아 구현(HandlerException 처리) 시 다음 메소드 구현

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  ModelAndView resolver(HttpServletRequest request, HttpServletResponse response, Object handler, Exception exception)
  ```

  </div>

  </div>

- 처리 불가능한 예외시 null 을 return하도록 처리

- Spring에서 총 4개의 HandlerExceptionResolver제공하며 그 중 3개는 Deault로 설정됨

## AnnotationMethodHandlerExceptionResolver

- @ExceptionHandler annotation이 붙은 메소드를 찾아 예외처리를 맡겨주는 Resolver

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @ExceptionHandler(DataAccessException) // DataAccessExcception처리하는 메소드 정의
  public ModelAndView dataAccessException(DataAccessException ex) {}
  ```

  </div>

  </div>

- 특정 controller의 작업 중에 발생하는 예외만 처리하는 예외 핸들러 구현시 사용

## ResponseStatusExceptionResolver

- 예외를 특정 HTTP 응답상태코드로 전환

- 예외 클래스에 @ResponseStatus를 붙이고 HttpStatus에정의되어 있는 HTTP 응답상태값을 value element에 지정

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @ResponseStatus(value=HttpStatus.SERVICE_UNAVAILABLE, reason='서비스 일시 중지')
  public class NotInServiceException extends RuntimeException {}
  ```

  </div>

  </div>

- @ResponseStatus를 정의한 별도의 exception 클래스 구현 필요

- 기존의 예외에 HTTP응답상태코드 부여시 @ExceptionHandler 방식의 method사용

## DefaultHandlerExceptionResolver

- spring 에서 내부적으로 발생하는 주요 예외를 처리하는 표준 예외 처리 로직을 담고 있음

- 다른 HandlerExceptionResolver를 등록해서 DefaultExceptionResolver가 자동으로 적용되지 않는 경우, DefaultHandlerExceptionResolver는 명시적으로 등록해서 적용하도록 하는게 좋음 =\> 이 ExceptionResolver는 유용하다는 말이네!!!

## SimpleMappingExceptionResolver

- 예외를 매핑처리하게 지원

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  <bean class="org...SimpleMappingExceptionResolver">
    <property name="mappingHandlers">
      <props>
        <prop key="DataAccessException">error/dao</prop> // DataAccessException : Exception , error/dat : view 이름
        <prop key="BusinessLogicException">error/login</prop>
      </props>
    </property>
  </bean>
  ```

  </div>

  </div>

- default 로 설정이 안 되어 있으므로 등록 필요

# LocaleResolver

- Locale 결정

- AcceptHeaderLocaleResolver : Http header 의 지역정보 사용

- 사용자 지정 SessionLocaleResolver 나 CookieLocaleResolver 사용

# MultipartResolver

- 파일 업로드, multipart format 요청정보 처리

- Apache commons Fileupload 라이브러리 사용하는 Commons MultipartResolver 지원

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  <bean id="multipartResolver" class="org.springframework...CommonsMultipartResolver">
    <property name="maxUploadSize" value="100000" />
  </bean>
  ```

  </div>

  </div>

- MulripartHttpServletRequest로 MulripartFile 처리

## RequestToViewNameTranslator

- Controller 에서 view 이름이나 view object를 돌려주지 않았을 때 HTTP 요청정보를 참고해서 view 이름을 생성해주는 로직

- Default. 로 DefaultRequestToViewNameTranslator 등록됨

  - /hello 면 view이름을 hello 로

  - /admin/user.do 이면 admin/user 등 view이름을 생성해줌(예제)
