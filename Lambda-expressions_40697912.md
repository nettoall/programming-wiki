# 개요

- 익명 클래스처럼 이름이 없는 함수

- 메서드를 인수로 전달

## 목적

- 람다 표현식을 어떻게 만드는지

- 어떻게 사용하는지

- 어떻게 코드를 간결하게 만들 수 있는지

## 람다 표현식이란

- 메서드로 전달할 수 있는 익명 함수를 단순화 한 것

- 단 하나의 메서드를 가진 함수형 인터페이스에 메서드를 인수로 사용 가능

- 메서드 내부에 처리되는 로직을 람다로 표현 가능 (ex: 인자 → 로직처리 )

- 람다 표현식은 메서드 명과 같은 이름을 가질 수 없으나 파라미터 리스트, 바디, 반환 형식, 발생하는 예외 리스트는 가질 수 있다.

- 함수와 같이 이름을 가질 수는 없으나 함수와 같은 구조 파라미터, 바디, 반환, 예외 처리 등이 가능

## 람다의 특징

- 익명 : 메서드와 달리 이름이 없음

- 함수 : 특정 클래스에 종속되지 않으므로 함수라 함. 클래스에 종속되면 메서드라 함

- 전달 : 람다 표현식을 메서드 인수로 전달하거나 변수로 저장할 수 있음

- 간결성 : 익명 클래스처럼 자질구레한 코드 구현 필요 없음

## 람다 표현식의 구조

- 동작 마파리머틑 이용할 때 익명 클래스처럼 자질구레한 코드를 구현할 필요가 없음 → 코드가 간결하고 유연해짐

- 예제1) 익명 클래스를 람다 표현식으로 변환

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  Comparator<Apple> byWeight = new Comparator<Apple>() {
    public int compare(Apple a1, Apple a2) {
      return a1.getWeight().compareTo(a2.getWeight());
    }
  };

  // 람다 표현식 변환
  Comparator<Apple> byWeight = (Apple a1, Apple a2) -> a1.getWeight().comparareTo(a2.getWeight());
  ```

  </div>

  </div>

  - 람다 파라미터 (Apple a1, Apple a2)

  - 화살표 →

  - 람다 바디 a1.getWeight().compareTo(a2.getWeight())

  - (parameters) → expression, (parameters) → { statements;} 로 구성

  - expression과 statements의 차이 중요

    - “Iron Man” : expression

    - return i + j : statements

  - 람다 표현식에는 return이 함축되어 명시적으로 사용하지 않아도 됨

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    (int x, int y ) -> {
      System.out.println("Reulst: " );
      System.out.println(x+y);
    }

    () -> 42
    (List<String> list) -> list.isEmpty()
    ```

    </div>

    </div>

- java8 이전의 list를 정렬하는 간단한 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  List<String> names = Arrays.asList("peter","anna","mike","xenia");

  Collections.sort(names, new Comparator<String>(){
    @Override
    public int compare(String a, String b){
      return b.compareTo(a);
    }
  });
  ```

  </div>

  </div>

- Collections.sort 메소드는 list와 Comparator로 주어진 list의 요소를 정렬하는데 사용

- 자주 익명의 comparator를 생성하여 sort 메소드를 호출함 → comparator를 생성해서 사용하는 대신 좀 더 간단하게 코딩을 할 수 있도록 도와줌

- lambda expressions

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  Collections.sort(names, (String a, String b) -> {
    return b.compareTo(a);
  });
  ```

  </div>

  </div>

- 더 간단하게 작성하면 braces “{}”와 return 키워드를 생략할 수 있다.

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  Collections.sort(names, (String a, String b) -> b.compareTo(a));
  ```

  </div>

  </div>

- java는 매개변수 유형을 인식할 수 있기 때문에 생략 가능하다.

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  Collections.sort(names, (a,b) -> b.compareTo(a));
  ```

  </div>

  </div>
