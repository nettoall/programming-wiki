# 릴리즈 시기

- 2022년 11월에 Spring Framework 6 공개됨

- 2022년 11월 Spring Boot 3도 Spring Framework 6 기반으로 출시됨

# 간단한 소개

- Spring Boot 3.0은 Spring Framework 6.0을 지원하는 최초의 Spring Boot GA release 임

- Spring Framework 6.0 release의 가장 큰 변화 중 하나는 이전 버전의 Java 에 대한 지원 중단

# 달라진 점들

## 기본 Java 버전을 Java17로 업그레이드

- Framework와 애플리켕션을 강화하기위해 필요한 최소 JDK 버전은 최신 LTS(장기 지원) 버전인 JDK17

- Java 17에는 언어(텍스트 블록, switch 표현식), 유형 시스템(collection factory method 등), 모듈 시스템(모듈 검사, module path scanning) 등의 측면에서 다양한 새로운 기능이 포함

- Spring Framework6.0은 Java 17의 API를 최대한 활용

- 앞으로 Spring은 사용가능한 최신 JDK/JVM 버전을 사용할 것을 권장

- Spring Framework 6.1 GA는 JDK 두가지 주요 LTS 버전인 JDK17 및 JDK21의 지원과 함께 출시될 예정

## Namesapce를 Java EE 에서 Jakarta EE로 변경

- Spring Framework 는 Java EE 에서 Jakarta EE 로 마이그레이션 됨

- javax 네임 스페이스 대신 jakarta 패키지 네임 스페이스 사용

  - javax.servlet 대신 jakarta.servlet 으로 업그레이드 됨

- Jakarta 9가 Spring Framework 6의 기준이지만 사용 가능한 최신 버전 10을 권장

- Tomcat 10은 Jakarta EE 9를 기반으로 하나 Tomcat 10.1 는 Jakarta EE 10 기반

## AOT(Ahead-Of-Time): bean 정의의 빌드 타임 전처리

- Spring 6에 도입된 새로운 개념

- 배포 전에 클래스 컴파일 및 패키징

- AOT는 컴파일과 패키징 사이에 있음

- AOT 개념은 빌드시 추가 단계를 수행하고 시작 시간을 줄이려는 경우에 유용

- 추가 빌드 설정이 필요하고 런타임시 유연성이 떨어짐

- Spring 애플리케이션을 분석하고 모든 runtime hint를 읽고 GraalVM 네이티브 이미지에 피드하고 GraalVM이 작업을 수행

- 더 상세한 내용은 검색 및 자료 수집 필요

## GraalVM 네이티브 수행

- GraalVM은 Java 및 기타 JVM 언어로 작성된 애플리케이션의 실행 속도를 높이기 위한 고성능 JDK

- GraalVM 네이티브 이미지는 컴파일된 Java 애플리케이션을 미리 처리하여 생성할 수 있는 독립 실행기

- GraalVM 에 대한 다른 자료 필요. 번역이 매끄럽지 않음

- 여기선 AOT와 GraalVM, 그리고 Spring Boot Stack 에 대해 언급

## Core Container 변환

- @Inject annotation은 JSR-330 annotations에 소속됨

- Spring Framework 6 부터는 jakarta.inject 에서 찾을 수 있음

- 마찬가지로 @PostConstruct @PreDestroy 는 JSR-250 annotation 에 있었으나 Spring 6 부터는 jakarta.annotation 에서 소속됨

- LocalVariableTableParameterNameDisconverer는 더 이상 사용되지 않음

- CompletableFuture를 위해 더 이상 ListenableFuture가 사용되지 않음

- SimpleEvaluationConntext는 배열할당을 비활성화함

## HttpMethod를 열거형에서 클래스로 변경

- HttpMethod는 더 이상 열거형이 아님

- EnumSet\<HttpMethod\> 에서 Set\<HttpMethod\> 로 변경

## Web Applications

- Spring 6 에서는 Jakarta EE 마이그레이션으로 인해 웹서버가 undertow-servlet-jakarta 를 사용하여 tomcat 10, jetty 11 또는 Undertow 2.2.19로 업그레이드됨

- javax.servlet에서 가져오기가 jakarta.servlet으로 업그레이드 됨

- 최신 서버를 생성하려면 tomcat 10.1 및 undertow2.3 고려

- Spring Framework 6 는 Apache Commons FileUpload, Apache Tiles, FreeMarker JSP 지원과 같은 오래된 Servlet 기반 통합을 중단

- 파일 업로드 및 필요한 경우 일반 FreeMarker 템플릿 보기를 위해 ‘org.springframework.web.multipart.support.StandardServletMultipartResolver’를 권장

- 새 버전은 REST 지향 웹 아키텍처에 더 중점을 둠

- Spring MVC 및 Spring WebFlux 에 대한 컴프톹러 감지의 변경 사항 도입

  - @RequestMapping annotaion 만으로 controller를 감지하지 않음

  - 이전 버전에서는 @Controller 또는 @RestController annotation 없이도 @RequestMapping 이 있으면 handler로 등록했음

  - Spring 6는 @Controller annotation 이 필요함

## Data Access 와 Transactions

- Spring Framework가 Jakarta EE 로 마이그레이션 중으로 Hibernate ORM은 hibernate-core-jakarta artifact가 있는 Hibernate ORM 5.6.x로 업게르이드 예정

- javax.persistence 에서 jakarta.persistence로 전환해야 함

- Hibernate Validator 생성은 Hibernate Validator 8.0 으로 업그레이드해야함

- CannotSerializeTransactionException, DeadlockLoserDataAccessException 은 deprecated

- PessmisticLockingFailureException 기본 클래스와 CannotAcquireLockException 를 선호 (의미가 애매)

## Servlet Mock 에는 Servlet 6.0이 필요

- Servlet 5.0과 6.0 API jar 간의 주요 변경 사용으로 인해 Spring 에서 제공하는 Servlet Mock(MockHttpServletRequest, MockHttpSession)은 Servlet 6.0이 필요

- Test 클래스에서는 Servlet 6.0 API 에 대해 실행해야 함.

- 컴파일은 Serlvet5.0에서 가능. Serlvet 5.0 기반 컨테이너에서의 실행도 가능하나 mock 기반 테스트는 Serlvet 6.0 기반으로 실행해야 함.

- Spring Boot는 1년에 두번 릴리즈 예정
