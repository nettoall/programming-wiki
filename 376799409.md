# 기타 bean 설정 메타 정보

page 182

- bean 식별자, application context 에서 공유해야 함

- bean id : 공백 no, 특수문자 사용 no, 첫글짜는 소문자, 생략 가능

- name : 식별자, 여러 이름 부여 가능, name=”1234,/hello” “,” 로 구분 \<alias name=”dataSource” alias=”별칭”/\> 다른 이름 별칭 사용

- annotation bean 이름

  - 클래스 이름을 사용, 자동 인식 때

  - 지정시 @Component("myUserService") or @Component @Name("myUserSerivce")@Bean(name="userDao")@Bean(name={"userDao","myDao"})

# Bean Lifecycle Method

- 초기화 메소드

  - 초기화 콜백 인터페이스

    - InitializingBean interface로 bean 구현

    - afterPropertiesSet() 메소드는 프로퍼티 설정까지 마친 후 호출됨

    - 권장하지 않는 방식(프레임워크 부가 기능이라 별도의 jar를 만들때, devon 같은 jar 생성시)

  - init-method 지정

    - xml \<bean\> 태그에 initmethod attribute 추가해서 초기화 작업을 할 메소드 지정

      <div class="code panel pdl" style="border-width: 1px;">

      <div class="codeContent panelContent pdl">

      ``` syntaxhighlighter-pre
      <bean id="myBean" class="MyBean" init-method="init"/>
      ```

      </div>

      </div>

      - \<beans\> 태그 안에 defualt-init-method를 지정해서 bean 초기화 메소드 일괄 지정 방식 지원 → 권장 안함

  - @PostConstruct

    - 초기화 담당할 메소드에 @PostConstruct annotation 부여

    - 사용 권장

  - @Bean(init-method=...)

    - 예제

      <div class="code panel pdl" style="border-width: 1px;">

      <div class="codeContent panelContent pdl">

      ``` syntaxhighlighter-pre
      @Bean(init-method="init")
      public class MyBean {}
      ```

      </div>

      </div>

    - xml의 init-method와 유사한 방법

- 제거 메소드

  - 컨테이너가 종료될 때 호출되서 bean이 사용한 리소스 반환하거나 종료전에 처리해야 할 작업 수행

  - 제거 콜백 인터페이스

    - Disposable Bean interface 구현해서 destory() 메소드 구현, 초기화 콜백와 유사

  - destroy-method

    - xml \<bean\> 태그에 destroy-method attribute 추가

  - @PreDestroy

    - 메소드에 @Predestroy 지정하여 컨테이너 종료전 수행하도록 구현

  - @Bean(destroy-method="...")

    - xml의 destroy-method와 유사

# FactoryBean과 Factory method

- 생성자 대신 오브젝트를 생성해주는 코드의 도움을 받아서 bean object를 생성 : Factory Bean

- 특정 Bean이나 클래스의 Factory Method 를 사용해서 Bean 등록

- FactoryBean interface

  - FactoryBean interface를 구현해서 getObject() 메소드를 구현하고 factory bean 으로 등록해서 사용 → factory bean 자신은 bean object 로 사용되지 않고 Bean Object를 만들어줌

  - 가장 단순, 자주 사용됨

- static Factory Method

  - 클래스의 static method를 호출해서 인스턴스를 생성

  - \<Bean\> 태그의 factory-method attribute 를 이용하는 것인 편리

  - singleton 캘르스는 생성자를 직접 호출해서 object를 못 만들기에 static factory method를 이용하여 object 생성

  - 예제

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    <bean id="counter" class="GlobalCounter" factory-method="create"/>
    ```

    </div>

    </div>

- instance factory method

  - object의 instance method 를 이용해 bean object 생성

  - FactoryBean interface를 구현한 FactoryBean이 getObject() 메소드로 Bean Object 만드는 게 예임

  - FactoryBean interface 구현한 Bean이 아닌 임의의 bean에서 인스턴스 메소드로 Bean Object 생성하려면 Factory-bean과 Factory-method 를 사용하여 인스턴스 팩토리 메소드 설정함

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    <bean id="logFactory" class="...LogFactory"/>  // factory bean 따로 등록
    <bean id="log" factory-bean="logFactory" factory-method="createLog" />
    ```

    </div>

    </div>

- @Bean 메소드

  - @Bean 메소드도 일종의 팩토리 메소드임
