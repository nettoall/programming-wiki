# NIO

- IO : stream 사용, Input, Output 각각 사용

- NIO : buffer 사용, Channel 사용(IN/OUT)

## IO vs NIO

<div class="table-wrap">

|                       |                    |                            |
|-----------------------|--------------------|----------------------------|
| **구분**              | **IO**             | **NIO**                    |
| 입출력 방식           | stream 방식        | channel 방식               |
| 버퍼 방식             | non-buffer         | buffer                     |
| 비동기 방식           | 지원 안함          | 지원                       |
| 블로킹/non블로킹 방식 | 블로킹 방식만 지원 | 블로킹/non블로킹 모두 지원 |

</div>

### Stream vs Channel

- Stream : 입력과 출력시 각각의 stream을 생성해서 사용

- Channel : 양방향으로 입력과 출력이 가능

### Non-buffer vs Buffer

- Non-buffer는 1바이트씩 입력 및 출력, IO는 Stream으로부터 읽은 데이터를 즉시 처리

- Buffer는 복수 개의 바이트를 한꺼번에 입력받고 출력, 읽은 데이터를 무조건 buffer에 저장하기 때문에 버퍼 내에서 데이터의 위치를 이동해 가면 필요한 부분만 읽고 쓸 수 있음

### 사용

- NIO : 연결 클라이언트 수가 많고, 하나의 입출력 처리 작업이 오래 걸리지 않는 경우 사용

- IO : 대용량 데이터 처리, 연결 클라이언트 수가 적고 순차적으로 처리될 필요성이 있을 경우 사용
