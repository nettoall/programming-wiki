## NPM

- Node.js의 패키지 관리 도구

- Node.js 패키지를 관리하는 도구

- Node.js 설치시 자동 설치됨

### 패키지

- `Node.js의 패키지에는 모듈에 필요한 모든 파일이 포함되어 있습니다`

- 프로젝트에 포함할 수 있는 자바스크립트 라이브러리

### 패키지 다운로드

- npm install \<패키지명\>

- My Project 하위에 node_modules 디렉토리에 다운로드한 패키지가 설치됨

### 대체 패키지 관리도구

- yarn

  - npm의 알려진 단점. 패키지 종속성 해결이 쉬운만큼 패키지 중복 설치 와 같은 문제 해결을 지원

  - lema workapce 패키지와의 궁합으로 효율적으로 다중 모듈 패키지를 관리

  - workspace 기능을 통한 다중 모듈이 요구되는 국내 환경에 많이 사용하고 있는 패키지 관리자

- pnpm

  - 빠른 속도와 디스크 용량 절약을 내세우는 패키지 관리자

  - 다양한 기능과 신속성, 절약성 때문에 패키지 프로젝트에서 채택

  - package.json에 정의한 패키지만 설치

#### 주요 패키지

- Express.js : 웹 프레임워크

- Mongoose : 비동기 환경에서 작동할 수 있도록 설계된 MongoDB 오브젝트 모델링 툴

- EJS : 템플릿을 사용해 HTML을 동적으로 생성할 수 있게 하는 패키지

<!-- -->

- 자바스크립트로 만든 다양한 패키지를 <a href="https://www.npmjs.com/" class="external-link" rel="nofollow">npm 온라인 데이터베이스</a>에 올림

- npm,yarn 으로 cli에서 패키지 설치,제거, 버전 관리,의존성 관리 등 수행

- 패키지 : npm에 업로드된 node 모듈

## yarn

- 패키지 매니저

### 설치

#### mac

- homebrew로 설치

- brew install yarn

- nodejs 제외하고 설치 : brew install yarn --without-node

#### windows

- chocolatey로 설치

- choco install yarn

- scoop을 사용한 설치 : scoop install yarn

#### npm

- npm install -g yarn

- 버전 확인 : yarn --version

### 명령어

- 패키지 정보 : package.json

- 라이브러리 dependency를 다운받을 폴더 생성

  - yarn을 사용할 수 있는 환경 setting

#### 모듈 설치

- 모듈 추가

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  yarn add [package]
  yarn add [package]@[version]
  yarn add [package]@[tag]
  ```

  </div>

  </div>

- devDependencies, peerDependencies, optionalDependencies 설치

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  yarn add [package] --dev
  yarn add [package] --peer
  yarn add [package] --optional
  ```

  </div>

  </div>

- 의존성 모듈 업그레이드

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  yarn upgrade [package]
  yarn upgrade [package]@[version]
  yarn upgrade [package]@[tag]
  ```

  </div>

  </div>

- 모듈제거

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  yarn remove [package]
  ```

  </div>

  </div>

- 기타

  - yarn.lock : 설치된 모듈의 버전을 저장,어디서나 같은 버전과 수조의 의존성을 갖게 함

  - yarn install 시마다 생성됨

  - package-lock.json과 동일한 역할
