## State

- React 에서 입력은 Props, 함수는 props 를 받아 return 으로 UI 출력

- React의 또 다른 입력으로 사용될 수 있는 것은 State

- 컴포넌트를 사용하는 외부자를 위한 데이터(외부 입력으로 받아 사용)

- 컴포넌트를 만드는 내부자를 위한 데이터

- 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  import logo from './log.svg';
  import './App.css';
  import {useState} from 'react';  // state 정의 

  function Article(props) {
    return <article>
    <h2>{props.title}</h2>
    {props.body}
    </article>
  }

  function Header(props) {
    return <header>
    <h1><a href="/" onClick={(event)=>{
      event.preventDefault(); 
      props.onChangeMode();  
    }}>{props.title}</a></h1>
    </header>
  }

  function Nav(props) { 
    const list = []
    for(let i = 0; i < props.topics.length; i++) {
      let t = props.topics[i];
      list.push(<li key={t.id}>
        <a id={t.id} href={'/read/'+ t.id} onClick={event=>{
        event.preventDefault();
        // Nav 에서 클릭시 id 값이 string 으로 넘어감. state 에서 조건식 처리할 때 오류남 -> Number로 변환처리
        props.onChangeMode(Number(event.target.id);  // target :  이벤트를 유발시킨 대상(<a>)
      }}>{t.title}</a></li>)
    }
    return <nav>
    <ol>
      {list}
    </ol>
    </nav>
  }

  function App() {
    // mode 라는 지역 변수 정의, 내부 변수
    //const _mode = useState('WELCOME'); 
    //const mode = _mode[0];
    //const setMode = _mode[1];
    const [mode, setMode] = useState('WELCOME');  // [변수, 함수명] = useState(초기값) 
    const [id, setId] = useState(null);  // 초기값 없을 시 
    
    // 배열 정의
    const topics = [
      {id:1, title:'html', body: 'html is ...'},
      {id:2, title:'css', body: 'cssis ...'},
      {id:3, title:'javascript', body: 'javascript is ...'}
    ]
    let content = null; // content 변수 선언 (Article 내용 정의 )
    if(mode === 'WELCOME') {
      content = <Article title="Welcome" body="Hello, WEB"></Article>
    } else if(mode === 'READ') {
      let title, body = null;
      for(let i = 0; i< topics.length; i++){
        console.log(topics[i].id, id);
        // id state 값과 topic의 id가 일치할 때, <Nav> 정의에서 넘어오는 id 값을 Number로 변환필요
        if(topics[i].id === id) {
          title = topics[i].title;
          body = topics[i].body;
        }
      }
      content = <Article title={title} body={body}></Article>
    }
    return (
      <div>
        <Header title="WEB" onChangeMode={() =>{
          setMode('WELCOME');
        }}></Header>
        <Nav topics={topics} onChangeMode={(_id)=>{
          setMode('READ'); // 값을 바꿀 때는 setMode 메소드 사용
          setId(_id);
        }}></Nav>
        {content}
      </div>
    );
  }

  export default App;
  ```

  </div>

  </div>

  - WEB을 클릭하면 mode 값이 ‘WELCOME’, 밑에 타이틀(html, css, javascript)을 클릭하면 mode가 ‘READ’로 바뀜 =\> UI가 바뀌지 않음. APP 함수가 다시 실행되지 않기 때문에 return 값이 변경되지 않음

  - mode값이 바뀌면 APP 가 재실행되어 새로운 return 값이 UI출력되어야 함 =\> userState 라는 hook import 필요

  - mode 변수를 state로 업그레이드 ( **const \[변수명 ,함수명\] = useState('초기값');** ) state 자주 쓰는 형식

  - 기존 코드에서 **Nav 함수에서 수정 필요**(이벤트에서 클릭한 id값을 넘겨줄 때 string 으로 넘겨줌 → if 조건식에서 처리할 때 string 값이 넘어 오면 오류 발생. Number로 변환 필요)
