# Spring Dependency Injection

## IoC(Inversion Of Control)

- 객체 또는 프로그램 블록의 제어가 컨테이너 또는 프레임워크로 전송되는 소프트웨어 엔지니어링의 개념

- IoC를 사용하면 컨테이너가 프로그램 흐름을 제어하고 라이브러리를 호출하는 대신 사용자 지정 코드를 호출

## Spring Container

- ApplicationContext interface 는 Spring Container 를 의미

- Spring Container 는 Spring Bean을 인스턴스화, 구성 및 조립, 생명 주기 관리

- Spring IoC 컨테이너와 혼용

## Dependency in Spring

- 개념 정리

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public class Invoice {
    String name;
    Double amout;
    List<String> items;
    Vendor vendor;
    TaxDetails taxDetails;
  }

  class Vendor {

  }

  interface TaxDetails {

  }
  ```

  </div>

  </div>

- Invoce class는 이름, 금액, 항목, 공급업체 및 TaxDetails 와 같은 변수가 있는 클래스

- Invoice 객체를 생성하려면 모든 인스턴스 변수가 필요

## dependent bean 정의

- Invoce class는 다른 객체 (Vendor, TaxDetails) 에 의존성이 있기 때문에 의존성 Bean 이라 함

### dependencies 유형

- Spring container는 종속 변수의 데이터 유형이 다음 중 하나인 경우 종속성을 주입 할 수 있음

  - 기본 유형(byte, short, int, long, float, double, boolean, char, String)

  - Collection 유형(List, Set, Map, Properties)

  - 참조유형(사용자 정의 유형) : class 또는 Interface를 데이터 유형으로 사용하여 변수 생성 - Vendor, TaxDetails -

## Object Dependency

- 객체가 다른 객체에 의존하는 경우 이를 객체 종속성이라고 함

- Spring framework 의 Context 에서 bean 이 다른 bean 들에 의존할 때 Bean 종속성(bean dependency)이라고 함.

- parent bean을 depedent bean(종속 bean)이라 하고 bean 내부의 인스턴스 변수를 dependencies(종속성) 라 함

## Spring Dependency Injection

- Injection 은 객체를 생성하면서 변수에 값을 삽입하는 과정

- Spring DI(Dependency Injection) 은 Spring container 가 객체에 다른 객체를 주입하는 Spring framework의 핵심 모듈 기능

- 주입되는 다른 객체는 Spring Bean에 선언된 종속성(dependencies) - 인스턴스 변수-임

## Spring Dependency Injection 방법

1.  XML Based

2.  Java Annotation Based

## Autowiring

- Spring Container는 애플리케이션을 부팅할 때 XML 구성 파일을 읽거나 Java annotation을 스캔하여 bean 간의 관계를 감지, 객체를 만들고 종속성 연결 =\> 이 모든 과정을 자동으로 수행 : Autowiring

- 자동으로 종속성을 연결할 변수에 @Autowiring annotation 정의

### autowire Disambiguation

- 동일한 유형의 두 개 이상의 Bean 에 Autowiring 설정시 NoUniqueBeanDefinitionExceptoin 발생 시킴 : autowire Disambiguation

- 해결하기 위해 명시적으로 주입하려는 Bean을 Spring에 알려야 함

- @Autowired annotation과 함께 @Primary 또는 @Qualifier annotation을 사용

- properties, setters, constructor에 autowire 사용 가능

## Spring Dependency Injection 유형

### Setter Based Injection

- Spring Container 가 setter 메소드를 사용하여 변수에 값 주입

- @Autowired annotation을 setter 메소드 위에 정의

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @Component
  public class Vendor{
    int code;
    String name;
  }

  public class Invoice {
    private Vendor vendor;
    
    @Autowired
    public void setVendor(Vendor vendor) {
      this.vendor = vendor;
    }
  }
  ```

  </div>

  </div>

### Constructor Based Injection

- Spring Container 가 매개변수화된 생성자를 사용하여 변수에 값 주입

- 클래스 생성자 위에 @Autowired annotation 정의

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @Component
  public class Vendor{
    int code;
    String name;
  }

  public class Invoice {
    private Vendor vendor;
    
    @Autowired
    public Invoice(Vendor vendor) {
      this.vendor = vendor;
    }
  }
  ```

  </div>

  </div>

### Field or Property Based Injection

- 클래스의 필드 또는 속성위에 @Autowired annotation 정의하여 필드나 속성에 값 주입

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @Component
  public class Vendor{
    int code;
    String name;
  }

  public class Invoice {
    @Autowired
    private Vendor vendor;
    
  }
  ```

  </div>

  </div>

## @Autowired(required=false) 사용 시점

- Bean이 Spring Container에 등록되어 있지 않을 때 @Autowired annotation을 적용하면 예외 발생됨

- 이 때 @Autowired(required=false) 를 정의하여 종속성이 선택적임을 Spring Container 에 알림

## Spring 에서 Java Collections 주입하는 방법

- bean 클래스 코드 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public class ListCollectionsBean {
    // 속성에 collection 타입의 인스턴스 변수 주입하도록 설정
    @Autowired
    private List<String> listOfLanguages;
    
    public void printLanguageList() {
      System.out.println(listOfLanguages);
    }
  }
  ```

  </div>

  </div>

- 설정 클래스에 ListCollectionsBean 등록 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @Configuration
  public class ListCollectionConfig {
    @Bean
    public ListCollectionBean getListCollectionBean() {
      return new ListCollectionBean();
    }
    
    // collection 타입 정의 
    @Bean
    public List<String> listOfLanguages() {
      return List.of("Java", "Scala", "Python");
    }
  }
  ```

  </div>

  </div>

## Spring 에서 변수에 collection 유형의 bean/객체 참조 삽입

- Bean 구현

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public class MyTestBean {
    private String name;
    
    public MyTestBean(String name){
      this.name = name;
    }
  }
  ```

  </div>

  </div>

- 앞서 정의한 MyTestBean의 List를 클래스 속성으로 정의한 bean 구현

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public class MyCollectionBean {
    // bean/object 의 collection 변수를 주입하도록 설정
    @Autowired(required = false)
    private List<MyTestBean> beanList;
    
    public void printBeanList() {
      System.out.println(beanList);
    }
  }
  ```

  </div>

  </div>

- 설정 클래스에 MyTestBean을 생성하도록 구현

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @Configuration
  public class CollectionConfig {
    @Bean
    public MyTestBean getFirstObject() {
      return new MyTestBean("bean1");
    }
    
    @Bean
    public MyTestBean getSecondObject() {
      return new MyTestBean("bean2");
    }
    
    @Bean
    public MyTestBean getThirdObject() {
      return new MyTestBean("bean3");
    }
  }
  ```

  </div>

  </div>

- Spring container 는 **CollectionConfig에서 생성한 MyTestBean 유형의 특정 bean들을** MyCollectionBean의 **beanList Collection 에 주입**함

- Spring Container 가 특정 순서로 Bean을 주입하도록 하려면 인덱스와 함께 @Order annotation 을 사용

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @Configuration
  public class CollectionConfig {
    @Bean
    @Order(1)
    public MyTestBean getFirstObject() {
      return new MyTestBean("bean1");
    }
    
    @Bean
    @Order(0)
    public MyTestBean getSecondObject() {
      return new MyTestBean("bean2");
    }
    
    @Bean
    @Order(2)
    public MyTestBean getThirdObject() {
      return new MyTestBean("bean3");
    }
  }
  ```

  </div>

  </div>

- Order annotation에 관련된 내용은 <a href="https://javatechonline.com/spring-boot-bean-annotations-with-examples/#Order" class="external-link" data-card-appearance="inline" rel="nofollow">https://javatechonline.com/spring-boot-bean-annotations-with-examples/#Order</a>

## Depedency Inject 장점

1.  객체간의 느슨한 결합 지원

2.  객체간의 느슨한 결합으로 인해 Mock 객체를 주입하여 모듈 테스트하는데 도움이 됨

3.  객체가 서로 독립적이고 주입가능함으로써 코드 재사용 범위가 증가

4.  코드를 더 읽기 쉽게 만듬

## Depedency Inject 단점

1.  느슨한 결합을 제공함으로써 클래스 및 interface 수를 늘리게 됨

2.  개발자가 아닌 Container 가 제어를 가져가게 되고, 그에 따라 개발자가 전체적인 구조와 동작방식을 배우기가 어려움
