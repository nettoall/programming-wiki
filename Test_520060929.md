# 테스트

## 테스트 유형

- 단위 테스트: 비즈니스 로직에서 사용하는 각각의 메소드를 테스트

- 슬라이스 테스트 : 어플리케이션을 특정 계층으로 잘라서 하는 테스트, Mock 객체 사용

- 통합 테스트: 클라이언트 툴 없이 개발자가 짜 놓은 테스트 코드를 실행 시켜서 기대 결과 확인

- 기능 테스트: 사용자 입장에서 어플리케이션의 기능이 올바르게 동작하는지 테스트. 일반적으로 테스트 전문 부서에서 진행 API 툴이나 데이터베이스까지 연관

## F.I.R.S.T 원칙

- 테스트하는데 기준이 될 원칙?

- Fast: 테스트는 빨라야 한다.

- Independent: 각 테스트 케이스는 독립적이러야 한다. 다른 테스트에 영향을 주지 않아야 한다.

- Repeatable: 어떤 환경에서도 반복해서 실행이 가능해야 한다.

- Self-Validating: 성공 또는 실패라는 자체 검증 결과를 보여줘야 한다.

- Timely: 테스트하려는 기능 구현을 하기 직전에 작성해야 한다.

## JUnit

- 오픈 소스 테스트 프레임워크

- Given - When - Then 세 단계로 구성함

- Given: 테스트에 필요한 전제 조건, 테스트 대상에 전달되는 입력 값

- When: 테스트할 동작 지정

- Then: 테스트 결과 검증, 예상 값과 수행 결과를 비교하여 기대한 동작을 수행하는지 검증

### Assertion

#### 전후 처리

- BeforeEach: 각 테스트 직전에 실행 (초기화 작업)

- BaforeAll: 클래스 레벨에서 한꺼번에 실행

- AfterEach

- AfterAll

### Assumption

- 조건부 테스트, 특정 환경에서만 실행되도록 처리

- assumeTrue(System.getProperty(“os.name”).startWith(“Windows”));

## Hamcrest

- JUnit 기반의 단위 테스트에서 사용

- Assertion Framework

- Assertion을 위한 matcher가 자연스러운 문장으로 가독성 좋음

- 테스트 실행 메시지의 가독성

- 다양한 matcher 제공

- http://hamcrest.org/JavaHamcrest/tutorial

## 슬라이스 테스트

- 특정 Layer에서의 테스트

- Mock 객체를 사용하여 다른 Layer의 기능은 모의 결과를 리턴

- Spring 에서 지원하는 annotation을 사용하여 지원

- https://docs.spring.io/spring-boot/docs/current/reference/html/test-auto-configuration.md#appendix.test-auto-configuration

### API 계층 테스트

- given: 테스트용 request body 생성

- when: MockMvc 객체로 테스트 대상 Controller 호출

- then: Controller Handler method의 응답인 Http Status와 Request Body 검증

- 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  //given
  MemberDto.Post post = new MemberDto.Post("ttt@gmail.com", "홍길동", "000-0000-0000");
  String content = gson.toJson(post);

  //when
  ResultActions actions = mockMvc.perform(post("/v11/members")).accept(MeditType.APPLICATION_JSON).contentType(MediaType.APPLICATION_JSON).content(content));

  // then
  MvcResult result = actions.andExpect(status().isCreated()).andExpect(jsonPath("$.data.name").value(post.getName())).andReturn();

### annotation 지원 테스트

- 대표적인 Test

#### @WebMvcTest

- 웹과 관련된 메소드 테스트

- @Controller, @ControllerAdvice, @JsonComponent, Converter, Filter, HandlerInterceptor 등 웹과 관현 bean을 스캔하여 등록

- MockMvc 객체가 자동으로 구성되어 테스트 용이(SpringBootTest 에서는 @AutoConfigureMockMvc 추가 필요)

#### @DataJpaTest

- @Entity 와 Spring Data JPA Repository 타입의 bean 자동 구성

- embedded database를 자동으로 구성하고 Test 에 사용

- 모든 테스트마다 rollback 자동 수행

- 테스트를 위해 TestEntityManager를 주입받아 사용 가능

- spring.jpa.show-sql 자동 설정으로 sql 문구 logging

### @RestClientTest

- RestTEmplate 등을 이용하여 원격 서버와 통신하는 모듈 테스트

- RestTemplateBuilder, RestClient.Builder 가 자동 주입됨(@Autowired 로 설정 가능?)

- MockRestServiceServer를 이용하여 서버 Mocking

- 테스트 하려는 대상 bean은 value 또는 compoennt 속성으로 지정 가능

## Mockito

- @MockBean : Bean 에 대한 Mock 객체 생성 및 주입

- Stubbing: 테스트를 위해 Mock 객체가 항상 일정한 동작을 하도록 지정

- Mockito.any(..): 파라미터에 파라미터 타입 입력

- willReturn(…) : 리턴할 Stub 데이터

- @InjectMocks : 생성한 Mock 객체를 해당 필드에 주입

## Annotation

- @ExtendWith(SpringExtension.class) : JUnit와 Spring을 연동하기 위해 사요하는 annotation. @Autowired 등 annotation을 Test 사용 가능

- @ContextConfiguratoin 을 이용하여 설정 파일을 연결하면 로드된 bean을 injection 받아 사용가능(Spring Boot 도 동일?)

- @ContextConfiguration(classes=ApplicationConfig.class)

- @SpringBootTest : @SpringBootApplication 클래스를 이용하여 ApplicationContext 로딩 → 별도의 @ContextConfiguration 추가 필요 없음
