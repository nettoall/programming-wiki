# 인터페이스의 역할 및 정의

- 개발코드와 객체가 서로 통신하는 접점 역할

- 인터페이스를 통해 사용하는 객체를 변경 가능

- interface 를 선언하여 클래스 정의

- java8 이후 default 메소드와 정적 메소드 선언 가능

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  interace 인터페이스명 {
    //상수
    public int MAX_VOLUME = 10;
    //추상 메소드
    public void turnOn();
    public void turnOff();
    // default 메소드
    default void setSound(boolean onOff) {
      ...
    }
    // 정적 메소드
    static void changeBattery() {
      ...
    }
  }
  ```

  </div>

  </div>

  - 상수 필드

    - public static final 로 선언

    - 인터페이스 상수는 static{} 블록으로 초기화할 수 없기 때문에 선언시 초기값 지정

  - 추상 메소드

    - 인터페이스를 통해 호출된 메소드는 객체에서 실행됨

    - 인터페이스를 상속받은 클래스에서 메소드 구현

    - public abstract 를 생략해도 컴파일 과정에 붙음

  - default 메소드

    - 추상 메소드와 달리 구현 소스를 갖는 메소드

    - 인터페이스 메소드를 직접 호출 가능하지 않음

    - 구현 객체가 있어서 실행 가능(구현 객체에 정의되지 않아도 됨)

    - 모든 구현 객체가 가지고 있는 기본 메소드. 재정의 가능

  - 정적 메소드

    - 클래스의 정적 메소드와 동일한 기능 수행

    - 인터페이스로 바로 호출 가능

# 인터페이스 구현

## 구현 클래스

- interface를 implements 하여 클래스 구현

- interface를 implements 하였지만 메소드를 구현하지 않으면 자동으로 abstract 클래스가 됨

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public abstract class 클래스 implements RemoteControl {

  }
  ```

## 익명 구현 객체

- 일회성의 구현 객체를 만들기 위해 소스 파일 및 클래스 선언하는 것을 대체하는 방법

- 익명 구현 객체 선언 방법

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre

    public void setVolume(int volume){...}
  }
  ```

# 타입변환과 다형성

## 필드의 다형성

- 클래스 필드를 선언할 때 인터페이스로 선언하면 여러 객체를 할당할 수 있으므로 필드의 다형성을 가짐

  - 예제

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    // interface 정의 
    public interface Tire {
      public void roll();
    }

    // 인터페이스를 구현한 클래스 
    public class HankookTire implements Tire {
      @Override
      public void roll() {
        System.out.println("한국 타이어가 굴러갑니다.");
      }
    }

    public class KummhjoTire implements Tire {
      @Override
      public void roll() {
        System.out.println("금호 타이어가 굴러갑니다.");
      }
    }

    // 필드를 인터페이스로 구현한 클래스
    public class Car {
      Tire frontleftTire = new HankookTire();
      Tire frontRightTire = new HankookTire();
      Tire backLeftTire = new HankookTire();
      Tire backRightTire = new HankookTire();
      
      void run() {
        frontLeftTire.roll();
        frontRightTire.roll();
        backLeftTire.roll();
        backRightTire.roll();
      }
    }

    // 필드 다형성 테스트
    public class CarExample() {
      public static void main(String[] args) {
        Car myCar = new Car();
        
        mycar.run();
        
        myCar.frontLeftTire = new KumhoTire();
        myCar.frontRightTire = new KumhoTire();
        
        myCar.run();
      }
    }
    ```

    </div>

    </div>

- 인터페이스 배열로 구현 객체 관리

  - 앞의 예제에서 4개의 Tire 필드를 인터페이스 배열로 관리

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    // 다형성 필드를 인터페이스 배열로 변경
    public class Car {
      Tire[] tires = {
        new HankookTire(),
        new HankookTire(),
        new HankookTire(),
        new HankookTire()
      } ;
      
      void run() {
        for(Tire tire : tires) {
          tire.roll();
        }
      }
    }
    ```

    </div>

    </div>

  - 필드 다형성 테스트 예제

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    // 필드 다형성 테스트
    public class CarExample() {
      public static void main(String[] args) {
        Car myCar = new Car();
        
        mycar.run();
        
        myCar.tires[0] = new KumhoTire();
        myCar.tires[1] = new KumhoTire();
        
        myCar.run();
      }
    }
    ```

    </div>

## 매개변수의 다형성

- 메소드에서 사용하는 매개 변수를 인터페이스를 상속받은 객체로 할당할 경우, 필요에 따라 매개변수를 다른 객체로 변경 가능 : 실행결과가 다양하게 나옴

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public void useRemoteControl(RemoteControl rc){...}
  ```

  </div>

  </div>

- 매개변수를 인터페이스 타입으로 선언, 호출 할 때에는 구현 객체 대입

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public interface Vehicle {
    public void run();
  }
  ```

  </div>

  - 매개변수를 인터페이스로 갖는 클래스 정의

## 강제 타입 변환(Casting)

- 구현 객체가 인터페이스 타입으로 자동 변환하면 인터페이스의 정의된 메소드만 사용 가능

  - 예로 인터페이스 정의 메소드 3개, 구현 객체 메소드 5개일 때, 자동 변환되면 메소드 3개만 사용가능

  - 경우에 따라 인터페이스에 없는 구현 객체에만 메소드 호출이 필요한 경우, 강제 타입 변환하여 호출

  - 강제 타입 변환 예제

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    public interface Vehicle {
      public void run();
    }

    public class Bus implements Vehicle {
      @Override
      public void run() {
        System.out.println("버스가 달립니다.");
      }
      
      public void checkFare() {
        System.out.println("승차요금을 체크합니다.");
      }
    }

    public class VehicleExample {
      public static void main(String[] args) {
        Vehicle vehicle = new Bus();
        vehicle.run();
        // vehicle.checkFare();  실행 안됨 
        
        Bus bus = (Bus)vehicle;  // 강제타입 변환
        bus.run();
        bus.checkFare();
      }
    }
    ```

    </div>

    </div>

## 객체유형확인

- 타입 변환을 하기 전 어떤 객체 유형인지 확인하기 위해 instanceof 으로 객체 타입을 확인

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public class Driver {
    public void drive(Vehicle vehicle) {
      if(vehicle instanceof Bus) {  // Bus가 아닌 다른 객체인 경우  오류 발생
        Bus bus = (Bus)Vehicle;
        bus.checkFare();
      }
      vehicle.run();
    }
  }
  ```

  </div>

# 디폴트 메소드와 인터페이스 확장

## 디폴트 메소드의 필요성

- 기존 인터페이스를 확장해서 새로운 기능을 추가

- 기존 인터페이스의 이름과 추상 메소드 변경 없이 메소드 추가가 가능

  - 인터페이스에 새로운 추상 메소드 추가시 구현 객체에 모든 구현 메소드를 추가해야 함.

  - 디폴트 메소드 추가시 기존 구현 객체에 영향을 미치지 않음

## 디폴트 메소드가 있는 인터페이스 상속

- 부모 인터페이스 정의

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public interface ParentInterface {
    public void method1();
    public default void method2(); // default 메소드
  }
  ```

  </div>

  </div>

### 디폴트 메소드를 상속

- 단순히 상속 받는 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public interface ChildInterface1 extends ParentInterface {
    public void method3();
  }
  ```

  </div>

  </div>

- 구현 객체

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  ChildInterface1 ci1 = new ChildInterface1() {
    @Override
    public void method1(){}
    public void method3(){}
  };

  ci1.method1();
  ci1.method2();  // parentInterface method2() 호출
  ci1.method3(); 
  ```

  </div>

  </div>

### 디폴트 메소드를 재정의해서 실행 내용 변경

- 재정의한 인터페이스

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public interface ChildInterface2 extends ParentInterface {
    @Override
    public default void method2() {}  // 재정의
    public void method3();
  }
  ```

  </div>

  </div>

- 구현객체

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  ChildInterface2 ci2 = new ChildInterface2() {
    @Override
    public void method1(){}
    @Override
    public void method3(){}
  };

  ci1.method1();
  ci1.method2();  // ChildInterface2 method2() 호출
  ci1.method3(); 
  ```

  </div>

  </div>

### 디폴트 메소드를 추상 메소드로 재선언

- 재정의한 인터페이스

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public interface ChildInterface3 extends ParentInterface {
    @Override
    public default void method2(); // 추상 메소드 재선언
    public void method3();
  }
  ```

  </div>

  </div>

- 구현객체

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  ChildInterface3 ci3 = new ChildInterface3() {
    @Override
    public void method1(){}
    @Override
    public void method2(){}
    @Override
    public void method3(){}
  };

  ci1.method1();
  ci1.method2();  // ChildInterface3 구현 객체의 method2() 호출
  ci1.method3(); 
  ```
