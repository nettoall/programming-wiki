# 개요

- Spring Security 3.2부터 Java 설정 지원

- Java Configuration은 SpringSecurityFilterChain 이라는 Application 내 모든 보안 처리를 담당하는 Servlet Filter 생성( Configuration 파일에 @EnableWebSecurity annotation 추가)

- Java Configuration 의 기능

  - URL에 대한 인증

  - 로그인 Form 생성

  - 사용자의 Form 기반 인증 지원

  - 사용자의 로그 아웃 지원

  - CSRF 공격 방어

  - Session Flxation 방어

  - 보안 헤더 통합

    - HTTP Strict Transport Security 로 요청 보호

    - X-Content-Type-Options 통합

    - Cache Control

    - X-XSS-Protection 통합

    - X-Frame-Options 통합으로 클릭 재킹 방어

  - Servlet API 메소드 통합

# 설정

## AbstractSecurityWebApplicationInitializer

- WAR 파일에 SpringSecurityFilterChain을 사용하도록 등록해주는 Spring의 WebApplicationInitializer 사용

## HttpSecurity

- Spring Security 에서 상세한 설정

- 샘플

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @Bean
  public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
      http
          .authorizeRequests(authorize -> authorize
                                 .anyRequest().authenticated() // 어플리케이션에 대한 모든 요청은 사용자가 인증되어야 함
           )
           .formLogin(withDefaults()) // 사용자 인증을 form based login 으로 처리
           .httpBasic(withDefaults());  // 사용자 인증을 HTTP Basic 인증 처리
      return http.build();
  }
  ```

- HttpSecurity를 다중 구성하여 Multiple SecurityFilterChain을 등록할 수 있음

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @Configuration
  @EnableWebSecurity
  public class MultiHttpSecurityConfig {
      @Bean
      @Order(1) // SecurityFilterChain 첫번째로 수행될 설정
      public SecurityFilterChain apiFilterChain(HttpSecurity http) throws Exception {
          http
              .securityMatcher("/api/**") // 오직 URL에 매칭되는 것만 실행
              .authorizedHttpRequest(authorize -> authorize
                                     .anyRequest().hasRole("ADMIN") 
               )
               .httpBasic(withDefaults());  
          return http.build();
      }
      
      @Bean
      public SecurityFilterChain formLoginFilterChain(HttpSecurity http) throws Exception {
          http
              .authorizeHttpRequests(authorize -> authorize  // /api로 시작되지 않는 URL 에 대해 적용
                                     .anyRequest().authenticated()
              )
              .formLogin(withDefaults());
          return http.build();
      }
  }
  ```
