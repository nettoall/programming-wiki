# Spring 개념 정리

## Spring Core 란

- Java 프로그래임 언어를 사용하여 엔터프라이즈 애플리케이션을 구축하는데 도움이 되는 일련의 기능을 제공하는 Spring Framework의 하위 모듈, 핵심 모듈

## Spring Core Container의 주요 특징

- Container 는 Core, Bean, context, expression language 의 네 구성 요소를 다루는 spring 생태계의 핵심

- Core와 Bean은 ID 및 IoC 를 포함하여 Spring framework의 가장 기본적인 기능 제공

- IoC 컨테이너 관리를 담당하여 Bean 인스턴스화, Bean 구성 및 Spring Container에 상주하는 Bean 파괴 등 담당

## Spring DI

- Spring container 가 객체에 다른 객체를 주입하는 핵심 기능

- Spring core에서 종속성 주입은 annotation 및 XML 구성을 사용

## IoC

- 객체 또는 프로그램 블럭의 제어가 Container 또는 Framework로 전송되는 소프트웨어 엔지니어링의 개념

- IoC를 사용하면 컨테이너가 프로그램 흐름을 제어하고 라이브러리를 호출하는 대신 사용자 지정 코드를 호출 할 수 있음

## Spring Context 모듈의 역할(책임)

- Context 모듈은 Beans 모듈의 기능을 상속하며 국제화(I18N), 이벤트 전파, 리소스 로드 및 투명한 Context 생성에 대한 지원 포함

- Context 모듈에는 EJB, JMX 및 기본 원격 지원과 같은 일부 Java EE 기능에 대한 지원도 포함

- ApplicationContext interface는 이러한 기능을 제공하는 Context 모듈의 핵심

## Spring Expression Language Module 의 역할

- Runtime에 객체 그래프를 쿼리하고 조작하기 위한 강력한 expression language 제공

- JSP2.1 사양에 명시된 통합 표현언어의 확장으로 간주

- 이 언어는 속성값 설정 및 가져오기, 속성 할당, 메소드 호출, 배열 컨텍스트 엑세스 .. Spring 의 IoC 컨테이너에서 이름으로 개체 검색 지원

## Spring core에서 bean 이란

- Spring Container에 의해 생성되고 관리되는 Java 클래스

## 클래스나 interface를 Spring Bean 으로 선언할 수 있나

- 추상 클래스나 interface를 제외한 모든 클래스는 Spring bean으로 간주할 수 있음

- POJO 클래스, 간단한 Java Bean 클래스 또는 기타 클래스 일 수 있음

## Spring Container 란

- 애플리케이션에서 Spring Bean 생성, 구성, 관리하는 Spring framework의 핵심 구성 요소

- 객체간의 연결 종속성을 담당

## Spring Container와 다른 java container 간의 차이점

- Spring container 는 Spring 애플리케이션에서 Bean을 관리하도록 설계된 특수 컨테이너

- Bean 생성, 관리, 소멸, 종속성 주입 등 기능 제공

- 서블릿 컨테이너, JSP 컨테이너와 같은 다른 Java 컨테이너는 모든 Java 응용 프로그램에서 각 객체를 관리하는데 사용하는 일반 컨테이너

## Spring core에서 bean scope 란

- bean scope 란 애플리케이션에서 bean의 범위 또는 수명을 정의

- Bean의 인스턴스 생성을 제어하고 Spring container 에 의해 관리

- 기본적으로 Bean의 객체가 인스턴스화 되는 시기, 해당 객ㅊ체의 수명 및 해당 Bean에 대해 생성되는 객체의 수를 제어

- Spring 에는 Singleton, prototype, request, session, global session 같은 bean scope가 있음

## Singleton bean scope 란

- ApplicationContext 에서 bean의 인스턴스가 하나만 있는 Bean 범위

- Spring의 기본 bean scope 임

## prototype bean scope

- 요청될 때마다 bean의 새 인스턴스가 작성되는 bean 범위

## Request bean scope

- 모든 Http 요청에 대해 Bean의 새 인스턴스가 작성되는 bean 범위

## session bean scope

- 모든 HTTP 세션에 대해 Bean의 새 인스턴스가 작성되는 Bean 범위

## global session bean scope

- 모든 global 세션에 대해 bean의 새 인스턴스가 작성되는 bean 범위

## bean writing in spring framework

- ApplicationContext 에서 bean 간의 종속성을 생성하는 프로세스

- Spring container 가 spring 애플리케이션에서 서로 다른 종속 객체 간의 관계를 관리하는 매커니즘

- XML 구성, Java 기반 구성, Annotation 기반의 구성 과 같은 다양한 Bean 연결 방식이 있음

## What is construector injection

- 클래스 생성자에 주입되는 종속성 주입이 한 형태

- Autowiring : 유형에 따라 Bean 간의 의존성을 자동으로 연결하는 매커니즘. Spring Container는 애플리케이션을 부팅할 때 XML 구성파일, Java Annotation 등을 스캔하여 빈간의 관계를 감지하고 자동으로 연결함

- Spring에서 Autowiring 은 이름별, 유형별, 생성자별 로 적용됨

- @Qualifier annotation 은 **동일한 유형의 bean이 여러개 있을 때** autowiring 시 사용할 bean을 특정하는데 사용

- @Value annotation은 Bean의 속성에 값을 주입하는데 사용

- @Import annotation은 하나 이상의 구성 클래스를 다른 구성 클래스내로 가져오는데 사용

- @PropertySource annotation은 애플리케이션에 대한 구성 속성을 포함하는 속성 파일의 위치를 지정하는데 사용

- @Profile annotation은 프로파일을 기반으로 로드되어야 하는 bean을 지정하기 위해 사용. profile은 개발, 테스트 또는 프로덕션과 같은 다양한 환경이나 상황에서 사용하는 구성 세트를 정의하는 방법

- BeanPostProcessor interface는 Spring에서 Bean의 인스턴스화 초기화를 사용자 정의하는데 사용

- BeanFactoryPostProcessor interface는 Spring 에서 BeanFactory의 구성을 사용자 정의하는데 사용

- InitializingBean interface는 Bean이 인스턴스화되고 모든 종속성이 설정된후 초기화 작업을 수행하는데 사용. bean lifecycle 방식을 선언할 때 사용

- DisposableBean inerface는 bean이 소멸되기 전에 정리 작업을 수행하는데 사용

- @PostContruct annotation 는 bean이 인스턴스화 되고 모든 종속성이 설정된 후에 호출되어야 하는 메소드를 정의하는데 사용. InitializingBean interface 대안

- @PreDestroy annotation 는 bean이 파괴되기 전에 호출되어야 하는 메소드를 정의하는데 사용. DisposalBean inerface의 대안

- lazy initialization 은 실제로 필요할 때까지 Bean 생성을 지연시키는 Spring 컨테이너의 기능. 이는 성능을 향상시키고 리소스 소비를 줄이는데 도움
