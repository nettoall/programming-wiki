# 정의

- 클래스 내부에 선언한 클래스

- 두 클래스의 멤버들을 서로 쉽게 접근 가능

- 외부에 불필요한 관계 클래스를 감춤으로 코드의 복잡성을 줄임

- 정의

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  class ClassName {
    class NestedClassName {  // 중첩 클래스 선언
    }
  }

  class ClassName { 
    interface NestedInterfaceName { // 중첩 인터페이스(UI 프로그래밍에서 이벤트 처리시 많이사용)
    }
  }
  ```

  </div>

  </div>

## 중첩 클래스 분류

- 클래스의 맴버로서 선언 : 맴버 클래스

  - 인스턴스 맴버 클래스

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    class A {
      class B() // A 객체를 생성해야만 사용 가능
    }

    A a = new A();
    A.B b = a.new B();
    b.field1 = 3;
    ```

    </div>

    </div>

  - 정적 맴버 클래스

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    class A {
      static class B {}  // A 클래스로 바로 접근할 수 있는 B 중첩 클래스 
    }

    A.C c = new A.C();
    c.field1 = 3;
    a.C.filed2 = 3;
    ```

    </div>

    </div>

  - 바이트 코드 이름 A\$B.class

- 메소드 내부에 선언 : 로컬 클래스 → 메소드가 실행될 때만 사용 가능

  - 정의

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    class A {
      void method(){
        class B{} // method가 실핼할 때만 사용 가능
        B b = new B();
        b.field1 = 3;
      }
    }
    ```

    </div>

    </div>

  - 바이트 코드 이름 A\$1B.class

  - 로컬 클래스에는 접근 제한자(public, private), 정적 클래스 (static) 선언 불가

  - 비동기 처리를 위해 스레드 객체 만들때 사용

## 중첩 클래스의 접근 제한

- 맴버 클래스가 인스턴스 또는 정적으로 선언됨에 따라 바깥 클래스의 필드와 메소드에 사용 제한이 생김

- 맴버 클래스 내부에서 바깥 클래스의 필드와 메소드를 접근할 때에도 제한이 생김

### 바깥 필드와 메소드에서 사용 제한

- 예제 소스

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public class A {
    // 인스턴스 필드
    B field1 = new B();
    C field2 = new C();
    
    // 인스턴스 메소드
    void method1() {
      B var1 = new B();
      C var2 = new C();
    }
    
    // 정적 필드 초기화
    // static B field3 = new B();
    static C field4 = new C();
    
    // 정적 메소드
    static void method2() {
      //B var1 = new B();
      C var2 = new C();
    }
    
    // 인스턴스 멤버 클래스
    class B {}
    
    // 정적 멤버 클래스
    static class C {}
  }
  ```

  </div>

  </div>

  - 정적 맴버 클래스 C는 모든 인스턴스 필드, 메소드, 정적 메소드, 필드에서 사용 가능

  - 인스턴스 맴버 클래스는 정적 필드, 메소드에서 사용 불가

### 맴버 클래스에서 사용 제한

- 인스턴스 맴버 클래스

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  class A {
    int field1;
    void method1(){}
    static int field2;
    static void method2(){}
    
    class B {
      void method(){
        field1 = 10;  // 접근 가능
        method1();    // 접근 가능
        
        field2 = 10;  // 접근 가능
        method1();    // 접근 가능
      }
    }
  }
  ```

  </div>

  </div>

- 정적 맴버 클래스

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  class A {
    int field1;
    void method1(){}
    static int field2;
    static void method2(){}
    
    static class C {
      void method(){
        field1 = 10;  // 접근 불가능
        method1();    // 접근 불가능
        
        field2 = 10;  // 접근 가능
        method1();    // 접근 가능
      }
    }
  }
  ```

  </div>

  </div>

### 로컬 클래스에서 사용 제한

- 로컬 클래스 내부에서는 바깥 클래스의 필드나 메소드를 제한없이 사용

- 메소드의 매개 변수나 로컬 변수를 로컬 클래스에서 사용시

  - 로컬 클래스의 객체는 메소드 실행이 끝나도 힙 메몸리에 존재해서 계속 사용 가능

  - 매개 변수나 로컬 변수는 메소드 실행이 끝나면 스택 메모리에서 사라지기 때문에 사용 제한 =\> 로컬 클래스에서 사용하는 매개 변수나 로컬 변수의 값을 로컬 클래스 내부에 복사해 두고 사용

  - 매개변수나 로컬 변수가 변경될 경우, 로컬 클래스 내부에 복사한 값만 달라지는 걸 막기 위해 final로 선언

- java8부터는 final을 선언 안 해도 한 것으로 취급. final 존재여부의 차이점은 로컬 클래스의 복사 위치, final 유 → 메소드 내의 지역 변수로 복사, 없을시 클래스의 필드로 복사

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  void outmethod(final int arg1, int arg2) {
    final int var1 = 1;
    int var2 = 2;
    
    class LocalClass {
      void method(){
        int result = arg1 + arg2 + var1 + var2;
      }
    }
  }

  // final 로 선언된 필드는 LocalClass의 method 내부에 복사됨
  class LocalClass {
    int arg2 = 매개값; // final 선언 안된 arg2
    int var2 = 2;      // final 선언 안 됨
    void method() {
      int arg1 = 매개값
      int var1 = 1;
      int reulst = arg1 + arg2 + var1 + var2;
    }
  }
  ```

  </div>

  </div>

- java7에서는 final을 꼭 선언해 주어야 필드 수정을 안 해주었으나 java8부터는 선언 안 해도 final 특성을 가지고 있음

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public class Outter {
    // java 7
    public void method1(final int arg) {
      final int localVariable = 1;
      // arg = 100; // final로 선언하여 수정 안 됨
      // localVariable = 100; // final로 선언
      class Inner {
        public void method(){
          int result = arg + localVariable;
        }
      }
    }
    
    // Java 8
    public void method2(int arg) {
      int localVariable = 1;
      //arg = 100; // final 선언 안 해도 final과 같은 처리
      //localVariable = 100; // final 선언 안 해도 final과 같은 처리
      class Inner {
        public void method() {
          int result = arg + localVariable;
        }
      }
    }
  }
  ```

### 중첩 클래스에서 바깥 클래스 참조 얻기

- 클래스 내부에서 this 는 객체 자신의 참조

- 중첩 클래스에서 바깥 클래스를 참조하려면 this 앞에 바깥 클래스 이름을 선언

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public class Outter {
    String field = "Outter-field";
    void method() {
      System.out.println("Outter-method");
    }
    
    class Nested {
      String field = "Nested-field";
      void method(){
        System.out.println("Nested-method");
      }
      void print(){
        System.out.println(this.field);
        this.method();
        System.out.println(Outter.this.field);  // 바깥 객체 참조
        Outter.this.method();  // 바깥 메소드 참조 
      }
    }
  }
  ```

## 중첩 인터페이스

- UI 프로그래밍에서 이벤트를 처리할 목적으로 많이 활용

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  class A {
    interface I{
      void method();
    }
  }
  ```

  </div>

- 버튼 클릭시 이벤트 처리 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public class Buton {
    OnClickListener listener;
    
    void setOnClickListener(OnClickListener listener) {
      this.listener = listener;
    }
    
    void touch() {
      listener.onClick();  // 구현 객체의 onClick() 메소드 호출 
    }
    
    interfacce OnClickListener{  // 중첩 인터페이스 
      void onClick();
    }
  }

  public class CallListener implements Button.OnClickListener {
    @Override
    public void onClick() {

      btn.setOnClickListener(new CallListener());
      btn.touch();
    }
  ```

## 익명 객체

- 클래스를 상속하거나 인터페이스를 구현해야만 생성

- 필드의 초기값이나 로컬 변수의 초기값, 매개 변수의 매개값으로 주로 대입

- UI 이벤트 처리 객체나 스레드 객체를 간편하게 생성할 목적으로 활용

### 익명 자신 객체 생성

- 부모 타입으로 필드나 변수 선언, 자식 객체를 초기값으로 대입

- 자식 객체를 클래스로 선언하기 보다는 익명 클래스로 처리(불필요한 자원 낭비 를 방지하기 위해)

- 일반적으로 자식 객체 초기값 대입 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  class Child extends Parent{}  // Child 라는 클래스를 선언해 주어야 함.

  class A {
    Parent field = new Child();
    void method() {
      Parent localVar = new Child();
    }
  }
  ```

  </div>

  </div>

- 익명 객체로 처리시 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  class A {
    Parent field = new Parent() {  // 익명 자식 객체 생성 
      int childField;
      void childMethod(){}
      
      @Override
      void parentMethod(){}
    };
  }

  class B {
    void method(){
      Parent localVar = new Parent(){   // 메소드내 로컬 변수 익명 자식 객체 생성
        int childField;
        void childMethod(){}
        @Override
        void parentMethod(){}
      }
    }
  }
  ```

  </div>

  </div>

- 메소드의 매개변수가 부모 타입일 때 메소드 내 자식 객체 생성 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  class A {
    void method1(Parent parent) {}
    
    void method2() {
      method1(  // 메소드1의 매개변수에 익명 자식 객체 생성 
        new Parent() {
          int childField;
          void childMethod(){}
          @Override
          void parentMethod(){}
        }
      )
    }
  }
  ```

  </div>

  </div>

### 익명 구현 객체 생성

- 인터페이스 타입으로 필드나 변수 선언, 구현 객체를 초기값으로 대입

- 익명 구현 객체가 아닌 실제 구현 객체 대입 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  class TV implements RemoteControl {}

  class A { 
    RemoteControl field = new TV();  //  필드에 구현 객체 대입
    void method() {
      RemoteControl localVar = new TV();  // 로컬 변수에 구현 객체 대입
    }
  }
  ```

  </div>

  </div>

- 인터페이스 익명 구현 객체 선언 방법은 앞에서 설명한 것과 비슷

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public interface RemoteControl {
    public void turnOn();
    public void turnOff();
  }

  public class Anonymous {
    // 필드 초기값으로 익명 구현 객체 생성
    RemoteControl field = new RemoteControl() {  // 필드 선언 
      @Override
      public void turnOn(){
        System.out.println("TV를 켭니다.");
      }
      
      @Override
      public void turnOff(){
        System.out.println("TV를 끕니다.");
      }
    };
    
    void method1(){
      RemoteControl localVar = new RemoteControl() {  // 로컬변수 선언과 초기값 대입 
      @Override
      public void turnOn(){
        System.out.println("Audio를 켭니다.");
      }
      
      @Override
      public void turnOff(){
        System.out.println("Audiio를 끕니다.");
      }
    };
    // 로컬 변수 사용
    localVar.turnOn();
  }
  ```

  </div>

### 익명 객체의 로컬 변수 사용

- 익명 객체 내부에서는 바깥 클래스의 필드나 메소드 제한 없이 사용

- 메소드의 매개 변수나 로컬 변수를 익명 객체에서 사용할 시

  - 메소드 내에서 생성된 익명 객체는 메소드 실행이 끝나도 힙 메모리에 존재

  - 매개 변수나 로컬 변수는 메소드 실행이 끝나면 스택 메모리에서 제거

- 이 때로 메소드이 매개변수나 로컬 변수를 익명 객체가 사용하기 위해서 final 로 선언되어야 함. (java8 이후 굳이 final 로 선언 안 해도 적용됨)

- 즉, 익명 객체에서 사용된 매개 변수와 로컬 변수는 모두 final 특성을 갖는다.

- 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public interface Calculatable {
    public int sum();
  }

  public class Anonymous {
    private int field;
    
    public void method(final int arg1, int arg2) {
      final int var1 = 0;
      int var2 = 0;
      
      field = 10;
      
      //arg1 = 20;
      //arg2 = 20;
      
      //var1 = 30;
      //var2 =30;
      
      Calculatable calc = new Calculatable() {  // 익명객체에서 사용된 필드와 매개변수는 final 특성을 가짐(arg2, var2 등)
        @Override
        public int sum() {
          int result = field + arg1 + arg2 + var1 + var2;
          return result;
        }
      };
      
      System.out.println(calc.num());
    }
  }
  ```
