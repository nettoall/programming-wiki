# HTTP Client

- Intellij 2023.1 파일 \> 설정 \> 플로그인 : Other Tool HTTP Client (번들)

- <a href="https://www.jetbrains.com/help/idea/exploring-http-syntax.md" class="external-link" data-card-appearance="inline" rel="nofollow">https://www.jetbrains.com/help/idea/exploring-http-syntax.md</a>

## 사용법

- 메소드 요청별 구분

  - \###

- Host Profile 별 관리

  - {{변수명}}

  - 실행환경 변수 지원

    - Project Root 에 http-client.env.json 으로 관리

      <div class="code panel pdl" style="border-width: 1px;">

      <div class="codeContent panelContent pdl">

      ``` syntaxhighlighter-pre
      {
        "local" : {
          "api-server" : "localhost:9090"
        }, 
        "dev" : {
          "api-server" : "dev.test.com"
        }
      }
      ```

      </div>

      </div>

  - 보안 등 숨겨야 할 정보(암호 나 운영에 적용되는 정보 등)

    - http-client.private.json 에 정의 (git 사용시 .gitignore에 등록)

      <div class="code panel pdl" style="border-width: 1px;">

      <div class="codeContent panelContent pdl">

      ``` syntaxhighlighter-pre
      {
        "local":{
          "username":"tom",
          "password":"password"
        }
      }
      ```

      </div>

      </div>

    - http-client.private.json 파일 gitignore에 등록시 다른 사용자가 해당 파일에 대해 감지하도록 힌트 파일을 제공

      - 예 : http-client.private.json.tmp 파일을 생성하여 git 에 반영 (프로젝트 md 파일에 template 파일 사용하라고 알림)

        <div class="code panel pdl" style="border-width: 1px;">

        <div class="codeContent panelContent pdl">

        ``` syntaxhighlighter-pre
        {
          "local":{
            "username":"Your-ID",   // 파일 내용을 알림
            "password":"Your-password"
          }
        }
        ```

        </div>

        </div>

- 메소드 요청 파일

  - \*.http 로 정의 (필요시 http test 메소드만 모아서 하나의 폴더로 관리)

- 인증이 필요한 API 테스트

  - 세션 방식 이용시 http client는 자동으로 요청 간의 세션값을 저장 및 전달 - 별도 관리 필요 없음

  - 토큰 방식 이용시 로직 추가 필요(javascript 사용)

## 토큰 방식 사용시

1.  사이트에서 로그인시 token 값을 리턴

    - Response header의 authorization 에 Bearer 양식에 토큰 정보 전달

2.  사이트 로그인 후 token 정보를 받아다 처리필요

    - 기존 테스트 코드

      <div class="code panel pdl" style="border-width: 1px;">

      <div class="codeContent panelContent pdl">

      ``` syntaxhighlighter-pre
      ### 로그인
      POST {{api-server}}/login
      Content-Type: application/json

      {
        "username":"username",
        "password":"password"
      }

      ### 내 정보 확인
      GET {{api-server}}/verify
      Content-Type: application/json
      Authorization : Bearer ???  // 기존에는 받은 정보 header에서 찾아서 복붙으로 처리
      ```

      </div>

      </div>

    - Javascript 로 Response 를 handle 하여 처리 가능

      <div class="code panel pdl" style="border-width: 1px;">

      <div class="codeContent panelContent pdl">

      ``` syntaxhighlighter-pre
      ### 로그인
      POST {{api-server}}/login
      Content-Type: application/json

      {
        "username":"username",
        "password":"password"
      }

      > {%
      client.test("Validate", function() {
      client.assert(response.status === 200,   
      client.assert(response.contentType.mimeType === "application/json", "Expected 'application/json'))
      client.assert(response.body.accessToken.trim().length > 0);
      });

      client.log(response.body.accessToken);
      client.global.set("access_token", response.body.accessToken)
      %}

      ### 내 정보 확인
      GET {{api-server}}/verify
      Content-Type: application/json
      Authorization : Bearer {{access_token}}
      ```

      </div>

      </div>

      - client : 세션에 대한 메타 데이터(전역 변수 목록을 포함한)를 저장해 놓은 객체

      - response : HTTP Response 객체

# Intellij 와 Docker 연동

## Docker 유형별 연결하는 방법

- Dockerfile

- Docker-image

- Docker-Compose

## Dockerfile이 포함된 프로젝트

1.  Intellij Run/Debug Configuration 실행

2.  Add New Configuration 선택

3.  Docker \> Dockerfile 선택

4.  Docer build, run 시 사용하는 이름, 옵션 등을 입력

- Dockerfile 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  FROM adoptopenjdk/openjdk11
  CMD ["./mvnw","clean","package"]
  ARG JAR_FILE_PATH=target/*.jar
  COPY ${JAR_FILE_PATH} app.jar
  ENTRYPOINT ["java","-jar","app.jar"]
  ```

  </div>

  </div>

- Dockerfile은 최상위 루트 프로젝트 경로에 위치

- Dockefile 내에 정의한 순서대로 Docker 데몬에 해당 명령어 실행

- Dockerfile로 이미지 빌드

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre

  ```

  </div>

  </div>

  - -t : 태그 옵션

- docker image 생성확인

## Intellij docker plugin 사용

- Docker Daemon이 유료화된 이후(정말 유료화됐는지 확인-환경별,OS별 등) 대체 Daemon(Colima, Rancher Desktop)

- plugin에서 Docker 활성화(Ctrl + Alt + S)

- 설정

  1.  Preference에 Build, Execution, Deployment \> Docker 선택

  2.  아무것도 등록되어 있지 않으면 Docker 추가

      - 이름 : Docker

      - WSL을 통해 Docker 데몬에 연결(WSL 사전 설치 필요)

      - 가상 머신 경로, 로컬 경로 설정(docker를 실행하거나 참조할 경로 지정)

        - /d/Dockers, <img src="./media/images/icons/emoticons/72/1f628.png" class="emoticon emoticon-blue-star" data-emoji-id="1f628" data-emoji-shortname=":fearful:" data-emoji-fallback="😨" data-emoticon-name="blue-star" width="16" height="16" alt="(파란별)" />\Projects\docker

- 의문사항 WSL을 구동시키지 않은 상태에서 Docker daemon 작동 하나? → intellij에서 test 결과 docker 연결하고 실행시 잘 동작됨

- Docker 연결 전에는 에러메시지를 Intellij 아래 바에 출력함 : 연결불가: com.intellij.docker.agent.ApiTaskException: Cannot connect to the Docker daemon ap npipe:////./pipe/docker_engine. Is the docker daemon running?

- Docker daemon이 정상적으로 실행되고 있을 경우

  1.  Intellij service에서 Docer connect 수행

  2.  Docker images, 실행중인 containers 목록 조회됨

  3.  로컬에 다운된 이미지 목록으로 container를 실행시, push image로 Docker hub에 업로드 가능

- Docker 연결

- 이미지 창 확장

  - 세가지 방법으로 사용 가능

    1.  Docker registry로부터 pre-build 이미지 가져오기

    2.  Dockerfile로 로컬에서 이미지 생성

    3.  docker registry에 image 올리기

  1.  서비스 창 위에 “+” 를 클릭(+ 명칭 : 서비스 추가)

  2.  Docker 레지스트리 추가

  3.  New Docker Registry 창에서 registry에 접속할 방법을 정의(Local Registry 명 포트와 id/pw 정의 필요)

      - Docker Registry로 Docker hub 추가도 가능

## Docker Compose

- multi-container 어플리케이션 실행할 때 사용

- multi-container docker를 실행하기

  1.  Docker Compose 파일 정의

  2.  Run\|Edit Configuration 창 열기

  3.  Docker 항목에서 Docker-compose 클릭

      - Compose files 선택

      - swl 상의 docker-compose 파일도 선택 가능 : \\wsl\$\Ubuntu-20.04\home/netwhale\posgres\docker-compose.yml

      - 환경변수 설정도 가능(Configuration 창의 옵션 수정 선택)

      - docker-compose up 에 “수정” 클릭하여 옵션 추가

  4.  “실행” 버튼 클릭하여 서비스 시작

  5.  서비스 창에서 Docker-compose 항목에서 수행 내역 볼 수 있음

# intellij에서 빌드시 gcloud logger 설정시 오류 발생

- 로컬환경에서 package 등 빌드시 다음과 같은 오류 발생

- Logback RuntimeException in Action for tag \[appender\] java.lang.IllegalArgumentException: A project ID is required for this service but could not be determined from the builder or the environment.

- 1\. intelllij에 google cloud 관련 plugin 설치(google cloud code) -이 것만으로 해결ㄷ히지는 않음)

- 2\. gcloud cli 설치후 초기화

- 메뉴얼 URL: https://cloud.google.com/sdk/docs/install?hl=ko

- windows (New-Object Net.WebClient).DownloadFile(“<a href="#" rel="nofollow">https://dl.google.com/dl/cloudsdk/channels/rapid/GoogleCloudSDKInstaller.exe”,”$env:Temp\GoogleCloudSDKInstaller.exe”</a>) & \$env:Temp\GoogleCloudSDKInstaller.exe

- 설정이 진행되면 중요한 내역이 Project id랑 gcp 계정, 서비스 region. 네트워크 접속이 안 된다고 해도 Proxy. 설치까지 할 필요 없음

- 설치후 gcloud components 명령어 그룹으로 설치 관리 가능

- gcloud 는 python3기반으로 동작함
