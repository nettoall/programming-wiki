## ServerSocket

- 클라이언트 연결 요청 수락

- 연결이 요청되면 Socket 객체를 리턴

- accept() 는 클라이언트가 연결 요청하기 전까지 블로킹됨

#### 예제\
Server

<div class="code panel pdl" style="border-width: 1px;">

<div class="codeContent panelContent pdl">

``` syntaxhighlighter-pre
public static void main(String[] args){
  ServerSocket listener = new ServerSocket(9090);
  try{
    Socket socket = listener.accept();
    try{
      PrintWriter out = new PrintWriter(socket.getOutputStream(), true);//출력 스트림 얻기
      out.println("test");
    }finally{
      socket.close();
    }
  }finally{
    listener.close();
  }
}
```

## Socket

- 서버에 연결 요청시 사용

- 서버에서 클라이언트와 연결시 사용

- Socket 생성자와 connect() 는 서버 연결될 때까지 블로킹됨

<div class="code panel pdl" style="border-width: 1px;">

<div class="codeContent panelContent pdl">

``` syntaxhighlighter-pre
try {
Socket socket = new Socket("localhost", 5000); // local 및 ip주소 입력시 
or
Socket socket = new Socket(new InetSocketAddress("localhost",5000)); // 도메인명만 알 경우, InetSocketAddress로 IP 주소를 조회
or
Socket socket = new Socket();
socket.connect(new InetSocketAddress("localhost",5000));

if(!socket.isClosed()) {
  try{
    socket.close();
  } catch(IOException ioe){}
}
} catch(UnknownHostException e) {
  // ip, 도메인명이 틀릴 시
} catch(IOException e){
  // 주어진 포트에 접속할 수 없을시 
}
```

#### 데이터 통신

- socket 으로 부터 InputStream, OutputStream 을 얻어 데이터 송수신 처리

- InputStream read()는 메시지를 보내기 전까지 블로킹됨.

- 정상적으로 데이터를 받거나 상대방 socket의 비정상 종료 및 정상 종료가 발생시 대응하는 소켓의 close 처리 필요 (IOException 처리)

### 스레드 병렬처리

- Blocking(대기 상태)가 되는 메소드

  - ServerSocket.accept()

  - Socket 생성 or connect()

  - Socket의 read(), write()

- 병렬처리의 필요성(동시 작업)

  - 스레드가 대기 상태가 되면 다른 작업 실행 못함 → 입출력시 다른 클라이언트의 연결 요청 처리 못함

  - UI생성/변경 스레드에서 블로킹 메소드를 호출하지 않도록 함

- 스레드 병렬처리

  - 클라이언트1이 연결 요청시 서버는 블록킹이 되어 클라이언트2의 연결 요청을 받지 못할 수 있으므로 스레드 병렬처리한다.
