# @RequestMapping Handler Mapping

## 특징

- Handler Mapping 과 handler adaptor의 대상이 Object가 아니라 method

- Handler Mapping을 위해 DefaultAnnotationHandlerMapping 필요 : Default Handler 전략

 

## Class/method 결합 매핑정보

- DefaultAnnotationHandlerMapping은 @RequestMapping 활용

- type(class, interface) 나 method에 @RequestMapping 부여

### URL 패턴

- 사용법

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @RequestMapping(value="/hello", method=RequestMethod.GET, params=”type=admin”, headers=”content-type=text/*”
  ```

  </div>

  </div>

#### String\[\] value() 정의

- 기본 element

- ant 스타일의 와인드 카드 사용 가능

- 사용 예

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @RequestMapping("/hello")
  @RequestMapping("/main*")
  @RequestMapping("/admin/**/user/")
  ```

  </div>

  </div>

- path variable {}, 파라미터 전달시 사용

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @RequestMapping("/usr/{userId}")
  ```

  </div>

  </div>

- 배열로 여러 개 정의 가능

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @RequestMapping("/hello", "/hi") 
  ```

  </div>

  </div>

- 각 URL은 Default suffix pattenr이 적용됨

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @RequestMapping("/hello")
  => /hello, /hello/, /hello.*, /hello.do, /hello.md 과 동일한 결과 
  ```

  </div>

  </div>

#### RequestMethod\[\] method()

- HTTP 요청 메소드

- RequestMethod 는 Enum (GET, HEAD, POST, PUT, DELETE, OPTIONS, TRACE )

- 사용법

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @RequestMapping(value="/user/add", method=RequestMethod.GET)
  ```

  </div>

  </div>

- URL 패턴에 정의된 RequestMethod는 배열 타입으로 하나 이상 지정 가능

#### String\[\] params()

- request로 넘겨주는 파라미터를 정의

- 사용 예

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @RequestMapping(value="/user/add", method=RequestMethod.GET, params="userId=tomcat")
  ```

  </div>

  </div>

- 특정 파라미터가 존재하지 않아야 하는 조건도 지정 가능(! 사용)

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @RequestMapping(value="/user/edit", params="!type") // type 이라는 파라미터가 아예 존재하지 않아야 함
  ```

  </div>

  </div>

#### String\[\] headers()

- HTTP 헤더 정보

- 자주 사용하지 않으나 필요시 사용

- 사용 예

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @RequestMapping(... , headers="content-type=text/*")
  ```

  </div>

  </div>

### 타입레벨 매핑과 method 레벨 매핑

- 타입(class, interface) 레벨 매핑 : class 나 interface 내 모든 매핑용 메소드의 공통 조건을 지정

- method 레벨 매핑 : 세분화시 사용

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @RequestMapping("/user")
  public class UserController {
    @RequestMapping("/add")
    public String add() {
      ...
    }
  }
  ```

  </div>

  </div>

- method 별 GET, POST 세분화시 사용

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @RequestMapping("/user/add")
  public class UserController {
    @RequestMapping(method=RequestMethod.GET)
    public String form() {
      ...
    }
    
    @RequestMapping(method=RequestMethod.POST)
    public String submit() {
       ...
    }
  }
  ```

  </div>

  </div>

 

### 메소드 레벨 단독 매핑

- 메소드 레벨의 매핑 조건에 공통점이 없으면 타입 레벨에 매핑 사용 안함

- 타입 레벨에 조건이 없는 @RequestMapping 부여

- 클래스에 @RequestMapping 을 부여하지 않으면 매핑대상이 안 됨

  - 의문점 : method만 @RequestMapping이 있는 경우에 메소드를 호출 못하나? =\> 테스트 필요

  - 해답 : Controller에 @Controller annotation을 붙이면 class 에 @RequestMapping을 부여 안 해도 됨. 즉, @Controller 가 없으면 class에 @RequestMapping을 붙여야 함

### 타입 레벨 단독 매핑

- <u>handller mapping은 원래 handler object를 결정하는 전략</u>

- <u>다른 타입 controller와의 일관성을 위해 annotation 방식의 handler mapping 에서도 일단 object 까지만 매핑을 하고 최종 실행항 method는 handler adapter가 선정 (정확한 의미가?)</u>

- <u>@RequestMapping을 타입 레벨에 단독으로 사용해서 다른 타입 컨트롤러에 대한 매핑을 위해 사용할 수 있음</u>

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @RequestMapping("/hello")
  public class HelloController implements Controller {
  }

  // Controller interface를 구현한 controller 클래스에 @RequestMapping 부여
  // Handler Mapping과 Handler Adaptor 는 독립적으로 조합될수 있기에 적용 가능한 방법임 
  ```

  </div>

  </div>

- 타입 레벨의 URL 패턴이 /\* 이 끝나는 경우 메소드 레벨의 URL 패턴으로 메소드 이름 사용 가능

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @RequestMapping("/user/*")
  public class UserController {
    @RequestMapping public String add(...) {}  // /user/add 로 호출
    @RequestMapping public String edit(...){}  // /user/edit 로 호출
  }
  ```

  </div>

  </div>

## 타입 상속과 매핑

- @RequestMapping 이 정의된 Controller 상속시 적용됨

- 재정의시 슈퍼 클래스의 @RequestMapping 은 무시됨

- 인터페이스에서의 매핑은 거의 사용하지 않으나 인터페이스 상속받아 구현을 통해 공통 매핑 정보를 일괄적으로 적용할 수 있음 : 이런 경우만 사용한다라고 이해정도…

### 매핑정보 상속의 종류

- 클래스나 인터페이스에서 매핑정보를 상속하고 사용하는 경우

#### 상위 타입과 메소드의 @RequestMapping 상속

- 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @RequestMapping("/user") public class Super {
    @RequestMapping("/list") public String list() {}
  }

  public class Sub extends Super {   // Sub 클래스는 /user/list 를 가진 list() 메소드를 상속받음
  }

  public Sub2 extneds Super {
    public String list(){}   // @RequestMapping을 부여하지 않았기 때문에 /user/list 에 list() 메소드 매핑됨
  }
  ```

  </div>

  </div>

- interface 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @RequestMapping("/user") public interface Intf{
    @RequestMapping("/list") String list();
  }

  public class Impl implements Intf {
    public String list(){}             // @RequestMapping을 부여하지 않았기 때문에 /user/list 가 list() 에 매핑됨
  }
  ```

  </div>

  </div>

#### 상위 타입의 @RequestMapping과 하위 타입의 메소드의 @ReqeustMapping 결합

- 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @RequestMapping("/user") public class Super {}

  public class Sub extends Super {
    @RequestMapping("/list") public String list() {}  // /user/list 에 매핑됨
  }

  @RequestMapping("/user") public interface Intf { public String list();}

  public class Impl implements Intf {
    @RequestMapping("/list") public String list(){}  // /user/list에 매핑됨 
  }
  ```

  </div>

  </div>

#### 상위 타입의 메소드의 @RequestMapping와 하위 타입의 @RequestMapping 결합

- 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public class Super {
    @RequestMapping("/list") public String list() {}
  }

  @RequestMapping("/user")
  public class Sub extends Super {  // /user/list 가 Sub의 상속받은 list() 에 매핑됨
  }
  // 메소드를 오버라이드 했더라고 @RequestMapping을 선언하여 다른 url을 재정의하지 않는 이상 동일한 url에 매핑된다. 
  ```

  </div>

  </div>

#### 하위 타입과 메소드의 @RequestMapping 재정의

- 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @RequestMapping("/user")
  public class Super {
    @RequestMapping(value="/catalog", method=RequestMethod.POST)
    public String list() {....}
  }

  @RequestMapping("/user")
  public class Sub extends Super {  
    @RequestMapping("/list") public String list() {.....}  // 다른 url 및 RequestMethod 등 default 값으로 설정됨
  }
  ```

  </div>

  </div>

#### 서브클래스 메소드의 URL 패턴없는 @ReqeustMapping 재정의

- 하위 타입의 @RequestMapping은 항상 상위 타입의 @RequestMapping 정보를 대신함

- 클래스 상속에서 오버라이드한 하위 메소드에 한해서는 <u>URL 조건이 없는 @RequestMapping</u>을 붙였을 경우 상위 메소드의 @RequestMapping의 URL 조건이 상속됨 : 버그일 수도. 최신 버전에도 동일한지 확인 필요

### 제네릭스와 매핑정보 상속을 이용한 controller 작성

- 전형적인 CRUD 형식을 가진 여러 Controller 를 제너릭스 타입으로 추상 클래스를 만들고 상속받아 사용하도록 정의 가능

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  /**
   * 전형적인 CRUD 형식의 Controller
   */
  public class UserController {
    UserService service;
    
    public void add(User user){...}
    public void update(User user){...}
    public User view(Integer id){...}
    public void delete(Integer id){...}
    public List<User> list() {...}
  }
  ```

  </div>

  </div>

  - 전형적인 CRUD 형식 add, update, view, delete, List 등 업무 Controller 에서 동일한 형식을 보일 때

- 제너릭 추상 클래스 정의 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public abstract class GenericController<T, K, S> {
    S service;
    
    public void add(T entity){...}
    public void update(T entity){...}
    public T view(K id){...}
    public void delete(K id){...}
    public List<T> list() {...}
  }
  ```

  </div>

  </div>

- 제너릭 추상 클래스 상속받은 UserController 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public class UserController extends GenericController<User, Integer, UserService> {} //메소드 정의 필요 없음 
  ```

  </div>

  </div>

- 제너릭 추상 클래스의 메소드 공통 매핑 정보 부여

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  public abstract class GenericController<T, K, S> {
    S service;
    
    @RequestMapping("/add") public void add(T entity){...}
    @RequestMapping("/update") public void update(T entity){...}
    @RequestMapping("/view") public T view(K id){...}
    @RequestMapping("/delete") public void delete(K id){...}
    @RequestMapping("/list") public List<T> list() {...}
  }
  ```

  </div>

  </div>

- 상속받은 클래스에서 타입레벨 @RequestMapping 정의

- public class 위에 @RequeatMapping 정의로 도메인 오브젝트 이름 부여

- /user/add, /user/update 등의 앞에 도메인 url 이 붙는 Generic Controller를 상속받는 클래스 생성

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @RequestMapping("/user")
  public class UserController extends GenericController<User, Integer, UserService> {
    //추가 메소드
    @RequestMapping("login")
    public String login(String userId, Stirng password){...}
  }
  ```

  </div>

  </div>

- 제너릭 추상 Controller를 사용함으로써 컨트롤러 코드는 간결해지고 개발 생산성은 향상 : 주로 이 목적임

참고 : 서비스 계층, DAO 까지 제너릭을 적용하여 사용한 예제 OSAF(<a href="http://whiteship.me/category/OSAF" class="external-link" rel="nofollow">http://whiteship.me/category/OSAF</a>)
