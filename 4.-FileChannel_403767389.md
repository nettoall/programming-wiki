# FileChannel 생성과 닫기

- staic open() 메소드로 FileChannel 얻음

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  FileChannel fileChannel = FileChannel.open(Path path, OpenOption... options);
  ```

  </div>

  </div>

- IO의 FileInputStream, FileOutputStream() 을 통해 얻기도 함

- read, write로 파일억세스

- Path, Paths : NIO에서 파일 접근 객체

- 파일을 읽기 위해 channel open\
  ex) FileChannel.open(path, option…)

  - 파일의 용도를 option으로 지정

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    // file read
    FileChannel.open(path, StandardOpenOption.READ);

    // file write(if file is not exist, create file)
    FileChannel.open(path, standardOpenOption.CREATE, StandardOpenOption.WRITE);
    ```

    </div>

    </div>

- option은 StandardOpenOption (enum)

- NIO는 항시 ByteBuffer 사용으로, 사용할 ByteBuffer를 선언

  - Direct - 시스템 메모리에 버퍼 생성

  - NonDirect - JVM내 heap 메모리에 버퍼 선언\
    ex) ByteBufer buffer = ByteBuffer.allocate(100);

- rea() : 파일 내용을 buffer size 만큼 데이터를 읽어옴\
  ex) int readCnt = FileChannel.read(buffer);

- write() :\
  ex) FileChannel.write(buffer);
