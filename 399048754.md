# 개요

- Network Model

  - 일련의 통신 과정을 여러 계층으로 분리

  - 인접한 계층은 상호 동작, 각 계층은 특정 기능만을 책임

- 계층 구조의 장점

  - 네트워크 통신을 단순화

  - 네트워크 구성 요소 표준화

  - 서로 다른 유형 네트워크 하드웨어/소프트웨어 통신 가능

  - 계층의 변화가 다른 계층에 영향을 미치지 않아 계층 기능 개발 속도 향상

- OSI 7 Layer (TCP/IP Suite)

- Protocol

  - Data message 형식과 교환 방법을 기술한 표준 규칙 집합

- Internet Address

  - 네트워크 주소 + 호스트 주소로 구성

  - subnet mask : IP Address 중 어느 bit까지가 네트워크 ID인지 식별하기 위한 필터

  - 네트워크 ID에 해당되는 부분이 1로 표시

  - IP 주소와 subnetmask를 AND 연산하여 Network ID와 Host ID로 구분

- Classful Addressing

  - 네트워크에 포함된 호스트 수에 다라 서로 다르게 구현되어야 한다는 가정에서 출발한 개념으로 네트워크 규모에 따라 구분

  - 호스트를 표시하는 octet가 모두 0인 경우 : 네트워크 자체 주소

  - 호스트를 표시하는 octet가 모두 1인 경우 : broadcast를 위한 주소

# TCP

## 개요

- 송신지에서 수신지까지 정확하고 신뢰성 잇게 정보 흐름을 제어하고 데이터를 전송하는 역할 수행

## TCP Connection Establishment & Termination

- 데이터 전송을 관리할 논리적인 컨넥션 생성

- 커넥션 생성/종료 의사를 flag로 전달, 상호 합의하에 생성/종료

## TCP 신뢰성

- 송신 패킷에 포함된 전송중인 Byte 정보를 이용하여 패킷의 수신 여부 확인

- 패킷이 정상 전달이 아니라고 판단할 경우 재전송

## TCP 흐름제어

- 송신 패킷에 대한 응답 패킷에 자신이 처리할 수 있는 수신 버퍼 여유 정보 포함

- 송신 호스트는 해당 여유분만큼만 패킷을 송신하고 다음 응답 패킷 기다림

# UDP

## 개요

- TCP의 오버 헤드를 감수하는 대신 좀 더 빠르게 세그먼트를 교환하는 Protocol

## TCP vs UDP
