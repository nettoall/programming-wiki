# Spring Bean 이란

- Java Bean, Bean 클래스, Spring Bean 이란 용어 사용

- 모든 것이 동일하고 서로 바꾸어 사용 가능하다고 생각

- 정확한 Spring Bean의 개념에 대해 정리

## Bean Class/Component Class 란

- Bean은 상태와 동작을 포함하는 Java 클래스

- non-static 멤버는 클래스이 상태, method는 클래스의 동작을 나타냄

- 맴버 변수는 메서드를 통해서만 엑세스 하도록 권장

- 예로 Service 클래스, DAO 클래스 등은 Bean 클래스 또는 Component 클래스로 간주

- Bean와 Component는 같은 의미로 사용됨

## Java Bean와 Bean class 의 차이점

- Bean 클래스에는 비즈니스 로직 메소드가 포함

- Java Bean 은 상태, 동작, getter, setter 가 있는 단순한 클래스(비즈니스 로직 포함 안 함)

## Spring Bean 정의

- **Spring Container에 의해 객체가 생성되고 관리되는 Java Class**

- 추상 클래스와 인터페이스를 제외한 모든 클래스를 Spring Bean 으로 간주

## class 를 Spring Bean 으로 표시하는 방법

- Spring Container가 클래스를 Spring Bean 으로 인식하는 방법

  - XML 구성 파일

  - annotation

### XML 구성 파일 사용

- XML 구성 파일에서 \<bean\> 태그 사용하여 Java class를 Spring Bean 으로 표시

- Spring Bean은 XML 구성 파일을 읽어 Spring Bean class 로딩, 클래스 객체 생성, 클래스 객체 관리, 클래스 파괴 등 Spring Bean의 전체 수명 주기 처리

- Employee 클래스 Spring Bean 정의

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  <bean id="employee" class="com.dev.example.Employee"/>
  ```

  </div>

  </div>

- id 는 Spring Bean의 객체 이름

### Annotation 사용

- Annotation 을 사용하여 Java class 를 Spring Bean 으로 표시

- @Component 를 적용함

- Employee 클래스 Spring Bean 정의

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @Component
  public class Employee {
    ...
  }
  ```

  </div>

  </div>

- annotaion에 이름을 지정하지 않으면 첫문자가 소문자인 클래스 이름이 Spring Bean 이름

  - Spring Bean 이름 정의 : @Component("employee")

- @Component 대용으로 사용할 수 있는 다양한 Stereotype annotation 존재 → 일부 특정 시나리오에서만 사용

- Component scan 중에 Spring Container 는 @Component를 자동으로 발견하고 Application Context 에 Spring Bean 으로 등록

### Java 기반의 구성 사용

- Spring 3.0 부터 순수 Java 구성 Container 제공

- Java 기반의 구성 사용시 XML 구성 필요 없음

- @Configuration 정의된 클래스 와 @Bean 으로 정의된 메소드 로 구성

- @Configuration 을 Bean 정의의 source 로, @Bean 을 Spring Container 에 Spring Bean으로 등록하라고 인식

- @Bean annotation 은 XML의 \<bean/\> 요소와 동일한 역할

- 코드 예제

  <div class="code panel pdl" style="border-width: 1px;">

  <div class="codeContent panelContent pdl">

  ``` syntaxhighlighter-pre
  @Configuration
  public class EmployeeConfig {
    @Bean
    public Employee GetEmployee(){
      return new Employee();
    }
  }
  ```

  </div>

  </div>

- Spring Bean 에서 설정 상속에 대해 다음 절에서 설명

## Spring Bean 에서 설정 상속

- Spring Bean에서의 설정 상속은 두 가지 방법을 제공

  1.  @Bean annotation 에서 ‘parent’ 속성 사용

  2.  XML 설정 파일에서 ‘parent’ 속성 사용

### @Bean Annotation 사용

1.  Configuration 클래스에서 @Bean annotation을 사용하여 상위 Bean 정의

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    @Configuration
    public class ParentConfig {
      @Bean
      public ParentBean parentBean() {
        // return the Bean instance
      }
    }
    ```

    </div>

    </div>

2.  다른 Configuration 클래스를 정의하고 parent 속성과 함께 @Bean annotaion을 사용하여 상쉬 Bean 에서 상속

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    @Configuration
    public class ChildConfig extends ParentConfig {
      @Bean(parent = "parentBean")
      public ChildBean childBean() {
        // return the Bean instance
      }
    }
    ```

    </div>

    </div>

### XML 구성 파일 사용

1.  XML 파일에서 부모 Bean 설정을 정의

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    <beans>
      <bean id="parentBean" class="com.example.ParentBean"/>
    </beans>
    ```

    </div>

    </div>

2.  child Bean 구성을 정의하면서 parent 속성을 사용하여 부모 bean 으로 부터 상속받음

    <div class="code panel pdl" style="border-width: 1px;">

    <div class="codeContent panelContent pdl">

    ``` syntaxhighlighter-pre
    <beans>
      <bean id="childBean" class="com.example.ChildBean" parent="parentBean"/>
    </beans>
    ```

    </div>

    </div>

- child Bean 으로 정의되면 종속성, 초기화, 소멸 방법 등을 포함하여 parent bean의 모든 속성을 상속받음

- 필요한 경우 child bean 에서 parent bean의 속성을 재정의 하거나 확장

Spring Bean 와 함께 사용 가능한 annotation

<a href="https://javatechonline.com/spring-boot-bean-annotations-with-examples/" class="external-link" data-card-appearance="inline" rel="nofollow">https://javatechonline.com/spring-boot-bean-annotations-with-examples/</a>

@Bean 과 @Component 의 차이점

- @Bean 어노테이션은 @Configuratoin 정의된 클래스에서 메소드에 정의할 수 있으나 @Component 는 클래스에만 정의됨

- @Bean 는 scope 설정, dependency 설정 및 Bean 초기화와 같은 Bean 구성에 많은 유연석을 제공하는 반면 @Component는 일반적인 annotatoin 이며 설정 유연성을 제공하지 않음
