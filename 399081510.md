# 보조스트림

- 자체적으로 입출력 수행이 불가능하지만, 입력 및 출력 소스와 연결되어 사용됨.

- 사용 목적에 맞추어 조립됨

  - 문자변환, 입출력 성능 향상, 기본 데이터 타입 입출력, 객체 입출력 등

## 문자변환

- 소스 스트림이 바이트 기반 스트림이지만 입출력 처리는 문자일시

<div class="code panel pdl" style="border-width: 1px;">

<div class="codeContent panelContent pdl">

``` syntaxhighlighter-pre
// 콘솔 입력 처리시
InputStream is= System.in;
Reader reader = new InputStream(is);  // 바이트 스트림은 문자 reader로 변환

// 파일 처리시
FileInputStream fis = new FileInputStream("C:\\temp\\file.txt");
Reader reader = new InputStreamReader(fis);

//파일 출력 처리시
FileOutputstream fos = new FileOutputStream("C\\temp\\fileout.txt");
Writer writer = new OutputStreamWriter(fos);
writer.write("테스트");
```

## 성능향상 처리

- 메모리 버퍼를 사용하여 입출력 처리 향상

<div class="code panel pdl" style="border-width: 1px;">

<div class="codeContent panelContent pdl">

``` syntaxhighlighter-pre
// Stream input 향상
FileInputStream fis = new FileInputstream("C:\\temp\\fileinput.txt");
BufferedInputStream bis = new BufferedInputStream(fis);
while(bis.read() != -1) {}

// reader input 향상
InputStream is = System.in;
Reader reader = new InputStreamReader(is);
BufferedReader br = new BufferedReader(reader);
br.readLine();

// Stream output 향상
FileOutputStream fos = null;
fos = new FileOutputStream("C:\\temp\\output.txt");
BufferedOutputStream bos = new BufferedOutputStream(fos);
bos.write("data");
```

## 객체 입출력

- 객체의 데이터(필드값)을 일렬로 늘어선 연속적인 바이트로 변경하여 출력(객체 직렬화)

- 파일이나 네트워크로부터 전송된 객체를 읽어 객체로 복원(역직렬화)

- ObjectInputStream/ObjectOutputStream

- Serializable 인터페이스 구현한 클래스만 직렬화 가능 (필드값만 가능, 메소드,생성자 제외)

- 필드 선언에 static, transient 는 직렬화 제외

- 직렬화시 동일한 클래스인지 확인을 위해 serialVersionUID 사용
